El SDK de Java para el [Protocolo de Contexto de Modelo (MCP)](https://modelcontextprotocol.org/docs/concepts/architecture) permite integrar de forma estandarizada modelos de IA y herramientas.

<div id="features">
  ## Funcionalidades
</div>

* Implementaciones de Cliente MCP y Servidor MCP que admiten:
  * Negociación de [compatibilidad de versiones](/es/specification/2025-06-18/basic/lifecycle#initialization) del protocolo
  * Descubrimiento, ejecución y notificaciones de cambios en la lista de [Herramientas](/es/specification/2025-06-18/server/tools/)
  * Gestión de [Recursos](/es/specification/2025-06-18/server/resources/) con plantillas de URI
  * Manejo y gestión de [Indicaciones](/es/specification/2025-06-18/server/prompts/)
  * Sugerencias de autocompletado de argumentos de [Completion](/es/specification/2025-06-18/server/utilities/completion/) para indicaciones y URI de recursos
  * Seguimiento de [progreso](/es/specification/2025-06-18/basic/utilities/progress/) para operaciones de larga duración
  * Mecanismo ligero de verificación de estado [Ping](/es/specification/2025-06-18/basic/utilities/ping/)
    * [Keepalive del servidor](/es/specification/2025-06-18/basic/utilities/ping#implementation-considerations/) para mantener activas las conexiones con el servidor
  * [Registro](/es/specification/2025-06-18/server/utilities/logging/) para enviar mensajes de log estructurados a los clientes
  * Gestión y notificaciones de la lista de [Raíces](/es/specification/2025-06-18/client/roots/)
  * Compatibilidad con [Muestreo](/es/specification/2025-06-18/client/sampling/) para interacciones con modelos de IA
  * [Elicitación](/es/specification/2025-06-18/client/elicitation/) para que los servidores soliciten información adicional a los usuarios a través del cliente
* Múltiples implementaciones de transporte:
  * Transportes predeterminados (incluidos en el módulo principal `mcp`, sin necesidad de frameworks web externos):
    * Transporte basado en STDIO para comunicación entre procesos
    * Transporte de cliente `SSE` y `Streamable-HTTP` basado en Java HttpClient
    * Transporte de servidor `SSE` y `Streamable-HTTP` basado en servlets
  * Transportes opcionales basados en Spring (prácticos si se usa Spring Framework):
    * Transportes de cliente y servidor WebFlux `SSE` y `Streamable-HTTP`
    * Transporte WebMVC `SSE` y `Streamable-HTTP` para streaming HTTP basado en servlets
* Compatible con paradigmas de programación sincrónica y asincrónica

<Tip>
  El módulo principal `io.modelcontextprotocol.sdk:mcp` proporciona implementaciones de transporte de cliente y servidor predeterminadas `STDIO`, `SSE` y `Streamable-HTTP` sin necesidad de frameworks web externos.

  Los transportes específicos de Spring están disponibles como dependencias opcionales para mayor comodidad al usar el framework [Spring AI](https://docs.spring.io/spring-ai/reference/1.1-SNAPSHOT/api/mcp/mcp-overview.html).
</Tip>

<div id="architecture">
  ## Arquitectura
</div>

El SDK sigue una arquitectura en capas con una clara separación de responsabilidades:

{/* ![MCP Stack Architecture](/images/java/mcp-stack.svg) */}

* **Capa Cliente/Servidor (McpClient/McpServer)**: Ambos usan McpSession para operaciones síncronas y asíncronas,
  con McpClient gestionando las operaciones del protocolo en el lado del cliente y McpServer administrando las operaciones del protocolo en el lado del servidor.
* **Capa de sesión (McpSession)**: Gestiona los patrones de comunicación y el estado utilizando la implementación DefaultMcpSession.
* **Capa de transporte (McpTransport)**: Se encarga de la serialización y deserialización de mensajes JSON-RPC mediante:
  * StdioTransport (stdin/stdout) en el módulo core
  * Transportes HTTP `Streamable-HTTP` y `SSE` en módulos de transporte dedicados (Java HttpClient, Spring WebFlux, Spring WebMVC)

El [Cliente MCP](/es/sdk/java/mcp-client) es un componente clave en la arquitectura del Protocolo de Contexto de Modelo (MCP), responsable de establecer y gestionar conexiones con servidores MCP.
Implementa el lado del cliente del protocolo.

![Arquitectura del Cliente MCP en Java](/images/java/java-mcp-client-architecture.jpg)

El [Servidor MCP](/es/sdk/java/mcp-server) es un componente fundamental en la arquitectura del Protocolo de Contexto de Modelo (MCP) que proporciona herramientas, recursos y capacidades a los clientes.
Implementa el lado del servidor del protocolo.

![Arquitectura del Servidor MCP en Java](/images/java/java-mcp-server-architecture.jpg)

Interacciones clave:

* **Inicialización Cliente/Servidor**: Configuración del transporte, verificación de compatibilidad del protocolo, negociación de capacidades e intercambio de detalles de implementación.
* **Flujo de mensajes**: Gestión de mensajes JSON-RPC con validación, procesamiento de respuestas con tipado seguro y manejo de errores.
* **Gestión de recursos**: Descubrimiento de recursos, acceso basado en plantillas de URI, sistema de suscripción y recuperación de contenido.

<div id="dependencies">
  ## Dependencias
</div>

Añade las siguientes dependencias a tu proyecto:

<Tabs>
  <Tab title="Maven">
    Funcionalidad principal de MCP:

    ```xml
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp</artifactId>
    </dependency>
    ```

    El módulo principal `mcp` ya incluye implementaciones predeterminadas de los transportes `STDIO`, `SSE` y `Streamable-HTTP`, y no requiere frameworks web externos.

    Si usas Spring Framework y quieres aprovechar implementaciones de transporte específicas de Spring, añade una de las siguientes dependencias opcionales:

    ```xml
    <!-- Opcional: transportes de cliente y servidor SSE y Streamable-HTTP basados en Spring WebFlux -->
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp-spring-webflux</artifactId>
    </dependency>

    <!-- Opcional: transportes de servidor SSE y Streamable-HTTP basados en Spring WebMVC -->
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp-spring-webmvc</artifactId>
    </dependency>
    ```
  </Tab>

  <Tab title="Gradle">
    Funcionalidad principal de MCP:

    ```groovy
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp")
      //...
    }
    ```

    El módulo principal `mcp` ya incluye implementaciones predeterminadas de los transportes `STDIO`, `SSE` y `Streamable-HTTP`, y no requiere frameworks web externos.

    Si usas Spring Framework y quieres aprovechar implementaciones de transporte específicas de Spring, añade una de las siguientes dependencias opcionales:

    ```groovy
    // Opcional: transportes de cliente y servidor SSE y Streamable-HTTP basados en Spring WebFlux
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-spring-webflux")
    }

    // Opcional: transportes de servidor SSE y Streamable-HTTP basados en Spring WebMVC
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-spring-webmvc")
    }
    ```
  </Tab>
</Tabs>

* `io.modelcontextprotocol.sdk:mcp-spring-webflux` - Cliente y servidor basados en WebFlux, con implementaciones de transporte `Streamable-HTTP` y `SSE`.
  La implementación basada en WebFlux puede usarse en aplicaciones reactivas, mientras que el Cliente MCP basado en WebClient puede emplearse tanto en aplicaciones reactivas como imperativas.
  Es una opción altamente escalable, adecuada y recomendada para escenarios de alto rendimiento.
* `io.modelcontextprotocol.sdk:mcp-spring-webmvc` - Servidor basado en WebMVC, con implementaciones de transporte `Streamable-HTTP` y `SSE` para aplicaciones basadas en servlets.

<div id="bill-of-materials-bom">
  ### Lista de materiales (BOM)
</div>

La lista de materiales (BOM) declara las versiones recomendadas de todas las dependencias usadas por una versión determinada.
Usar la BOM en el script de compilación de tu aplicación evita que tengas que especificar y mantener tú mismo las versiones de las dependencias.
En su lugar, la versión de la BOM que utilices determina las versiones de las dependencias en uso.
También garantiza que, de forma predeterminada, utilices versiones compatibles y probadas de las dependencias, a menos que decidas reemplazarlas.

Agrega la BOM a tu proyecto:

<Tabs>
  <Tab title="Maven">
    ```xml
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>io.modelcontextprotocol.sdk</groupId>
                <artifactId>mcp-bom</artifactId>
                <version>0.12.1</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
    ```
  </Tab>

  <Tab title="Gradle">
    ```groovy
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-bom:0.12.1")
      //...
    }
    ```

    Los usuarios de Gradle también pueden usar la BOM de MCP aprovechando la compatibilidad nativa de Gradle (5.0+) para declarar restricciones de dependencias mediante una BOM de Maven.
    Esto se implementa agregando un método controlador de dependencias &#39;platform&#39; en la sección dependencies de tu script de compilación de Gradle.
    Como se muestra en el fragmento anterior, a continuación puedes declarar, sin versión, los módulos del SDK de MCP que quieras usar.
  </Tab>
</Tabs>

Reemplaza el número de versión por la versión de la BOM que quieras usar.
