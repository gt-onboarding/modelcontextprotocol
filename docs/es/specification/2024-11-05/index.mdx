[Model Context Protocol](https://modelcontextprotocol.io) (MCP) es un protocolo abierto que
permite la integración fluida entre aplicaciones LLM y fuentes de datos externas y
herramientas. Ya sea que estés creando un IDE con IA, mejorando una interfaz de chat o creando
flujos de trabajo de IA personalizados, MCP proporciona una forma estandarizada de conectar los LLM con el contexto
que necesitan.

Esta especificación define los requisitos normativos del protocolo, basados en el
esquema de TypeScript en
[schema.ts](https://github.com/modelcontextprotocol/specification/blob/main/schema/2024-11-05/schema.ts).

Para guías de implementación y ejemplos, visita
[modelcontextprotocol.io](https://modelcontextprotocol.io).

Las palabras clave &quot;MUST&quot;, &quot;MUST NOT&quot;, &quot;REQUIRED&quot;, &quot;SHALL&quot;, &quot;SHALL NOT&quot;, &quot;SHOULD&quot;, &quot;SHOULD
NOT&quot;, &quot;RECOMMENDED&quot;, &quot;NOT RECOMMENDED&quot;, &quot;MAY&quot; y &quot;OPTIONAL&quot; en este documento deben
interpretarse como se describe en [BCP 14](https://datatracker.ietf.org/doc/html/bcp14)
[[RFC2119](https://datatracker.ietf.org/doc/html/rfc2119)]
[[RFC8174](https://datatracker.ietf.org/doc/html/rfc8174)] cuando, y solo cuando, aparezcan
en mayúsculas, como se muestra aquí.

<div id="overview">
  ## Descripción general
</div>

MCP ofrece una forma estandarizada para que las aplicaciones:

* Compartan información contextual con modelos de lenguaje
* Expongan herramientas y capacidades a sistemas de IA
* Construyan integraciones y flujos de trabajo componibles

El protocolo utiliza mensajes [JSON-RPC](https://www.jsonrpc.org/) 2.0 para establecer
la comunicación entre:

* **Anfitriones**: aplicaciones de LLM que inician conexiones
* **Clientes**: conectores dentro de la aplicación anfitriona
* **Servidores**: servicios que proporcionan contexto y capacidades

MCP se inspira en el
[Language Server Protocol](https://microsoft.github.io/language-server-protocol/), que
estandariza cómo añadir compatibilidad con lenguajes de programación en todo un ecosistema de
herramientas de desarrollo. De manera similar, MCP estandariza cómo integrar contexto adicional
y herramientas en el ecosistema de aplicaciones de IA.

<div id="key-details">
  ## Información clave
</div>

<div id="base-protocol">
  ### Protocolo base
</div>

* Formato de mensajes [JSON-RPC](https://www.jsonrpc.org/)
* Conexiones con estado
* Negociación de capacidades entre cliente y servidor

<div id="features">
  ### Funciones
</div>

Los servidores ofrecen cualquiera de las siguientes funciones a los clientes:

* **Recursos**: Contexto y datos para que los utilicen el usuario o el modelo de IA
* **Indicaciones**: Mensajes y flujos de trabajo parametrizados para los usuarios
* **Herramientas**: Operaciones que el modelo de IA puede ejecutar

Los clientes pueden ofrecer la siguiente función a los servidores:

* **Muestreo**: Comportamientos agentivos iniciados por el servidor e interacciones recursivas con LLM

<div id="additional-utilities">
  ### Utilidades adicionales
</div>

* Configuración
* Seguimiento del progreso
* Cancelación
* Informe de errores
* Registro

<div id="security-and-trust-safety">
  ## Seguridad y Confianza
</div>

El Protocolo de Contexto de Modelo habilita potentes capacidades mediante el acceso arbitrario a datos y rutas de ejecución de código. Con este poder surgen importantes consideraciones de seguridad y confianza que todos los implementadores deben abordar cuidadosamente.

<div id="key-principles">
  ### Principios clave
</div>

1. **Consentimiento y control del usuario**
   * Los usuarios deben dar su consentimiento explícito y comprender todo acceso a datos y operaciones
   * Los usuarios deben mantener el control sobre qué datos se comparten y qué acciones se realizan
   * Los implementadores deben proporcionar interfaces claras para revisar y autorizar actividades

2. **Privacidad de los datos**
   * Los Anfitriones deben obtener el consentimiento explícito del usuario antes de exponer datos del usuario a los servidores
   * Los Anfitriones no deben transmitir datos de Recursos a ningún otro lugar sin el consentimiento del usuario
   * Los datos del usuario deben protegerse con controles de acceso adecuados

3. **Seguridad de las Herramientas**
   * Las Herramientas implican la ejecución de código arbitrario y deben tratarse con la debida
     cautela
   * Los Anfitriones deben obtener el consentimiento explícito del usuario antes de invocar cualquier Herramienta
   * Los usuarios deben comprender qué hace cada Herramienta antes de autorizar su uso

4. **Controles de Muestreo de LLM**
   * Los usuarios deben aprobar explícitamente cualquier solicitud de Muestreo de LLM
   * Los usuarios deben controlar:
     * Si se realiza o no el Muestreo
     * La indicación (prompt) exacta que se enviará
     * Qué resultados puede ver el Servidor
   * El protocolo limita intencionalmente la visibilidad del Servidor sobre las Indicaciones

<div id="implementation-guidelines">
  ### Recomendaciones de implementación
</div>

Si bien el MCP en sí no puede hacer cumplir estos principios de seguridad a nivel de protocolo,
quienes implementen **DEBERÍAN**:

1. Incorporar flujos sólidos de consentimiento y autorización en sus aplicaciones
2. Proporcionar documentación clara sobre las implicaciones de seguridad
3. Implementar controles de acceso y protecciones de datos adecuados
4. Seguir las mejores prácticas de seguridad en sus integraciones
5. Considerar las implicaciones de privacidad en el diseño de sus funcionalidades
