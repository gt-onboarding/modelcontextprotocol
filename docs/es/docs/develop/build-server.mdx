<div id="core-mcp-concepts">
  ### Conceptos fundamentales del MCP
</div>

Los Servidores MCP pueden proporcionar tres tipos principales de capacidades:

1. **[Recursos](/es/docs/learn/server-concepts#resources)**: Datos similares a archivos que los clientes pueden leer (como respuestas de API o contenido de archivos)
2. **[Herramientas](/es/docs/learn/server-concepts#tools)**: Funciones que el LLM puede invocar (con aprobación del usuario)
3. **[Indicaciones](/es/docs/learn/server-concepts#prompts)**: Plantillas prediseñadas que ayudan a los usuarios a realizar tareas específicas

Este tutorial se centrará principalmente en las Herramientas.

    ### Registro en Servidores MCP

    Al implementar Servidores MCP, tenga cuidado con cómo gestiona el registro:

    **Para servidores basados en STDIO:** No escribas nunca en la salida estándar (stdout). Esto incluye:

    * Instrucciones `print()` en Python
    * `console.log()` en JavaScript
    * `fmt.Println()` en Go
    * Funciones similares de stdout en otros lenguajes

    Escribir en stdout corromperá los mensajes JSON-RPC y hará que su servidor falle.

    **Para servidores basados en HTTP:** El registro en la salida estándar es adecuado, ya que no interfiere con las respuestas HTTP.

    ### Mejores prácticas

    1. Usa una biblioteca de registro que escriba en stderr o en archivos.

    ### Ejemplos rápidos

    ```python
    # ❌ Malo (STDIO)
    print("Procesando la solicitud")

    # ✅ Bueno (STDIO)
    import logging
    logging.info("Procesando la solicitud")
    ```

    ## Crear tu servidor

    ### Importar paquetes y configurar la instancia

    Añade lo siguiente al inicio de tu `weather.py`:

    ```python
    from typing import Any
    import httpx
    from mcp.server.fastmcp import FastMCP

    # Inicializar el servidor FastMCP
    mcp = FastMCP("weather")

    # Constantes
    NWS_API_BASE = "https://api.weather.gov"
    USER_AGENT = "weather-app/1.0"
    ```

    La clase FastMCP utiliza anotaciones de tipo de Python y docstrings para generar automáticamente definiciones de Herramientas, lo que facilita la creación y el mantenimiento de Herramientas MCP.

    ### Funciones de ayuda

    A continuación, añadamos nuestras funciones auxiliares para consultar y formatear los datos de la API del Servicio Meteorológico Nacional:

    ```python
    async def make_nws_request(url: str) -> dict[str, Any] | None:
        """Realiza una solicitud a la API de NWS con el manejo de errores adecuado."""
        headers = {
            "User-Agent": USER_AGENT,
            "Accept": "application/geo+json"
        }
        async with httpx.AsyncClient() as client:
            try:
                response = await client.get(url, headers=headers, timeout=30.0)
                response.raise_for_status()
                return response.json()
            except Exception:
                return None

    def format_alert(feature: dict) -> str:
        """Format an alert feature into a readable string."""
        props = feature["properties"]
        return f"""
    Event: {props.get('event', 'Desconocido')}
    Area: {props.get('areaDesc', 'Desconocido')}
    Severity: {props.get('severity', 'Desconocido')}
    Description: {props.get('description', 'No hay ninguna descripción disponible')}
    Instructions: {props.get('instruction', 'No se proporcionan instrucciones específicas')}
    """
    ```

    ### Implementación de la ejecución de Herramientas

    El controlador de ejecución de la herramienta es el responsable de ejecutar la lógica de cada herramienta. Añadámoslo:

    ```python
    @mcp.tool()
    async def get_alerts(state: str) -> str:
        """Obtener alertas meteorológicas para un estado de EE. UU.

        Args:
            state: Código de estado de EE. UU. de dos letras (p. ej., CA, NY)
        """
        url = f"{NWS_API_BASE}/alerts/active/area/{state}"
        data = await make_nws_request(url)

        if not data or "features" not in data:
            return "No se pudieron obtener las alertas o no se encontró ninguna."

        if not data["features"]:
            return "No hay alertas activas para este estado."

        alerts = [format_alert(feature) for feature in data["features"]]
        return "\n---\n".join(alerts)

    @mcp.tool()
    async def get_forecast(latitude: float, longitude: float) -> str:
        """Obtener el pronóstico del tiempo para una ubicación.

        Args:
            latitude: Latitud de la ubicación
            longitude: Longitud de la ubicación
        """
        # First get the forecast grid endpoint
        points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
        points_data = await make_nws_request(points_url)

        if not points_data:
            return "No se pudieron obtener datos de pronóstico para esta ubicación."

        # Get the forecast URL from the points response
        forecast_url = points_data["properties"]["forecast"]
        forecast_data = await make_nws_request(forecast_url)

        if not forecast_data:
            return "No se pudo obtener el pronóstico detallado."

        # Formatear los periodos en un pronóstico legible
        periods = forecast_data["properties"]["periods"]
        forecasts = []
        for period in periods[:5]:  # Mostrar solo los próximos 5 periodos
            forecast = f"""
    {period['name']}:
    Temperature: {period['temperature']}°{period['temperatureUnit']}
    Viento: {period['windSpeed']} {period['windDirection']}
    Pronóstico: {period['detailedForecast']}
    """
            forecasts.append(forecast)

        return "\n---\n".join(forecasts)
    ```

    Por último, inicialicemos y ejecutemos el servidor:

    ```python
        # Inicializar y ejecutar el servidor
        mcp.run(transport='stdio')
    ```

    ## Crear tu servidor

    ### Importación de paquetes y configuración de la instancia

    Agrega lo siguiente al inicio de tu `src/index.ts`:

    ```typescript
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
    import { z } from "zod";

    const NWS_API_BASE = "https://api.weather.gov";
    const USER_AGENT = "app-del-clima/1.0";

    // Crear una instancia del servidor
    const server = new McpServer({
      name: "weather",
      version: "1.0.0",
      capabilities: {
        resources: {},
        tools: {},
      },
    });
    ```

    ### Funciones de ayuda

    A continuación, añadamos nuestras funciones auxiliares para consultar y formatear los datos de la API del Servicio Meteorológico Nacional:

    ```typescript
    // Función auxiliar para realizar solicitudes a la API de NWS
    async function makeNWSRequest<T>(url: string): Promise<T | null> {
      const headers = {
        "User-Agent": USER_AGENT,
        Accept: "application/geo+json",
      };

      try {
        const response = await fetch(url, { headers });
        if (!response.ok) {
          throw new Error(`¡Error HTTP! Estado: ${response.status}`);
        }
        return (await response.json()) as T;
      } catch (error) {
        console.error("Error al realizar la solicitud a NWS:", error);
        return null;
      }
    }

    interface AlertFeature {
      properties: {
        event?: string;
        areaDesc?: string;
        severity?: string;
        status?: string;
        headline?: string;
      };
    }

    // Formatear datos de alerta
    function formatAlert(feature: AlertFeature): string {
      const props = feature.properties;
      return [
        `Evento: ${props.event || "Desconocido"}`,
        `Área: ${props.areaDesc || "Desconocida"}`,
        `Gravedad: ${props.severity || "Desconocida"}`,
        `Estado: ${props.status || "Desconocido"}`,
        `Encabezado: ${props.headline || "Sin encabezado"}`,
        "---",
      ].join("\n");
    }

    interface ForecastPeriod {
      name?: string;
      temperature?: number;
      temperatureUnit?: string;
      windSpeed?: string;
      windDirection?: string;
      shortForecast?: string;
    }

    interface AlertsResponse {
      features: AlertFeature[];
    }

    interface PointsResponse {
      properties: {
        forecast?: string;
      };
    }

    interface ForecastResponse {
      properties: {
        periods: ForecastPeriod[];
      };
    }
    ```

    ### Implementación de la ejecución de Herramientas

    El manejador de ejecución de la herramienta es el responsable de ejecutar la lógica de cada herramienta. Agreguémoslo:

    ```typescript
    // Registrar Herramientas meteorológicas
    server.tool(
      "get_alerts",
      "Obtener alertas meteorológicas para un estado",
      {
        state: z.string().length(2).describe("Código de estado de dos letras (p. ej., CA, NY)"),
      },
      async ({ state }) => {
        const stateCode = state.toUpperCase();
        const alertsUrl = `${NWS_API_BASE}/alerts?area=${stateCode}`;
        const alertsData = await makeNWSRequest<AlertsResponse>(alertsUrl);

        if (!alertsData) {
          return {
            content: [
              {
                type: "text",
                text: "No se pudieron recuperar los datos de alertas",
              },
            ],
          };
        }

        const features = alertsData.features || [];
        if (features.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: `No hay alertas activas para ${stateCode}`,
              },
            ],
          };
        }

        const formattedAlerts = features.map(formatAlert);
        const alertsText = `Alertas activas para ${stateCode}:\n\n${formattedAlerts.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: alertsText,
            },
          ],
        };
      },
    );

    server.tool(
      "get_forecast",
      "Obtener el pronóstico del tiempo para una ubicación",
      {
        latitude: z.number().min(-90).max(90).describe("Latitud de la ubicación"),
        longitude: z
          .number()
          .min(-180)
          .max(180)
          .describe("Longitud de la ubicación"),
      },
      async ({ latitude, longitude }) => {
        // Obtener datos del punto de cuadrícula
        const pointsUrl = `${NWS_API_BASE}/points/${latitude.toFixed(4)},${longitude.toFixed(4)}`;
        const pointsData = await makeNWSRequest<PointsResponse>(pointsUrl);

        if (!pointsData) {
          return {
            content: [
              {
                type: "text",
                text: `No se pudieron recuperar los datos del punto de cuadrícula para las coordenadas: ${latitude}, ${longitude}. Es posible que esta ubicación no sea compatible con la API de NWS (solo se admiten ubicaciones en EE. UU.).`,
              },
            ],
          };
        }

        const forecastUrl = pointsData.properties?.forecast;
        if (!forecastUrl) {
          return {
            content: [
              {
                type: "text",
                text: "No se pudo obtener la URL del pronóstico a partir de los datos del punto de cuadrícula",
              },
            ],
          };
        }

        // Obtener datos del pronóstico
        const forecastData = await makeNWSRequest<ForecastResponse>(forecastUrl);
        if (!forecastData) {
          return {
            content: [
              {
                type: "text",
                text: "No se pudieron recuperar los datos del pronóstico",
              },
            ],
          };
        }

        const periods = forecastData.properties?.periods || [];
        if (periods.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: "No hay periodos de pronóstico disponibles",
              },
            ],
          };
        }

        // Format forecast periods
        const formattedForecast = periods.map((period: ForecastPeriod) =>
          [
            `${period.name || "Desconocido"}:`,
            `Temperatura: ${period.temperature || "Desconocido"}°${period.temperatureUnit || "F"}`,
            `Viento: ${period.windSpeed || "Desconocido"} ${period.windDirection || ""}`,
            `${period.shortForecast || "No hay pronóstico disponible"}`,
            "---",
          ].join("\n"),
        );

        const forecastText = `Pronóstico para ${latitude}, ${longitude}:\n\n${formattedForecast.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: forecastText,
            },
          ],
        };
      },
    );
    ```

    ```typescript
      console.error("Error fatal en main():", error);

    });
    ```

    ### Registro en servidores MCP

    Al implementar Servidores MCP, tenga cuidado con cómo gestiona el registro:

    **Para servidores basados en STDIO:** No escribas nunca en la salida estándar (stdout). Esto incluye:

    * instrucciones `print()` en Python
    * `console.log()` en JavaScript
    * `fmt.Println()` en Go
    * Funciones stdout similares en otros idiomas

    Escribir en stdout corromperá los mensajes JSON-RPC y hará que su servidor falle.

    **Para servidores basados en HTTP:** El registro en la salida estándar está bien, ya que no interfiere con las respuestas HTTP.

    ### Buenas prácticas

    1. Usa una biblioteca de registro que escriba en stderr o en archivos.
    2. Asegúrate de que cualquier biblioteca de registro configurada no escriba en STDOUT

    ```kotlin
    // Función de extensión para obtener el pronóstico para una latitud y longitud determinadas
    suspend fun HttpClient.getForecast(latitude: Double, longitude: Double): List<String> {
        val points = this.get("/points/$latitude,$longitude").body<Points>()
        val forecast = this.get(points.properties.forecast).body<Forecast>()
        return forecast.properties.periods.map { period ->
            """
                ${period.name}:
                Temperatura: ${period.temperature} ${period.temperatureUnit}
                Viento: ${period.windSpeed} ${period.windDirection}
                Pronóstico: ${period.detailedForecast}
            """.trimIndent()
        }
    }

    // Función de extensión para obtener alertas meteorológicas para un estado determinado
    suspend fun HttpClient.getAlerts(state: String): List<String> {
        val alerts = this.get("/alerts/active/area/$state").body<Alert>()
        return alerts.features.map { feature ->
            """
                Evento: ${feature.properties.event}
                Área: ${feature.properties.areaDesc}
                Gravedad: ${feature.properties.severity}
                Descripción: ${feature.properties.description}
                Instrucciones: ${feature.properties.instruction}
            """.trimIndent()
        }
    }

    @Serializable
    data class Points(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val forecast: String)
    }

    @Serializable
    data class Forecast(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val periods: List<Period>)

        @Serializable
        data class Period(
            val number: Int, val name: String, val startTime: String, val endTime: String,
            val isDaytime: Boolean, val temperature: Int, val temperatureUnit: String,
            val temperatureTrend: String, val probabilityOfPrecipitation: JsonObject,
            val windSpeed: String, val windDirection: String,
            val shortForecast: String, val detailedForecast: String,
        )
    }

    @Serializable
    data class Alert(
        val features: List<Feature>
    ) {
        @Serializable
        data class Feature(
            val properties: Properties
        )

        @Serializable
        data class Properties(
            val event: String, val areaDesc: String, val severity: String,
            val description: String, val instruction: String?,
        )
    }
    ```

    ### Implementación de la ejecución de Herramientas

    El controlador de ejecución de la herramienta es el responsable de ejecutar la lógica de cada herramienta. Agreguémoslo:

    ```kotlin
    // Crear un cliente HTTP con una configuración de solicitud predeterminada y negociación de contenido JSON
    val httpClient = HttpClient {
        defaultRequest {
            url("https://api.weather.gov")
            headers {
                append("Accept", "application/geo+json")
                append("User-Agent", "WeatherApiClient/1.0")
            }
            contentType(ContentType.Application.Json)
        }
    // Instalar el complemento de negociación de contenido para la serialización/deserialización de JSON
        install(ContentNegotiation) { json(Json { ignoreUnknownKeys = true }) }
    }

    // Registrar una herramienta para obtener alertas meteorológicas por estado
    server.addTool(
        name = "get_alerts",
        description = """
            Obtener alertas meteorológicas para un estado de EE. UU. La entrada es el código de estado de dos letras (p. ej., CA, NY)
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("state") {
                    put("type", "string")
                    put("description", "Código de estado de EE. UU. de dos letras (p. ej., CA, NY)")
                }
            },
            required = listOf("state")
        )
    ) { request ->
        val state = request.arguments["state"]?.jsonPrimitive?.content
        if (state == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("El parámetro 'state' es obligatorio."))
            )
        }

        val alerts = httpClient.getAlerts(state)

        CallToolResult(content = alerts.map { TextContent(it) })
    }

    // Registrar una herramienta para obtener el pronóstico del tiempo por latitud y longitud
    server.addTool(
        name = "get_forecast",
        description = """
            Obtener el pronóstico del tiempo para una latitud/longitud específica
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("latitude") { put("type", "number") }
                putJsonObject("longitude") { put("type", "number") }
            },
            required = listOf("latitude", "longitude")
        )
    ) { request ->
        val latitude = request.arguments["latitude"]?.jsonPrimitive?.doubleOrNull
        val longitude = request.arguments["longitude"]?.jsonPrimitive?.doubleOrNull
        if (latitude == null || longitude == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("Los parámetros 'latitude' y 'longitude' son obligatorios."))
            )
        }

        val forecast = httpClient.getForecast(latitude, longitude)

        CallToolResult(content = forecast.map { TextContent(it) })
    }
    ```

    Después de ejecutar `dotnet new console`, se creará un nuevo proyecto de C#.
    Puedes abrir el proyecto en tu IDE favorito, como [Visual Studio](https://visualstudio.microsoft.com/) o [Rider](https://www.jetbrains.com/rider/).
    Como alternativa, puedes crear una aplicación de C# con el [asistente de proyectos de Visual Studio](https://learn.microsoft.com/en-us/visualstudio/get-started/csharp/tutorial-console?view=vs-2022).
    Después de crear el proyecto, agrega el paquete NuGet para el SDK del Protocolo de Contexto de Modelo (MCP) y el host:

    ```bash
    # Agrega el paquete NuGet del Protocolo de Contexto de Modelo (MCP)
    dotnet add package ModelContextProtocol --prerelease
    # Agrega el paquete NuGet de .NET Hosting
    dotnet add package Microsoft.Extensions.Hosting
    ```

    Ahora pasemos a construir tu servidor.

    ## Crear tu servidor

    Abre el archivo `Program.cs` de tu proyecto y reemplaza su contenido por el siguiente código:

    ```csharp
    using Microsoft.Extensions.DependencyInjection;
    using Microsoft.Extensions.Hosting;
    using ModelContextProtocol;
    using System.Net.Http.Headers;

    var builder = Host.CreateEmptyApplicationBuilder(settings: null);

    builder.Services.AddMcpServer()
        .WithStdioServerTransport()
        .WithToolsFromAssembly();

    builder.Services.AddSingleton(_ =>
    {
        var client = new HttpClient() { BaseAddress = new Uri("https://api.weather.gov") };
        client.DefaultRequestHeaders.UserAgent.Add(new ProductInfoHeaderValue("weather-tool", "1.0"));
        return client;
    });

    var app = builder.Build();

    await app.RunAsync();
    ```

    <Note>
      Al crear el `ApplicationHostBuilder`, asegúrate de usar `CreateEmptyApplicationBuilder` en lugar de `CreateDefaultBuilder`. Así te aseguras de que el servidor no escriba mensajes adicionales en la consola. Esto solo es necesario para servidores que usan el transporte STDIO.
    </Note>

    Este código configura una aplicación de consola básica que utiliza el SDK del Protocolo de Contexto de Modelo (MCP) para crear un Servidor MCP con transporte de E/S estándar.

    ### Funciones auxiliares de la API de clima

    Crea una clase de extensión para `HttpClient` que simplifique el manejo de solicitudes JSON:

    ```csharp
    using System.Text.Json;

    internal static class HttpClientExt
    {
        public static async Task<JsonDocument> ReadJsonDocumentAsync(this HttpClient client, string requestUri)
        {
            using var response = await client.GetAsync(requestUri);
            response.EnsureSuccessStatusCode();
            return await JsonDocument.ParseAsync(await response.Content.ReadAsStreamAsync());
        }
    }
    ```

    A continuación, define una clase con los manejadores de ejecución de la herramienta para consultar la API del Servicio Meteorológico Nacional y convertir sus respuestas:

    ```csharp
    using ModelContextProtocol.Server;
    using System.ComponentModel;
    using System.Globalization;
    using System.Text.Json;

    namespace QuickstartWeatherServer.Tools;

    [McpServerToolType]
    public static class WeatherTools
    {
        [McpServerTool, Description("Obtener alertas meteorológicas para un estado de EE. UU.")]
        public static async Task<string> GetAlerts(
            HttpClient client,
            [Description("El estado de EE. UU. del que obtener alertas.")]
        {
            using var jsonDocument = await client.ReadJsonDocumentAsync($"/alerts/active/area/{state}");
            var jsonElement = jsonDocument.RootElement;
            var alerts = jsonElement.GetProperty("features").EnumerateArray();

            if (!alerts.Any())
            {
                return "No hay alertas activas para este estado.";
            }

            return string.Join("\n--\n", alerts.Select(alert =>
            {
                JsonElement properties = alert.GetProperty("properties");
                return $"""
                        Evento: {properties.GetProperty("event").GetString()}
                        Área: {properties.GetProperty("areaDesc").GetString()}
                        Severidad: {properties.GetProperty("severity").GetString()}
                        Descripción: {properties.GetProperty("description").GetString()}
                        Instrucciones: {properties.GetProperty("instruction").GetString()}
                        """;
            }));
        }

        [McpServerTool, Description("Obtener el pronóstico del tiempo para una ubicación.")]
        public static async Task<string> GetForecast(
            HttpClient client,
            [Description("Latitud de la ubicación.")] double latitude,
            [Description("Longitud de la ubicación.")] double longitude)
        {
            var pointUrl = string.Create(CultureInfo.InvariantCulture, $"/points/{latitude},{longitude}");
            using var jsonDocument = await client.ReadJsonDocumentAsync(pointUrl);
            var forecastUrl = jsonDocument.RootElement.GetProperty("properties").GetProperty("forecast").GetString()
                ?? throw new Exception($"No se proporcionó la URL del pronóstico por {client.BaseAddress}points/{latitude},{longitude}")

            using var forecastDocument = await client.ReadJsonDocumentAsync(forecastUrl);
            var periods = forecastDocument.RootElement.GetProperty("properties").GetProperty("periods").EnumerateArray();

            return string.Join("\n---\n", periods.Select(period => $"""
                    {period.GetProperty("name").GetString()}
                    Temperatura: {period.GetProperty("temperature").GetInt32()}°F
                    Viento: {period.GetProperty("windSpeed").GetString()} {period.GetProperty("windDirection").GetString()}
                    Pronóstico: {period.GetProperty("detailedForecast").GetString()}
                    """));
        }
    }
    ```

<div id="whats-happening-under-the-hood">
  ## Qué ocurre tras bambalinas
</div>

Cuando haces una pregunta:

1. El cliente envía tu pregunta a Claude
2. Claude analiza las Herramientas disponibles y decide cuáles usar
3. El cliente ejecuta las Herramientas elegidas a través del Servidor MCP
4. Los resultados se envían de vuelta a Claude
5. Claude formula una respuesta en lenguaje natural
6. ¡Se te muestra la respuesta!

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Problemas de integración con Claude para Desktop">
    **Obtener registros de Claude para Desktop**

    El registro de Claude.app relacionado con el Protocolo de Contexto de Modelo (MCP) se escribe en archivos en `~/Library/Logs/Claude`:

    * `mcp.log` contendrá registros generales sobre conexiones MCP y fallas de conexión.
    * Los archivos llamados `mcp-server-SERVERNAME.log` contendrán el registro de errores (stderr) del servidor indicado.

    Puedes ejecutar el siguiente comando para listar los registros recientes y seguir cualquier registro nuevo:

    ```bash
    # Revisa los registros de Claude en busca de errores
    tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
    ```

    **El servidor no aparece en Claude**

    1. Revisa la sintaxis de tu archivo `claude_desktop_config.json`
    2. Asegúrate de que la ruta a tu proyecto sea absoluta y no relativa
    3. Reinicia por completo Claude para Desktop

    **Las llamadas a Herramientas fallan silenciosamente**

    Si Claude intenta usar las Herramientas pero fallan:

    1. Revisa los registros de Claude en busca de errores
    2. Verifica que tu servidor compile y se ejecute sin errores
    3. Intenta reiniciar Claude para Desktop

    **Nada de esto funciona. ¿Qué hago?**

    Consulta nuestra [guía de depuración](/es/legacy/tools/debugging) para obtener mejores herramientas de depuración y orientación más detallada.
  </Accordion>

  <Accordion title="Problemas con la API del tiempo">
    **Error: Failed to retrieve grid point data**

    Esto suele significar que:

    1. Las coordenadas están fuera de EE. UU.
    2. La API de NWS está teniendo problemas
    3. Estás sujeto a limitación de tasa

    Solución:

    * Verifica que estés usando coordenadas dentro de EE. UU.
    * Agrega un pequeño retraso entre solicitudes
    * Revisa la página de estado de la API de NWS

    **Error: No active alerts for [STATE]**

    Esto no es un error: solo significa que no hay alertas meteorológicas vigentes para ese estado. Prueba con otro estado o verifica durante un episodio de clima severo.
  </Accordion>
</AccordionGroup>
