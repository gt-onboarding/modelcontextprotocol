    ## Creación del cliente

    ### Estructura básica del cliente

    Primero, configuremos las importaciones y creemos la clase básica de cliente:

    ```python
    import asyncio
    from typing import Optional
    from contextlib import AsyncExitStack

    from mcp import ClientSession, StdioServerParameters
    from mcp.client.stdio import stdio_client

    from anthropic import Anthropic
    from dotenv import load_dotenv

    load_dotenv()  # cargar las variables de entorno desde .env

    class MCPClient:
        def __init__(self):
            # Inicializar los objetos de sesión y cliente
            self.session: Optional[ClientSession] = None
            self.exit_stack = AsyncExitStack()
            self.anthropic = Anthropic()
        # aquí irán los métodos
    ```

    ### Gestión de conexiones del servidor

    A continuación, implementaremos el método para conectarnos a un Servidor MCP:

    ```python
    async def connect_to_server(self, server_script_path: str):
        """Conectarse a un Servidor MCP

        Args:
            server_script_path: Ruta al script del servidor (.py o .js)
        """
        is_python = server_script_path.endswith('.py')
        is_js = server_script_path.endswith('.js')
        if not (is_python or is_js):
            raise ValueError("El script del servidor debe ser un archivo .py o .js")

        command = "python" if is_python else "node"
        server_params = StdioServerParameters(
            command=command,
            args=[server_script_path],
            env=None
        )

        stdio_transport = await self.exit_stack.enter_async_context(stdio_client(server_params))
        self.stdio, self.write = stdio_transport
        self.session = await self.exit_stack.enter_async_context(ClientSession(self.stdio, self.write))

        await self.session.initialize()

        # Listar las Herramientas disponibles
        response = await self.session.list_tools()
        tools = response.tools
        print("\nConectado al servidor con Herramientas:", [tool.name for tool in tools])
    ```

    ### Lógica de procesamiento de consultas

    Ahora agreguemos la funcionalidad principal para procesar consultas y manejar llamadas a Herramientas:

    ```python
    async def process_query(self, query: str) -> str:
        """Procesa una consulta con Claude y las herramientas disponibles"""
        messages = [
            {
                "role": "user",
                "content": query
            }
        ]

        response = await self.session.list_tools()
        available_tools = [{
            "name": tool.name,
            "description": tool.description,
            "input_schema": tool.inputSchema
        } for tool in response.tools]

        # Primera llamada a la API de Claude
        response = self.anthropic.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=1000,
            messages=messages,
            tools=available_tools
        )

        # Procesar la respuesta y gestionar las llamadas a herramientas
        final_text = []

        assistant_message_content = []
        for content in response.content:
            if content.type == 'text':
                final_text.append(content.text)
                assistant_message_content.append(content)
            elif content.type == 'tool_use':
                tool_name = content.name
                tool_args = content.input

                # Ejecutar la llamada a la herramienta
                result = await self.session.call_tool(tool_name, tool_args)
                final_text.append(f"[Llamando a la herramienta {tool_name} con argumentos {tool_args}]")

                assistant_message_content.append(content)
                messages.append({
                    "role": "assistant",
                    "content": assistant_message_content
                })
                messages.append({
                    "role": "user",
                    "content": [
                        {
                            "type": "tool_result",
                            "tool_use_id": content.id,
                            "content": result.content
                        }
                    ]
                })

                # Obtener la siguiente respuesta de Claude
                response = self.anthropic.messages.create(
                    model="claude-3-5-sonnet-20241022",
                    max_tokens=1000,
                    messages=messages,
                    tools=available_tools
                )

                final_text.append(response.content[0].text)

        return "\n".join(final_text)
    ```

    ### Interfaz de chat interactivo

    Ahora añadiremos el bucle de chat y la funcionalidad de limpieza:

    ```python
    async def chat_loop(self):
        """Ejecuta un bucle de chat interactivo"""
        print("\n¡Cliente MCP iniciado!")
        print("Escribe tus consultas o 'quit' para salir.")

        while True:
            try:
                query = input("\nConsulta: ").strip()

                if query.lower() == 'quit':
                    break

                response = await self.process_query(query)
                print("\n" + response)

            except Exception as e:
                print(f"\nError: {str(e)}")

    async def cleanup(self):
        """Liberar recursos"""
        await self.exit_stack.aclose()
    ```

    ### Punto de entrada principal

    Por último, añadiremos la lógica principal de ejecución:

    ```python
    async def main():
        if len(sys.argv) < 2:
            print("Uso: python client.py <ruta_al_script_del_servidor>")
            sys.exit(1)

        client = MCPClient()
        try:
            await client.connect_to_server(sys.argv[1])
            await client.chat_loop()
        finally:
            await client.cleanup()

    if __name__ == "__main__":
        import sys
        asyncio.run(main())
    ```

    ## Componentes clave: explicación

    ### 1. Inicialización del cliente

    * La clase `MCPClient` se inicializa con gestión de sesiones y clientes de API
    * Usa `AsyncExitStack` para gestionar correctamente los recursos
    * Configura el cliente de Anthropic para interactuar con Claude

    ### 2. Conexión al servidor

    * Compatible con servidores de Python y de Node.js
    * Valida el tipo de guion del servidor
    * Establece canales de comunicación adecuados
    * Inicializa la sesión y lista las Herramientas disponibles

    ### 3. Procesamiento de consultas

    * Mantiene el contexto de la conversación
    * Gestiona las respuestas de Claude y las llamadas a herramientas
    * Gestiona el flujo de mensajes entre Claude y las Herramientas
    * Combina los resultados en una respuesta coherente

    ### 4. Interfaz interactiva

    * Ofrece una interfaz de línea de comandos sencilla
    * Gestiona la entrada del usuario y muestra las respuestas
    * Incluye gestión básica de errores
    * Permite una salida ordenada

    ### 5. Gestión de Recursos

    * Limpieza adecuada de Recursos
    * Manejo de errores ante problemas de conexión
    * Procedimientos de apagado controlado

    ## Puntos comunes de personalización

    1. **Gestión de Herramientas**
       * Modificar `process_query()` para manejar tipos específicos de Herramientas
       * Agregar manejo de errores personalizado para llamadas de Herramientas
       * Implementar formato de respuestas específico de Herramientas

    2. **Procesamiento de respuestas**
       * Personaliza el formato de los resultados de las herramientas

       * Añadir una interfaz gráfica o web

       * Añadir historial de comandos o autocompletado

    ## Cómo funciona

    Cuando envías una consulta:

    1. El cliente obtiene del servidor la lista de Herramientas disponibles
    2. Tu consulta se envía a Claude junto con las descripciones de las Herramientas
    3. Claude decide qué Herramientas (si las hay) usar
    4. El cliente ejecuta cualquier llamada a Herramientas solicitada a través del servidor
    5. Los resultados se envían de vuelta a Claude
    6. Claude proporciona una respuesta en lenguaje natural
    7. La respuesta se muestra para ti

    ## Solución de problemas

    ### Problemas con la ruta del servidor

    * Verifica que la ruta a tu script del servidor sea correcta
    * Use la ruta absoluta si la ruta relativa no funciona
    * Para los usuarios de Windows, asegúrate de usar barras diagonales (/) o barras invertidas escapadas (\) en la ruta
    * Verifica que el archivo del servidor tenga la extensión correcta (.py para Python o .js para Node.js)

    Ejemplo de uso correcto de la ruta de acceso:

    ```bash
    # Ruta relativa
    uv run client.py ./server/weather.py

    # Ruta absoluta
    uv run client.py /Users/username/projects/mcp-server/weather.py

    # Ruta en Windows (cualquiera de los formatos funciona)
    uv run client.py C:/projects/mcp-server/weather.py
    uv run client.py C:\\projects\\mcp-server\\weather.py
    ```

    ### Temporización de la respuesta

    * La primera respuesta puede tardar hasta 30 segundos en llegar
    * Esto es normal y sucede mientras:
      * El servidor se inicia
      * Claude procesa la consulta
      * Se están ejecutando las herramientas
    * Las respuestas siguientes suelen ser más rápidas
    * No interrumpas el proceso durante este período de espera inicial

    ### Mensajes de errores comunes

    Si ves:

    * `FileNotFoundError`: Comprueba la ruta del servidor
    * `Connection refused`: Asegúrate de que el servidor esté en ejecución y de que la ruta sea correcta
    * `Tool execution failed`: Verifica que las variables de entorno requeridas de la herramienta estén configuradas
    * `Error de tiempo de espera`: Considera aumentar el tiempo de espera en la configuración de tu cliente

    ## Creación del cliente

    ### Estructura básica del cliente

    Primero, configuremos las importaciones y creemos la clase básica del cliente en `index.ts`:

    ```typescript
    import { Anthropic } from "@anthropic-ai/sdk";
    import {
      MessageParam,
      Tool,
    } from "@anthropic-ai/sdk/resources/messages/messages.mjs";
    import { Client } from "@modelcontextprotocol/sdk/client/index.js";
    import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";
    import readline from "readline/promises";
    import dotenv from "dotenv";

    dotenv.config();

    const ANTHROPIC_API_KEY = process.env.ANTHROPIC_API_KEY;
    if (!ANTHROPIC_API_KEY) {
      throw new Error("ANTHROPIC_API_KEY no está configurada");
    }

    class MCPClient {
      private mcp: Client;
      private anthropic: Anthropic;
      private transport: StdioClientTransport | null = null;
      private tools: Tool[] = [];

      constructor() {
        this.anthropic = new Anthropic({
          apiKey: ANTHROPIC_API_KEY,
        });
        this.mcp = new Client({ name: "mcp-client-cli", version: "1.0.0" });
      }
      // methods will go here
    }
    ```

    ### Gestión de la conexión del servidor

    A continuación, implementaremos el método para conectarnos a un Servidor MCP:

    ```typescript
    async connectToServer(serverScriptPath: string) {
      try {
        const isJs = serverScriptPath.endsWith(".js");
        const isPy = serverScriptPath.endsWith(".py");
        if (!isJs && !isPy) {
          throw new Error("El script del servidor debe ser un archivo .js o .py");
        }
        const command = isPy
          ? process.platform === "win32"
            ? "python"
            : "python3"
          : process.execPath;

        this.transport = new StdioClientTransport({
          command,
          args: [serverScriptPath],
        });
        await this.mcp.connect(this.transport);

        const toolsResult = await this.mcp.listTools();
        this.tools = toolsResult.tools.map((tool) => {
          return {
            name: tool.name,
            description: tool.description,
            input_schema: tool.inputSchema,
          };
        });
        console.log(
          "Conectado al servidor con herramientas:",
          this.tools.map(({ name }) => name)
        );
      } catch (e) {
        console.log("No se pudo conectar al Servidor MCP: ", e);
        throw e;
      }
    }
    ```

    ### Lógica de procesamiento de consultas

    Ahora agreguemos la funcionalidad principal para procesar consultas y manejar llamadas de Herramientas:

    ```typescript
    async processQuery(query: string) {
      const messages: MessageParam[] = [
        {
          role: "user",
          content: query,
        },
      ];

      const response = await this.anthropic.messages.create({
        model: "claude-3-5-sonnet-20241022",
        max_tokens: 1000,
        messages,
        tools: this.tools,
      });

      const finalText = [];

      for (const content of response.content) {
        if (content.type === "text") {
          finalText.push(content.text);
        } else if (content.type === "tool_use") {
          const toolName = content.name;
          const toolArgs = content.input as { [x: string]: unknown } | undefined;

          const result = await this.mcp.callTool({
            name: toolName,
            arguments: toolArgs,
          });
          finalText.push(
            `[Ejecutando la herramienta ${toolName} con argumentos ${JSON.stringify(toolArgs)}]`
          );

          messages.push({
            role: "user",
            content: result.content as string,
          });

          const response = await this.anthropic.messages.create({
            model: "claude-3-5-sonnet-20241022",
            max_tokens: 1000,
            messages,
          });

          finalText.push(
            response.content[0].type === "text" ? response.content[0].text : ""
          );
        }
      }

      return finalText.join("\n");
    }
    ```

    ### Interfaz de chat interactivo

    Ahora agregaremos el bucle de chat y la función de limpieza:

    ```typescript
    async chatLoop() {
      const rl = readline.createInterface({
        input: process.stdin,
        output: process.stdout,
      });

      try {
        console.log("\n¡Se inició el Cliente MCP!");
        console.log("Escribe tu consulta o 'quit' para salir.");

        while (true) {
          const message = await rl.question("\nConsulta: ");
          if (message.toLowerCase() === "quit") {
            break;
          }
          const response = await this.processQuery(message);
          console.log("\n" + response);
        }
      } finally {
        rl.close();
      }
    }

    async cleanup() {
      await this.mcp.close();
    }
    ```

    ### Gestión de la conexión del servidor

    A continuación, implementaremos el método para conectarnos a un Servidor MCP:

    ```kotlin
    suspend fun connectToServer(serverScriptPath: String) {
        try {
            val command = buildList {
                when (serverScriptPath.substringAfterLast(".")) {
                    "js" -> add("node")
                    "py" -> add(if (System.getProperty("os.name").lowercase().contains("win")) "python" else "python3")
                    "jar" -> addAll(listOf("java", "-jar"))
                    else -> throw IllegalArgumentException("El script del servidor debe ser un archivo .js, .py o .jar")
                }
                add(serverScriptPath)
            }

            val process = ProcessBuilder(command).start()
            val transport = StdioClientTransport(
                input = process.inputStream.asSource().buffered(),
                output = process.outputStream.asSink().buffered()
            )

            mcp.connect(transport)

            val toolsResult = mcp.listTools()
            tools = toolsResult?.tools?.map { tool ->
                ToolUnion.ofTool(
                    Tool.builder()
                        .name(tool.name)
                        .description(tool.description ?: "")
                        .inputSchema(
                            Tool.InputSchema.builder()
                                .type(JsonValue.from(tool.inputSchema.type))
                                .properties(tool.inputSchema.properties.toJsonValue())
                                .putAdditionalProperty("required", JsonValue.from(tool.inputSchema.required))
                                .build()
                        )
                        .build()
                )
            } ?: emptyList()
            println("Conectado al servidor con herramientas: ${tools.joinToString(", ") { it.tool().get().name() }}")
        } catch (e: Exception) {
            println("Error al conectar con el Servidor MCP: $e")
            throw e
        }
    }
    ```

    Crea también una función auxiliar para convertir de `JsonObject` a `JsonValue` para Anthropic:

    ```kotlin
    private fun JsonObject.toJsonValue(): JsonValue {
        val mapper = ObjectMapper()
        val node = mapper.readTree(this.toString())
        return JsonValue.fromJsonNode(node)
    }
    ```

    ### Lógica de procesamiento de consultas

    Ahora añadamos la funcionalidad esencial para procesar consultas y gestionar llamadas a Herramientas:

    ```kotlin
    private val messageParamsBuilder: MessageCreateParams.Builder = MessageCreateParams.builder()
        .model(Model.CLAUDE_3_5_SONNET_20241022)
        .maxTokens(1024)

    suspend fun processQuery(query: String): String {
        val messages = mutableListOf(
            MessageParam.builder()
                .role(MessageParam.Role.USER)
                .content(query)
                .build()
        )

        val response = anthropic.messages().create(
            messageParamsBuilder
                .messages(messages)
                .tools(tools)
                .build()
        )

        val finalText = mutableListOf<String>()
        response.content().forEach { content ->
            when {
                content.isText() -> finalText.add(content.text().getOrNull()?.text() ?: "")

                content.isToolUse() -> {
                    val toolName = content.toolUse().get().name()
                    val toolArgs =
                        content.toolUse().get()._input().convert(object : TypeReference<Map<String, JsonValue>>() {})

                    val result = mcp.callTool(
                        name = toolName,
                        arguments = toolArgs ?: emptyMap()
                    )
                    finalText.add("[Usando la herramienta $toolName con argumentos $toolArgs]")

                    messages.add(
                        MessageParam.builder()
                            .role(MessageParam.Role.USER)
                            .content(
                                """
                                    "type": "tool_result",
                                    "tool_name": $toolName,
                                    "result": ${result?.content?.joinToString("\n") { (it as TextContent).text ?: "" }}
                                """.trimIndent()
                            )
                            .build()
                    )

                    val aiResponse = anthropic.messages().create(
                        messageParamsBuilder
                            .messages(messages)
                            .build()
                    )

                    finalText.add(aiResponse.content().first().text().getOrNull()?.text() ?: "")
                }
            }
        }

        return finalText.joinToString("\n", prefix = "", postfix = "")
    }
    ```

    6. Claude ofrece una respuesta en lenguaje natural
    7. La respuesta se muestra para ti

    ### 1. Inicialización del Cliente

    * El cliente se inicializa con `McpClientFactory.CreateAsync()`, que configura el tipo de transporte y el comando para ejecutar el servidor.

    ### 2. Conexión con el servidor

    * Compatible con servidores de Python, Node.js y .NET.
    * El servidor se inicia con el comando especificado en los argumentos.
    * Configura el uso de STDIO para la comunicación con el servidor.
    * Inicializa la sesión y las Herramientas disponibles.

    ### 3. Procesamiento de consultas

    * Utiliza [Microsoft.Extensions.AI](https://learn.microsoft.com/dotnet/ai/ai-extensions) para el cliente de chat.
    * Configura `IChatClient` para usar la invocación automática de Herramientas (funciones).
    * El cliente lee la entrada del usuario y la envía al servidor.
    * El servidor procesa la consulta y devuelve una respuesta.
    * La respuesta se muestra al usuario.

    ## Ejecución del Cliente

    Para ejecutar tu cliente con cualquier Servidor MCP:

    ```bash
    dotnet run -- path/to/server.csproj # servidor de .NET
    dotnet run -- path/to/server.py # servidor de Python
    dotnet run -- path/to/server.js # servidor de Node.js
    ```

    <Note>
      Si continúas con el tutorial del clima a partir del inicio rápido del servidor, tu comando podría verse así: `dotnet run -- path/to/QuickstartWeatherServer`.
    </Note>

    El cliente hará:

    1. Conectar con el servidor especificado
    2. Listar las Herramientas disponibles
    3. Inicia una sesión de chat interactiva en la que puedes:
       * Introducir consultas
       * Ver ejecuciones de Herramientas
       * Obtener respuestas de Claude
    4. Sal de la sesión cuando termines
