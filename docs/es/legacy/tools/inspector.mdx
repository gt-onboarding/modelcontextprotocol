El [Inspector MCP](https://github.com/modelcontextprotocol/inspector) es una herramienta interactiva para desarrolladores destinada a probar y depurar servidores MCP. Si bien la [Guía de depuración](/es/legacy/tools/debugging) aborda el Inspector como parte del conjunto general de herramientas de depuración, este documento ofrece una exploración detallada de las funciones y capacidades del Inspector.

<div id="getting-started">
  ## Introducción
</div>

<div id="installation-and-basic-usage">
  ### Instalación y uso básico
</div>

El Inspector se ejecuta directamente con `npx`, sin necesidad de instalación:

```bash
npx @modelcontextprotocol/inspector <command>
```

```bash
npx @modelcontextprotocol/inspector <command> <arg1> <arg2>
```

<div id="inspecting-servers-from-npm-or-pypi">
  #### Inspeccionar servidores desde NPM o PyPi
</div>

Una forma común de iniciar paquetes de servidor desde [NPM](https://npmjs.com) o [PyPi](https://pypi.org).

<Tabs>
  <Tab title="Paquete de NPM">
    ```bash
    npx -y @modelcontextprotocol/inspector npx <package-name> <args>
    # Por ejemplo
    npx -y @modelcontextprotocol/inspector npx @modelcontextprotocol/server-filesystem /Users/username/Desktop
    ```
  </Tab>

  <Tab title="Paquete de PyPi">
    ```bash
    npx @modelcontextprotocol/inspector uvx <package-name> <args>
    # Por ejemplo
    npx @modelcontextprotocol/inspector uvx mcp-server-git --repository ~/code/mcp/servers.git
    ```
  </Tab>
</Tabs>

<div id="inspecting-locally-developed-servers">
  #### Inspección de servidores desarrollados localmente
</div>

Para inspeccionar servidores desarrollados localmente o descargados como un repositorio, la forma más común es:

<Tabs>
  <Tab title="TypeScript">
    ```bash
    npx @modelcontextprotocol/inspector node path/to/server/index.js args...
    ```
  </Tab>

  <Tab title="Python">
    ```bash
    npx @modelcontextprotocol/inspector \
      uv \
      --directory path/to/server \
      run \
      package-name \
      args...
    ```
  </Tab>
</Tabs>

Lee atentamente cualquier README adjunto para obtener las instrucciones más precisas.

<div id="feature-overview">
  ## Descripción general de funciones
</div>

<Frame caption="La interfaz del Inspector MCP">
  <img src="/images/mcp-inspector.png" />
</Frame>

El Inspector ofrece varias funciones para interactuar con tu Servidor MCP:

<div id="server-connection-pane">
  ### Panel de conexión al servidor
</div>

* Permite seleccionar el [transporte](/es/legacy/concepts/transports) para conectarse al servidor
* Para servidores locales, permite personalizar los argumentos de la línea de comandos y el entorno

<div id="resources-tab">
  ### Pestaña Recursos
</div>

* Lista todos los Recursos disponibles
* Muestra metadatos de Recursos (tipos MIME, descripciones)
* Permite inspeccionar el contenido de los Recursos
* Permite probar suscripciones

<div id="prompts-tab">
  ### Pestaña de Indicaciones
</div>

* Muestra las plantillas de indicaciones disponibles
* Muestra los argumentos y descripciones de las indicaciones
* Permite probar las indicaciones con argumentos personalizados
* Muestra una vista previa de los mensajes generados

<div id="tools-tab">
  ### Pestaña Herramientas
</div>

* Lista las Herramientas disponibles
* Muestra los esquemas y descripciones de las Herramientas
* Permite probar Herramientas con entradas personalizadas
* Muestra los resultados de la ejecución de las Herramientas

<div id="notifications-pane">
  ### Panel de notificaciones
</div>

* Muestra todos los registros del servidor
* Muestra las notificaciones recibidas del servidor

<div id="best-practices">
  ## Prácticas recomendadas
</div>

<div id="development-workflow">
  ### Flujo de trabajo de desarrollo
</div>

1. Iniciar el desarrollo
   * Iniciar el Inspector con tu servidor
   * Verificar la conectividad básica
   * Comprobar la negociación de capacidades

2. Pruebas iterativas
   * Realizar cambios en el servidor
   * Volver a compilar el servidor
   * Volver a conectar el Inspector
   * Probar las funciones afectadas
   * Supervisar los mensajes

3. Probar casos límite
   * Entradas no válidas
   * Argumentos de indicación faltantes
   * Operaciones concurrentes
   * Verificar el manejo y las respuestas de error
