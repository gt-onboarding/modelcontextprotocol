La depuración eficaz es esencial al desarrollar Servidores MCP o al integrarlos con aplicaciones. Esta guía cubre las Herramientas y enfoques de depuración disponibles en el ecosistema MCP.

<Info>
  Esta guía es para macOS. Pronto habrá guías para otras plataformas.
</Info>

<div id="debugging-tools-overview">
  ## Descripción general de las herramientas de depuración
</div>

MCP proporciona varias herramientas para depurar en diferentes niveles:

1. **Inspector MCP**
   * Interfaz de depuración interactiva
   * Pruebas directas del servidor
   * Consulta la [guía del Inspector](/es/legacy/tools/inspector) para más detalles

2. **Herramientas para desarrolladores de Claude Desktop**
   * Pruebas de integración
   * Recopilación de registros
   * Integración con Chrome DevTools

3. **Registro del servidor**
   * Implementaciones de registro personalizadas
   * Seguimiento de errores
   * Supervisión del rendimiento

<div id="debugging-in-claude-desktop">
  ## Depuración en Claude Desktop
</div>

<div id="checking-server-status">
  ### Comprobar el estado del servidor
</div>

La interfaz de Claude.app proporciona información básica sobre el estado del servidor:

1. Haz clic en el icono <img src="/images/claude-desktop-mcp-plug-icon.svg" style={{display: 'inline', margin: 0, height: '1.3em'}} /> para ver:
   * Servidores conectados
   * Indicaciones y recursos disponibles

2. Haz clic en el icono «Search and tools» <img src="/images/claude-desktop-mcp-slider.svg" style={{display: 'inline', margin: 0, height: '1.3em'}} /> para ver:
   * Herramientas disponibles para el modelo

<div id="viewing-logs">
  ### Ver registros
</div>

Consulta los registros detallados de MCP en Claude Desktop:

```bash
# Seguir los registros en tiempo real
tail -n 20 -F ~/Library/Logs/Claude/mcp*.log
```

Los registros incluyen:

* Eventos de conexión del servidor
* Problemas de configuración
* Errores en tiempo de ejecución
* Intercambios de mensajes

<div id="using-chrome-devtools">
  ### Uso de Chrome DevTools
</div>

Accede a las herramientas de desarrollador de Chrome dentro de Claude Desktop para investigar errores del lado del cliente:

1. Crea un archivo `developer_settings.json` con `allowDevTools` en true:

```bash
echo '{"allowDevTools": true}' > ~/Library/Application\ Support/Claude/developer_settings.json
```

2. Abre DevTools: `Command-Option-Shift-i`

Nota: Verás dos ventanas de DevTools:

* Ventana de contenido principal
* Ventana de la barra de título de la aplicación

Usa el panel Console para inspeccionar errores del lado del cliente.

Usa el panel Network para inspeccionar:

* Cargas de mensajes
* Tiempos de conexión

<div id="common-issues">
  ## Problemas frecuentes
</div>

<div id="working-directory">
  ### Directorio de trabajo
</div>

Al usar servidores MCP con Claude Desktop:

* El directorio de trabajo de los servidores iniciados mediante `claude_desktop_config.json` puede ser indefinido (como `/` en macOS) porque Claude Desktop puede iniciarse desde cualquier ubicación
* Utiliza siempre rutas absolutas en tu configuración y en los archivos `.env` para garantizar un funcionamiento fiable
* Para probar servidores directamente desde la línea de comandos, el directorio de trabajo será aquel desde el que ejecutes el comando

Por ejemplo, en `claude_desktop_config.json`, usa:

```json
{
  "command": "npx",
  "args": [
    "-y",
    "@modelcontextprotocol/server-filesystem",
    "/Users/username/data"
  ]
}
```

En lugar de rutas relativas como `./data`

<div id="environment-variables">
  ### Variables de entorno
</div>

Los Servidores MCP heredan automáticamente solo un subconjunto de variables de entorno, como `USER`, `HOME` y `PATH`.

Para anular las variables predeterminadas o definir las propias, puedes especificar una clave `env` en `claude_desktop_config.json`:

```json
{
  "myserver": {
    "command": "mcp-server-myapp",
    "env": {
      "MYAPP_API_KEY": "some_key"
    }
  }
}
```

<div id="server-initialization">
  ### Inicialización del servidor
</div>

Problemas comunes durante la inicialización:

1. **Problemas de ruta**
   * Ruta incorrecta del ejecutable del servidor
   * Archivos requeridos ausentes
   * Problemas de permisos
   * Prueba usar una ruta absoluta para `command`

2. **Errores de configuración**
   * Sintaxis JSON no válida
   * Campos obligatorios ausentes
   * Incompatibilidades de tipo

3. **Problemas de entorno**
   * Variables de entorno ausentes
   * Valores de variables incorrectos
   * Restricciones de permisos

<div id="connection-problems">
  ### Problemas de conexión
</div>

Cuando los servidores no logran conectarse:

1. Revisa los registros de Claude Desktop
2. Verifica que el proceso del servidor esté en ejecución
3. Prueba de forma independiente con el [Inspector](/es/legacy/tools/inspector)
4. Verifica la compatibilidad del protocolo

<div id="implementing-logging">
  ## Implementación del registro
</div>

<div id="server-side-logging">
  ### Registro en el lado del servidor
</div>

Al crear un servidor que usa el [transporte](/es/legacy/concepts/transports) local stdio, todos los mensajes registrados en stderr (error estándar) serán capturados automáticamente por la aplicación host (p. ej., Claude Desktop).

<Warning>
  Los servidores MCP locales no deben registrar mensajes en stdout (salida estándar), ya que esto interferirá con el funcionamiento del protocolo.
</Warning>

Para todos los [transportes](/es/legacy/concepts/transports), también puedes proporcionar registros al cliente enviando una notificación de mensaje de registro:

<CodeGroup>
  ```python Python
  server.request_context.session.send_log_message(
    level="info",
    data="Server started successfully",
  )
  ```

  ```typescript TypeScript
  server.sendLoggingMessage({
    level: "info",
    data: "Server started successfully",
  });
  ```
</CodeGroup>

Eventos importantes para registrar:

* Pasos de inicialización
* Acceso a Recursos
* Ejecución de Herramientas
* Condiciones de error
* Métricas de rendimiento

<div id="client-side-logging">
  ### Registro del lado del cliente
</div>

En aplicaciones cliente:

1. Habilita el registro de depuración
2. Supervisa el tráfico de red
3. Rastrea el intercambio de mensajes
4. Registra los estados de error

<div id="debugging-workflow">
  ## Flujo de trabajo de depuración
</div>

<div id="development-cycle">
  ### Ciclo de desarrollo
</div>

1. Desarrollo inicial
   * Usa el [Inspector](/es/legacy/tools/inspector) para pruebas básicas
   * Implementa la funcionalidad central
   * Añade puntos de registro

2. Pruebas de integración
   * Prueba en Claude Desktop
   * Supervisa los registros
   * Verifica el manejo de errores

<div id="testing-changes">
  ### Prueba de cambios
</div>

Para probar cambios de forma eficiente:

* **Cambios de configuración**: Reinicia Claude Desktop
* **Cambios en el código del servidor**: Usa Command-R para recargar
* **Iteración rápida**: Usa el [Inspector](/es/legacy/tools/inspector) durante el desarrollo

<div id="best-practices">
  ## Mejores prácticas
</div>

<div id="logging-strategy">
  ### Estrategia de registro
</div>

1. **Registro estructurado**
   * Usar formatos coherentes
   * Incluir contexto
   * Añadir marcas de tiempo
   * Registrar IDs de solicitud

2. **Gestión de errores**
   * Registrar trazas de pila
   * Incluir contexto del error
   * Registrar patrones de error
   * Supervisar la recuperación

3. **Seguimiento del rendimiento**
   * Registrar el tiempo de las operaciones
   * Supervisar el uso de recursos
   * Registrar tamaños de mensajes
   * Medir la latencia

<div id="security-considerations">
  ### Consideraciones de seguridad
</div>

Al depurar:

1. **Datos sensibles**
   * Depura/limpia los registros
   * Protege las credenciales
   * Enmascara la información personal

2. **Control de acceso**
   * Verifica los permisos
   * Comprueba la autenticación
   * Supervisa los patrones de acceso

  ## Obtener ayuda

2. **Canales de soporte**

3. **Información a proporcionar**
   * Extractos de registros

   * Pasos para reproducir el problema
   * Detalles del entorno
