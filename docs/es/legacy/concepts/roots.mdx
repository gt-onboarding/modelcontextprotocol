---
title: "Raíces"
description: "Comprender las raíces en MCP"
---

Las raíces son un concepto en MCP que define los límites dentro de los cuales pueden operar los servidores. Proporcionan una forma para que los clientes informen a los servidores sobre los recursos relevantes y sus ubicaciones.

<div id="what-are-roots">
  ## ¿Qué son las Raíces?
</div>

Una raíz es un URI en el que un Cliente MCP sugiere que un Servidor MCP debería centrarse. Cuando un cliente se conecta a un servidor, declara con qué raíces debería trabajar el servidor. Aunque se usan principalmente para rutas de sistema de archivos, las raíces pueden ser cualquier URI válido, incluidas las URL HTTP.

Por ejemplo, las raíces podrían ser:

```
file:///home/user/projects/myapp
https://api.example.com/v1
```

<div id="why-use-roots">
  ## ¿Por qué usar Raíces?
</div>

Las Raíces cumplen varios propósitos importantes:

1. **Guía**: Informan a los servidores sobre los recursos y las ubicaciones relevantes
2. **Claridad**: Las Raíces dejan claro qué recursos forman parte de tu espacio de trabajo
3. **Organización**: Múltiples Raíces te permiten trabajar con diferentes recursos simultáneamente

<div id="how-roots-work">
  ## Cómo funcionan las Raíces
</div>

Cuando un cliente admite Raíces, este:

1. Declara la capacidad `roots` durante la conexión
2. Proporciona una lista de Raíces sugeridas al servidor
3. Notifica al servidor cuando cambian las Raíces (si es compatible)

Aunque las Raíces son informativas y no estrictamente obligatorias, los servidores deberían:

1. Respetar las Raíces proporcionadas
2. Usar URIs de Raíz para ubicar y acceder a Recursos
3. Priorizar operaciones dentro de los límites de la Raíz

<div id="common-use-cases">
  ## Casos de uso comunes
</div>

Las Raíces se utilizan comúnmente para definir:

- Directorios de proyecto
- Ubicaciones de repositorio
- Puntos de conexión de API
- Ubicaciones de configuración
- Límites de Recursos

<div id="best-practices">
  ## Mejores prácticas
</div>

Al trabajar con Raíces:

1. Solo sugiere los Recursos necesarios
2. Usa nombres claros y descriptivos para las Raíces
3. Supervisa la accesibilidad de las Raíces
4. Maneja los cambios en las Raíces de forma adecuada

<div id="example">
  ## Ejemplo
</div>

Así es como un Cliente MCP típico podría exponer Raíces:

```json
{
  "roots": [
    {
      "uri": "file:///home/user/projects/frontend",
      "name": "Frontend Repository"
    },
    {
      "uri": "https://api.example.com/v1",
      "name": "API Endpoint"
    }
  ]
}
```

Esta configuración sugiere que el Servidor enfoque su atención tanto en un repositorio local como en un endpoint de API, manteniéndolos lógicamente separados.
