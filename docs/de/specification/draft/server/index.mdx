Server stellen die grundlegenden Bausteine bereit, um Sprachmodelle über
den Model Context Protocol (MCP) mit Kontext zu versorgen. Diese Bausteine ermöglichen vielseitige Interaktionen zwischen Clients, Servern und Sprachmodellen:

* **Prompts**: Vordefinierte Vorlagen oder Anweisungen, die die Interaktionen mit dem Sprachmodell steuern
* **Ressourcen**: Strukturierte Daten oder Inhalte, die dem Modell zusätzlichen Kontext liefern
* **Werkzeuge**: Ausführbare Funktionen, mit denen Modelle Aktionen ausführen oder Informationen abrufen können

Jeder dieser Bausteine lässt sich in der folgenden Steuerungshierarchie zusammenfassen:

| Primitive | Steuerung              | Beschreibung                                       | Beispiel                        |
| --------- | ---------------------- | -------------------------------------------------- | ------------------------------- |
| Prompts   | Benutzerkontrolliert   | Interaktive Vorlagen, die durch Benutzerauswahl aufgerufen werden | Slash-Befehle, Menüoptionen     |
| Ressourcen | Anwendungsseitig kontrolliert | Kontextdaten, die vom Client angehängt und verwaltet werden | Dateiinhalte, Git-Verlauf       |
| Werkzeuge | Modellkontrolliert     | Funktionen, die dem LLM zum Ausführen von Aktionen bereitgestellt werden | API-POST-Anfragen, Datei-Schreibvorgänge |
