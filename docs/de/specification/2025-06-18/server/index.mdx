Server liefern die grundlegenden Bausteine, um über das Model Context Protocol (MCP) Kontext zu Sprachmodellen hinzuzufügen. Diese Primitiven ermöglichen umfassende Interaktionen zwischen Clients, Servern und Sprachmodellen:

* **Prompts**: Vordefinierte Vorlagen oder Anweisungen, die die Interaktion mit dem Sprachmodell steuern
* **Ressourcen**: Strukturierte Daten oder Inhalte, die dem Modell zusätzlichen Kontext liefern
* **Werkzeuge**: Ausführbare Funktionen, mit denen Modelle Aktionen ausführen oder Informationen abrufen können

Jedes dieser Primitiven lässt sich in der folgenden Steuerungshierarchie zusammenfassen:

| Primitive | Steuerung              | Beschreibung                                       | Beispiel                        |
| --------- | ---------------------- | -------------------------------------------------- | ------------------------------- |
| Prompts   | Nutzerkontrolliert     | Interaktive Vorlagen, die nach Nutzerwahl aufgerufen werden | Slash-Befehle, Menüoptionen     |
| Ressourcen | Anwendungskontrolliert | Kontextdaten, die vom Client angehängt und verwaltet werden | Dateiinhalte, Git-Verlauf       |
| Werkzeuge | Modellkontrolliert     | Funktionen, die dem LLM bereitgestellt werden, um Aktionen auszuführen | API-POST-Anfragen, Dateischreiben |
