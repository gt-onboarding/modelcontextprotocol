---
title: Was ist das Model Context Protocol (MCP)?
sidebarTitle: Was ist MCP?
---

MCP (Model Context Protocol) ist ein Open-Source-Standard, der KI-Anwendungen mit externen Systemen verbindet.

Mit MCP können KI-Anwendungen wie Claude oder ChatGPT sich mit Datenquellen (z. B. lokalen Dateien, Datenbanken), Werkzeugen (z. B. Suchmaschinen, Taschenrechnern) und Workflows (z. B. spezialisierten Prompts) verbinden—so erhalten sie Zugriff auf wichtige Informationen und können Aufgaben ausführen.

Stellen Sie sich MCP wie einen USB‑C‑Anschluss für KI-Anwendungen vor. So wie USB‑C eine standardisierte Möglichkeit bietet, elektronische Geräte zu verbinden, bietet MCP eine standardisierte Möglichkeit, KI-Anwendungen mit externen Systemen zu verbinden.

<div id="what-can-mcp-enable">
  ## Was kann MCP ermöglichen?
</div>

* Agents können auf deinen Google-Kalender und Notion zugreifen und als persönlicherer KI-Assistent auftreten.
* Claude Code kann anhand eines Figma-Designs eine komplette Web-App erstellen.
* Enterprise-Chatbots können sich mit mehreren Datenbanken in einer Organisation verbinden und Nutzende dabei unterstützen, Daten per Chat zu analysieren.
* KI-Modelle können 3D-Designs in Blender erstellen und sie mit einem 3D-Drucker ausgeben.

<div id="why-does-mcp-matter">
  ## Warum ist MCP wichtig?
</div>

Je nachdem, wo Sie sich im Ökosystem befinden, kann MCP eine Reihe von Vorteilen bieten.

* **Entwickler**: MCP reduziert die Entwicklungszeit und -komplexität beim Erstellen oder bei der Integration einer KI-Anwendung oder eines Agenten.
* **KI-Anwendungen oder Agenten**: MCP bietet Zugriff auf ein Ökosystem aus Datenquellen, Werkzeugen und Apps, das die Fähigkeiten erweitert und die Nutzererfahrung verbessert.
* **Endnutzer**: MCP führt zu leistungsfähigeren KI-Anwendungen oder Agenten, die auf Ihre Daten zugreifen und bei Bedarf in Ihrem Namen handeln können.
