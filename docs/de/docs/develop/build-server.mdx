<div id="core-mcp-concepts">
  ### Zentrale MCP-Konzepte
</div>

MCP-Server können drei Hauptarten von Fähigkeiten bereitstellen:

1. **[Ressourcen](/de/docs/learn/server-concepts#resources)**: Dateiliche Daten, die von Clients gelesen werden können (z. B. API-Antworten oder Dateiinhalte)
2. **[Werkzeuge](/de/docs/learn/server-concepts#tools)**: Funktionen, die vom LLM aufgerufen werden können (mit Zustimmung der Nutzer)
3. **[Prompts](/de/docs/learn/server-concepts#prompts)**: Vorgefertigte Vorlagen, die Nutzerinnen und Nutzern helfen, bestimmte Aufgaben zu erledigen

Dieses Tutorial konzentriert sich in erster Linie auf Werkzeuge.

    ### Logging in MCP-Servern

    Beim Implementieren von MCP-Servern seien Sie vorsichtig, wie Sie das Logging handhaben:

    **Für STDIO-basierte Server:** Niemals in die Standardausgabe (stdout) schreiben. Dazu zählt:

    * `print()`-Anweisungen in Python
    * `console.log()` in JavaScript
    * `fmt.Println()` in Go
    * Ähnliche stdout-Funktionen in anderen Programmiersprachen

    Das Schreiben auf stdout beschädigt die JSON-RPC-Nachrichten und lässt Ihren Server abstürzen.

    **Für HTTP-basierte Server:** Protokollierung über die Standardausgabe ist unproblematisch, da sie HTTP-Antworten nicht beeinträchtigt.

    ### Bewährte Verfahren

    1. Verwenden Sie eine Logging-Bibliothek, die auf stderr oder in Dateien schreibt.

    ### Kurze Beispiele

    ```python
    # ❌ Schlecht (STDIO)
    print("Verarbeitung der Anfrage")

    # ✅ Gut (STDIO)
    import logging
    logging.info("Verarbeitung der Anfrage")
    ```

    ## Ihren Server erstellen

    ### Pakete importieren und Instanz einrichten

    Fügen Sie dies oben in Ihrer Datei `weather.py` hinzu:

    ```python
    from typing import Any
    import httpx
    from mcp.server.fastmcp import FastMCP

    # FastMCP-Server initialisieren
    mcp = FastMCP("weather")

    # Konstanten
    NWS_API_BASE = "https://api.weather.gov"
    USER_AGENT = "weather-app/1.0"
    ```

    Die Klasse FastMCP nutzt Python-Typannotationen und Docstrings, um Werkzeugdefinitionen automatisch zu generieren. So wird das Erstellen und die Pflege von MCP-Werkzeugen erleichtert.

    ### Hilfsfunktionen

    Als Nächstes fügen wir Hilfsfunktionen hinzu, um Daten aus der National-Weather-Service-API abzurufen und zu formatieren:

    ```python
    async def make_nws_request(url: str) -> dict[str, Any] | None:
        """Stellt eine Anfrage an die NWS-API mit geeigneter Fehlerbehandlung."""
        headers = {
            "User-Agent": USER_AGENT,
            "Accept": "application/geo+json"
        }
        async with httpx.AsyncClient() as client:
            try:
                response = await client.get(url, headers=headers, timeout=30.0)
                response.raise_for_status()
                return response.json()
            except Exception:
                return None

    def format_alert(feature: dict) -> str:
        """Formatiert ein Warnhinweis-Feature als gut lesbaren String."""
        props = feature["properties"]
        return f"""
    Ereignis: {props.get('event', 'Unbekannt')}
    Gebiet: {props.get('areaDesc', 'Unbekannt')}
    Schweregrad: {props.get('severity', 'Unbekannt')}
    Beschreibung: {props.get('description', 'Keine Beschreibung verfügbar')}
    Anweisungen: {props.get('instruction', 'Keine spezifischen Anweisungen vorhanden')}
    """
    ```

    ### Ausführung von Werkzeugen implementieren

    Der Handler für die Werkzeugausführung ist dafür verantwortlich, die Logik jedes einzelnen Werkzeugs tatsächlich auszuführen. Fügen wir ihn hinzu:

    ```python
    @mcp.tool()
    async def get_alerts(state: str) -> str:
        """Wetterwarnungen für einen US-Bundesstaat abrufen.

        Args:
            state: US-Bundesstaat mit zweibuchstabigem Code (z. B. CA, NY)
        """
        url = f"{NWS_API_BASE}/alerts/active/area/{state}"
        data = await make_nws_request(url)

        if not data or "features" not in data:
            return "Warnungen konnten nicht abgerufen werden oder es wurden keine Warnungen gefunden."

        if not data["features"]:
            return "Keine aktiven Warnungen für diesen Bundesstaat."

        alerts = [format_alert(feature) for feature in data["features"]]
        return "\n---\n".join(alerts)

    @mcp.tool()
    async def get_forecast(latitude: float, longitude: float) -> str:
        """Wettervorhersage für einen Ort abrufen.

        Args:
            latitude: Breitengrad des Ortes
            longitude: Längengrad des Ortes
        """
        # Zuerst den Vorhersage-Grid-Endpunkt abrufen
        points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
        points_data = await make_nws_request(points_url)

        if not points_data:
            return "Vorhersagedaten für diesen Ort konnten nicht abgerufen werden."

        # Die Vorhersage-URL aus der Points-Antwort abrufen
        forecast_url = points_data["properties"]["forecast"]
        forecast_data = await make_nws_request(forecast_url)

        if not forecast_data:
            return "Detaillierte Vorhersage konnte nicht abgerufen werden."

        # Die Zeitabschnitte zu einer lesbaren Vorhersage formatieren
        periods = forecast_data["properties"]["periods"]
        forecasts = []
        for period in periods[:5]:  # Nur die nächsten 5 Zeitabschnitte anzeigen
            forecast = f"""
    {period['name']}:
    Temperatur: {period['temperature']}°{period['temperatureUnit']}
    Wind: {period['windSpeed']} {period['windDirection']}
    Prognose: {period['detailedForecast']}
    """
            forecasts.append(forecast)

        return "\n---\n".join(forecasts)
    ```

    ### Server ausführen

    Zum Schluss initialisieren und starten wir den Server:

    ```python
    if __name__ == "__main__":
        # Server initialisieren und starten
        mcp.run(transport='stdio')
    ```

    Ihr Server ist fertig! Führen Sie `uv run weather.py` aus, um den MCP-Server zu starten, der auf Nachrichten von MCP-Hosts wartet.

    ### Logging in MCP-Servern

    Beim Implementieren von MCP-Servern sollten Sie darauf achten, wie Sie das Logging handhaben:

    **Für STDIO-basierte Server:** Niemals in die Standardausgabe (stdout) schreiben. Dazu zählen:

    * `print()`-Anweisungen in Python
    * `console.log()` in JavaScript
    * `fmt.Println()` in Go
    * Ähnliche stdout-Funktionen in anderen Programmiersprachen

    Ausgaben auf stdout beschädigen die JSON-RPC-Nachrichten und legen Ihren Server lahm.

    **Für HTTP-basierte Server:** Protokollierung über die Standardausgabe ist unproblematisch, da sie HTTP-Antworten nicht beeinträchtigt.

    ### Best Practices

    1. Verwenden Sie eine Protokollierungsbibliothek, die nach stderr oder in Dateien schreibt, etwa `logging` in Python.
    2. Für JavaScript ist besondere Vorsicht geboten – `console.log()` schreibt standardmäßig auf stdout.

    ### Kurze Beispiele

    ```javascript
    // ❌ Schlecht (STDIO)
    console.log("Server gestartet");

    // ✅ Gut (STDIO)
    console.error("Server gestartet"); // stderr ist unbedenklich
    ```

    ## Ihren Server erstellen

    ### Pakete importieren und Instanz einrichten

    Fügen Sie diese am Anfang Ihrer Datei `src/index.ts` hinzu:

    ```typescript
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
    import { z } from "zod";

    const NWS_API_BASE = "https://api.weather.gov";
    const USER_AGENT = "weather-app/1.0";

    // Serverinstanz erstellen
    const server = new McpServer({
      name: "weather",
      version: "1.0.0",
      capabilities: {
        resources: {},
        tools: {},
      },
    });
    ```

    ### Hilfsfunktionen

    Als Nächstes fügen wir unsere Hilfsfunktionen hinzu, um die Daten der National Weather Service API abzufragen und zu formatieren:

    ```typescript
    // Hilfsfunktion für NWS-API-Anfragen
    async function makeNWSRequest<T>(url: string): Promise<T | null> {
      const headers = {
        "User-Agent": USER_AGENT,
        Accept: "application/geo+json",
      };

      try {
        const response = await fetch(url, { headers });
        if (!response.ok) {
          throw new Error(`HTTP-Fehler! Status: ${response.status}`);
        }
        return (await response.json()) as T;
      } catch (error) {
        console.error("Fehler bei der NWS-Anfrage:", error);
        return null;
      }
    }

    interface AlertFeature {
      properties: {
        event?: string;
        areaDesc?: string;
        severity?: string;
        status?: string;
        headline?: string;
      };
    }

    // Warnmeldungen formatieren
    function formatAlert(feature: AlertFeature): string {
      const props = feature.properties;
      return [
        `Ereignis: ${props.event || "Unbekannt"}`,
        `Gebiet: ${props.areaDesc || "Unbekannt"}`,
        `Schweregrad: ${props.severity || "Unbekannt"}`,
        `Status: ${props.status || "Unbekannt"}`,
        `Titel: ${props.headline || "Kein Titel"}`,
        "---",
      ].join("\n");
    }

    interface ForecastPeriod {
      name?: string;
      temperature?: number;
      temperatureUnit?: string;
      windSpeed?: string;
      windDirection?: string;
      shortForecast?: string;
    }

    interface AlertsResponse {
      features: AlertFeature[];
    }

    interface PointsResponse {
      properties: {
        forecast?: string;
      };
    }

    interface ForecastResponse {
      properties: {
        periods: ForecastPeriod[];
      };
    }
    ```

    ### Implementierung der Werkzeugausführung

    Der Handler für die Werkzeugausführung ist dafür zuständig, die Logik jedes Werkzeugs auszuführen. Fügen wir ihn hinzu:

    ```typescript
    // Wetterwerkzeuge registrieren
    server.tool(
      "get_alerts",
      "Wetterwarnungen für einen Bundesstaat abrufen",
      {
        state: z.string().length(2).describe("Zweistelliger Bundesstaatscode (z. B. CA, NY)"),
      },
      async ({ state }) => {
        const stateCode = state.toUpperCase();
        const alertsUrl = `${NWS_API_BASE}/alerts?area=${stateCode}`;
        const alertsData = await makeNWSRequest<AlertsResponse>(alertsUrl);

        if (!alertsData) {
          return {
            content: [
              {
                type: "text",
                text: "Warnungsdaten konnten nicht abgerufen werden",
              },
            ],
          };
        }

        const features = alertsData.features || [];
        if (features.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: `Keine aktiven Warnungen für ${stateCode}`,
              },
            ],
          };
        }

        const formattedAlerts = features.map(formatAlert);
        const alertsText = `Aktive Warnungen für ${stateCode}:\n\n${formattedAlerts.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: alertsText,
            },
          ],
        };
      },
    );

    server.tool(
      "get_forecast",
      "Wettervorhersage für einen Ort abrufen",
      {
        latitude: z.number().min(-90).max(90).describe("Breitengrad des Ortes"),
        longitude: z
          .number()
          .min(-180)
          .max(180)
          .describe("Längengrad des Ortes"),
      },
      async ({ latitude, longitude }) => {
        // Gitterpunktdaten abrufen
        const pointsUrl = `${NWS_API_BASE}/points/${latitude.toFixed(4)},${longitude.toFixed(4)}`;
        const pointsData = await makeNWSRequest<PointsResponse>(pointsUrl);

        if (!pointsData) {
          return {
            content: [
              {
                type: "text",
                text: `Gitterpunktdaten für die Koordinaten konnten nicht abgerufen werden: ${latitude}, ${longitude}. Dieser Ort wird möglicherweise nicht von der NWS-API unterstützt (nur US-Standorte werden unterstützt).`,
              },
            ],
          };
        }

        const forecastUrl = pointsData.properties?.forecast;
        if (!forecastUrl) {
          return {
            content: [
              {
                type: "text",
                text: "Vorhersage-URL aus Gitterpunktdaten konnte nicht ermittelt werden",
              },
            ],
          };
        }

        // Vorhersagedaten abrufen
        const forecastData = await makeNWSRequest<ForecastResponse>(forecastUrl);
        if (!forecastData) {
          return {
            content: [
              {
                type: "text",
                text: "Vorhersagedaten konnten nicht abgerufen werden",
              },
            ],
          };
        }

        const periods = forecastData.properties?.periods || [];
        if (periods.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: "Keine Vorhersagezeiträume verfügbar",
              },
            ],
          };
        }

        // Vorhersagezeiträume formatieren
        const formattedForecast = periods.map((period: ForecastPeriod) =>
          [
            `${period.name || "Unbekannt"}:`,
            `Temperatur: ${period.temperature || "Unbekannt"}°${period.temperatureUnit || "F"}`,
            `Wind: ${period.windSpeed || "Unbekannt"} ${period.windDirection || ""}`,
            `${period.shortForecast || "Keine Vorhersage verfügbar"}`,
            "---",
          ].join("\n"),
        );

        const forecastText = `Vorhersage für ${latitude}, ${longitude}:\n\n${formattedForecast.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: forecastText,
            },
          ],
        };
      },
    );
    ```

    ### Logging in MCP-Servern

    Beim Implementieren von MCP-Servern sollten Sie sorgfältig darauf achten, wie Sie das Logging handhaben:

    **Für STDIO-basierte Server:** Niemals auf die Standardausgabe (stdout) schreiben. Dies umfasst:

    * `print()`-Anweisungen in Python
    * `console.log()` in JavaScript
    * `fmt.Println()` in Go
    * Ähnliche stdout-Funktionen in anderen Programmiersprachen

    Ausgaben auf stdout beschädigen die JSON-RPC 2.0‑Nachrichten und bringen Ihren Server zum Absturz.

    **Für HTTP-basierte Server:** Protokollierung über die Standardausgabe ist unproblematisch, da sie HTTP-Antworten nicht beeinträchtigt.

    ### Bewährte Verfahren

    1. Verwenden Sie eine Logging-Bibliothek, die auf stderr oder in Dateien schreibt.

    fun `run mcp server`() {
        // Erstelle die MCP-Server-Instanz mit einer einfachen Implementierung
        val server = Server(
            Implementation(
                name = "weather", // Werkzeugname ist „weather“
                version = "1.0.0" // Version der Implementierung
            ),
            ServerOptions(
                capabilities = ServerCapabilities(tools = ServerCapabilities.Tools(listChanged = true))
            )
        )

        // Erstelle einen Transport über Standard-I/O für die Serverkommunikation
        val transport = StdioServerTransport(
            System.`in`.asInput(),
            System.out.asSink().buffered()
        )

        runBlocking {
            server.connect(transport)
            val done = Job()
            server.onClose {
                done.complete()
            }
            done.join()
        }
    }

    ### Hilfsfunktionen für die Wetter-API

    Als Nächstes fügen wir Funktionen und Datenklassen hinzu, um Antworten der National-Weather-Service-API abzurufen und zu konvertieren:

    ```kotlin
    // Erweiterungsfunktion zum Abrufen von Vorhersageinformationen für eine bestimmte Breite und Länge
    suspend fun HttpClient.getForecast(latitude: Double, longitude: Double): List<String> {
        val points = this.get("/points/$latitude,$longitude").body<Points>()
        val forecast = this.get(points.properties.forecast).body<Forecast>()
        return forecast.properties.periods.map { period ->
            """
                ${period.name}:
                Temperatur: ${period.temperature} ${period.temperatureUnit}
                Wind: ${period.windSpeed} ${period.windDirection}
                Prognose: ${period.detailedForecast}
            """.trimIndent()
        }
    }

    // Erweiterungsfunktion zum Abrufen von Wetterwarnungen für einen bestimmten Bundesstaat
    suspend fun HttpClient.getAlerts(state: String): List<String> {
        val alerts = this.get("/alerts/active/area/$state").body<Alert>()
        return alerts.features.map { feature ->
            """
                Ereignis: ${feature.properties.event}
                Gebiet: ${feature.properties.areaDesc}
                Schweregrad: ${feature.properties.severity}
                Beschreibung: ${feature.properties.description}
                Hinweise: ${feature.properties.instruction}
            """.trimIndent()
        }
    }

    @Serializable
    data class Points(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val forecast: String)
    }

    @Serializable
    data class Forecast(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val periods: List<Period>)

        @Serializable
        data class Period(
            val number: Int, val name: String, val startTime: String, val endTime: String,
            val isDaytime: Boolean, val temperature: Int, val temperatureUnit: String,
            val temperatureTrend: String, val probabilityOfPrecipitation: JsonObject,
            val windSpeed: String, val windDirection: String,
            val shortForecast: String, val detailedForecast: String,
        )
    }

    @Serializable
    data class Alert(
        val features: List<Feature>
    ) {
        @Serializable
        data class Feature(
            val properties: Properties
        )

        @Serializable
        data class Properties(
            val event: String, val areaDesc: String, val severity: String,
            val description: String, val instruction: String?,
        )
    }
    ```

    ### Ausführung von Werkzeugen implementieren

    Der Handler für die Werkzeugausführung ist dafür verantwortlich, die Logik jedes Werkzeugs wirklich auszuführen. Fügen wir ihn hinzu:

    ```kotlin
    // HTTP-Client mit Standard-Request-Konfiguration und JSON-Content-Negotiation erstellen
    val httpClient = HttpClient {
        defaultRequest {
            url("https://api.weather.gov")
            headers {
                append("Accept", "application/geo+json")
                append("User-Agent", "WeatherApiClient/1.0")
            }
            contentType(ContentType.Application.Json)
        }
        // Content-Negotiation-Plugin für JSON-Serialisierung/Deserialisierung installieren
        install(ContentNegotiation) { json(Json { ignoreUnknownKeys = true }) }
    }

    // Ein Werkzeug registrieren, um Wetterwarnungen nach Bundesstaat abzurufen
    server.addTool(
        name = "get_alerts",
        description = """
            Wetterwarnungen für einen US-Bundesstaat abrufen. Eingabe ist der zweibuchstabige Code des US-Bundesstaats (z. B. CA, NY)
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("state") {
                    put("type", "string")
                    put("description", "Zweibuchstabiger Code des US-Bundesstaats (z. B. CA, NY)")
                }
            },
            required = listOf("state")
        )
    ) { request ->
        val state = request.arguments["state"]?.jsonPrimitive?.content
        if (state == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("Der Parameter „state“ ist erforderlich."))
            )
        }

        val alerts = httpClient.getAlerts(state)

        CallToolResult(content = alerts.map { TextContent(it) })
    }

    // Ein Werkzeug registrieren, um die Wettervorhersage anhand von Breiten- und Längengrad abzurufen
    server.addTool(
        name = "get_forecast",
        description = """
            Wettervorhersage für eine bestimmte Breite/Länge abrufen
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("latitude") { put("type", "number") }
                putJsonObject("longitude") { put("type", "number") }
            },
            required = listOf("latitude", "longitude")
        )
    ) { request ->
        val latitude = request.arguments["latitude"]?.jsonPrimitive?.doubleOrNull
        val longitude = request.arguments["longitude"]?.jsonPrimitive?.doubleOrNull
        if (latitude == null || longitude == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("Die Parameter „latitude“ und „longitude“ sind erforderlich."))
            )
        }

        val forecast = httpClient.getForecast(latitude, longitude)

        CallToolResult(content = forecast.map { TextContent(it) })
    }
    ```

    ### Logging in MCP-Servern

    Beim Implementieren von MCP-Servern achten Sie sorgfältig darauf, wie Sie das Logging handhaben:

    **Für STDIO-basierte Server:** Niemals auf die Standardausgabe (stdout) schreiben. Dies umfasst:

    * `print()`-Anweisungen in Python
    * `console.log()` in JavaScript
    * `fmt.Println()` in Go
    * Ähnliche stdout-Funktionen in anderen Programmiersprachen

    Ausgaben auf stdout beschädigen die JSON-RPC 2.0-Nachrichten und bringen Ihren Server zum Absturz.

    **Für HTTP-basierte Server:** Logging auf der Standardausgabe ist unproblematisch, da es HTTP-Antworten nicht beeinträchtigt.

    ## Ihren Server erstellen

    Öffnen Sie die Datei `Program.cs` in Ihrem Projekt und ersetzen Sie ihren Inhalt durch den folgenden Code:

    ```csharp
    using Microsoft.Extensions.DependencyInjection;
    using Microsoft.Extensions.Hosting;
    using ModelContextProtocol;
    using System.Net.Http.Headers;

    var builder = Host.CreateEmptyApplicationBuilder(settings: null);

    builder.Services.AddMcpServer()
        .WithStdioServerTransport()
        .WithToolsFromAssembly();

    builder.Services.AddSingleton(_ =>
    {
        var client = new HttpClient() { BaseAddress = new Uri("https://api.weather.gov") };
        client.DefaultRequestHeaders.UserAgent.Add(new ProductInfoHeaderValue("weather-tool", "1.0"));
        return client;
    });

    var app = builder.Build();

    await app.RunAsync();
    ```

    <Note>
      Wenn Sie den `ApplicationHostBuilder` erstellen, verwenden Sie `CreateEmptyApplicationBuilder` statt `CreateDefaultBuilder`. So wird verhindert, dass der Server zusätzliche Nachrichten auf die Konsole schreibt. Dies ist nur für Server erforderlich, die den STDIO-Transport verwenden.
    </Note>

    Dieser Code richtet eine einfache Konsolenanwendung ein, die das Model Context Protocol (MCP) SDK verwendet, um einen MCP-Server mit STDIO-Transport (Standard Input/Output) zu erstellen.

    ### Hilfsfunktionen für die Wetter-API

    Erstellen Sie eine Erweiterungsklasse für `HttpClient`, die die Verarbeitung von JSON-Anfragen vereinfacht:

    ```csharp
    using System.Text.Json;

    internal static class HttpClientExt
    {
        public static async Task<JsonDocument> ReadJsonDocumentAsync(this HttpClient client, string requestUri)
        {
            using var response = await client.GetAsync(requestUri);
            response.EnsureSuccessStatusCode();
            return await JsonDocument.ParseAsync(await response.Content.ReadAsStreamAsync());
        }
    }
    ```

    Als Nächstes definieren Sie eine Klasse mit den Handlern zur Werkzeugausführung, um Antworten der National Weather Service API abzufragen und zu konvertieren:

    ```csharp
    using ModelContextProtocol.Server;
    using System.ComponentModel;
    using System.Globalization;
    using System.Text.Json;

    namespace QuickstartWeatherServer.Tools;

    [McpServerToolType]
    public static class WeatherTools
    {
        [McpServerTool, Description("Wetterwarnungen für einen US-Bundesstaat abrufen.")]
        public static async Task<string> GetAlerts(
            HttpClient client,
            [Description("US-Bundesstaat, für den Warnungen abgerufen werden sollen.")] string state)
        {
            using var jsonDocument = await client.ReadJsonDocumentAsync($"/alerts/active/area/{state}");
            var jsonElement = jsonDocument.RootElement;
            var alerts = jsonElement.GetProperty("features").EnumerateArray();

            if (!alerts.Any())
            {
                return "Keine aktiven Warnungen für diesen Bundesstaat.";
            }

            return string.Join("\n--\n", alerts.Select(alert =>
            {
                JsonElement properties = alert.GetProperty("properties");
                return $"""
                        Ereignis: {properties.GetProperty("event").GetString()}
                        Gebiet: {properties.GetProperty("areaDesc").GetString()}
                        Schweregrad: {properties.GetProperty("severity").GetString()}
                        Beschreibung: {properties.GetProperty("description").GetString()}
                        Hinweise: {properties.GetProperty("instruction").GetString()}
                        """;
            }));
        }

        [McpServerTool, Description("Wettervorhersage für einen Ort abrufen.")]
        public static async Task<string> GetForecast(
            HttpClient client,
            [Description("Breitengrad des Ortes.")] double latitude,
            [Description("Längengrad des Ortes.")] double longitude)
        {
            var pointUrl = string.Create(CultureInfo.InvariantCulture, $"/points/{latitude},{longitude}");
            using var jsonDocument = await client.ReadJsonDocumentAsync(pointUrl);
            var forecastUrl = jsonDocument.RootElement.GetProperty("properties").GetProperty("forecast").GetString()
                ?? throw new Exception($"Keine Vorhersage-URL bereitgestellt von {client.BaseAddress}points/{latitude},{longitude}");

            using var forecastDocument = await client.ReadJsonDocumentAsync(forecastUrl);
            var periods = forecastDocument.RootElement.GetProperty("properties").GetProperty("periods").EnumerateArray();

            return string.Join("\n---\n", periods.Select(period => $"""
                    {period.GetProperty("name").GetString()}
                    Temperatur: {period.GetProperty("temperature").GetInt32()}°F
                    Wind: {period.GetProperty("windSpeed").GetString()} {period.GetProperty("windDirection").GetString()}
                    Vorhersage: {period.GetProperty("detailedForecast").GetString()}
                    """));
        }
    }
    ```

<div id="whats-happening-under-the-hood">
  ## Was unter der Haube passiert
</div>

Wenn Sie eine Frage stellen:

1. Der Client sendet Ihre Frage an Claude
2. Claude analysiert die verfügbaren Werkzeuge und entscheidet, welche verwendet werden sollen
3. Der Client führt die ausgewählten Werkzeuge über den MCP-Server aus
4. Die Ergebnisse werden an Claude zurückgesendet
5. Claude formuliert eine Antwort in natürlicher Sprache
6. Die Antwort wird Ihnen angezeigt!

  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="Integrationsprobleme mit Claude für Desktop">
    **Protokolle von Claude für Desktop abrufen**

    Protokolle der Claude.app im Zusammenhang mit MCP werden in Protokolldateien unter `~/Library/Logs/Claude` geschrieben:

    * `mcp.log` enthält allgemeine Protokolle zu MCP-Verbindungen und Verbindungsfehlern.
    * Dateien namens `mcp-server-SERVERNAME.log` enthalten Fehlerausgaben (stderr) des jeweiligen Servers.

    Führen Sie den folgenden Befehl aus, um aktuelle Protokolle aufzulisten und neue Einträge live zu verfolgen:

    ```bash
    # Claudes Protokolle auf Fehler prüfen
    tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
    ```

    **Server wird in Claude nicht angezeigt**

    1. Prüfen Sie die Syntax Ihrer Datei `claude_desktop_config.json`
    2. Stellen Sie sicher, dass der Pfad zu Ihrem Projekt absolut ist und nicht relativ
    3. Starten Sie Claude für Desktop vollständig neu

    **Werkzeugaufrufe schlagen stillschweigend fehl**

    Wenn Claude versucht, die Werkzeuge zu verwenden, diese jedoch fehlschlagen:

    1. Prüfen Sie Claudes Protokolle auf Fehler
    2. Stellen Sie sicher, dass Ihr Server fehlerfrei gebaut wird und läuft
    3. Versuchen Sie, Claude für Desktop neu zu starten

    **Nichts davon funktioniert. Was soll ich tun?**

    Bitte sehen Sie in unserem [Debugging-Leitfaden](/de/legacy/tools/debugging) nach. Dort finden Sie bessere Debugging-Werkzeuge und ausführlichere Anleitungen.
  </Accordion>

  <Accordion title="Probleme mit der Weather-API">
    **Fehler: Abrufen der Grid-Point-Daten fehlgeschlagen**

    Das bedeutet in der Regel:

    1. Die Koordinaten liegen außerhalb der USA
    2. Die NWS-API hat derzeit Probleme
    3. Sie unterliegen einer Ratenbegrenzung

    Lösung:

    * Vergewissern Sie sich, dass Sie US-Koordinaten verwenden
    * Fügen Sie eine kurze Verzögerung zwischen Anfragen ein
    * Prüfen Sie die Statusseite der NWS-API

    **Fehler: Keine aktiven Warnungen für [STATE]**

    Das ist kein Fehler – es bedeutet lediglich, dass es derzeit keine Wetterwarnungen für diesen Bundesstaat gibt. Versuchen Sie einen anderen Bundesstaat oder prüfen Sie erneut bei Unwetter.
  </Accordion>
</AccordionGroup>
