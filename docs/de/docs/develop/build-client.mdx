    ## Den Client erstellen

    ### Grundstruktur des Clients

    Zuerst richten wir die Importe ein und erstellen die grundlegende Client-Klasse:

    ```python
    import asyncio
    from typing import Optional
    from contextlib import AsyncExitStack

    from mcp import ClientSession, StdioServerParameters
    from mcp.client.stdio import stdio_client

    from anthropic import Anthropic
    from dotenv import load_dotenv

    load_dotenv()  # Umgebungsvariablen aus .env laden

    class MCPClient:
        def __init__(self):
            # Sitzung und Client-Objekte initialisieren
            self.session: Optional[ClientSession] = None
            self.exit_stack = AsyncExitStack()
            self.anthropic = Anthropic()
        # Methoden werden hier hinzugefügt
    ```

    ### Verwaltung von Server-Verbindungen

    Als Nächstes implementieren wir die Methode zum Herstellen einer Verbindung zu einem MCP-Server:

    ```python
    async def connect_to_server(self, server_script_path: str):
        """Verbindung zu einem MCP-Server herstellen

        Args:
            server_script_path: Pfad zum Serverskript (.py oder .js)
        """
        is_python = server_script_path.endswith('.py')
        is_js = server_script_path.endswith('.js')
        if not (is_python or is_js):
            raise ValueError("Server-Skript muss eine .py- oder .js-Datei sein")

        command = "python" if is_python else "node"
        server_params = StdioServerParameters(
            command=command,
            args=[server_script_path],
            env=None
        )

        stdio_transport = await self.exit_stack.enter_async_context(stdio_client(server_params))
        self.stdio, self.write = stdio_transport
        self.session = await self.exit_stack.enter_async_context(ClientSession(self.stdio, self.write))

        await self.session.initialize()

        # Verfügbare Werkzeuge auflisten
        response = await self.session.list_tools()
        tools = response.tools
        print("\nMit Server verbunden. Verfügbare Werkzeuge:", [tool.name for tool in tools])
    ```

    ### Logik der Abfrageverarbeitung

    Fügen wir nun die Kernfunktionalität zum Verarbeiten von Abfragen und zum Handhaben von Werkzeugaufrufen hinzu:

    ```python
    async def process_query(self, query: str) -> str:
        """Eine Anfrage mit Claude und den verfügbaren Werkzeugen verarbeiten"""
        messages = [
            {
                "role": "user",
                "content": query
            }
        ]

        response = await self.session.list_tools()
        available_tools = [{
            "name": tool.name,
            "description": tool.description,
            "input_schema": tool.inputSchema
        } for tool in response.tools]

        # Erste Anfrage an die Claude-API
        response = self.anthropic.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=1000,
            messages=messages,
            tools=available_tools
        )

        # Antwort verarbeiten und Tool-Aufrufe behandeln
        final_text = []

        assistant_message_content = []
        for content in response.content:
            if content.type == 'text':
                final_text.append(content.text)
                assistant_message_content.append(content)
            elif content.type == 'tool_use':
                tool_name = content.name
                tool_args = content.input

                # Tool-Aufruf ausführen
                result = await self.session.call_tool(tool_name, tool_args)
                final_text.append(f"[Rufe Tool {tool_name} mit Argumenten {tool_args} auf]")

                assistant_message_content.append(content)
                messages.append({
                    "role": "assistant",
                    "content": assistant_message_content
                })
                messages.append({
                    "role": "user",
                    "content": [
                        {
                            "type": "tool_result",
                            "tool_use_id": content.id,
                            "content": result.content
                        }
                    ]
                })

                # Nächste Antwort von Claude abrufen
                response = self.anthropic.messages.create(
                    model="claude-3-5-sonnet-20241022",
                    max_tokens=1000,
                    messages=messages,
                    tools=available_tools
                )

                final_text.append(response.content[0].text)

        return "\n".join(final_text)
    ```

    ### Interaktive Chatoberfläche

    Jetzt fügen wir die Chat-Schleife und die Aufräumlogik hinzu:

    ```python
    async def chat_loop(self):
        """Einen interaktiven Chat-Durchlauf starten"""
        print("\nMCP-Client gestartet!")
        print("Gib deine Eingaben ein oder 'quit' zum Beenden.")

        while True:
            try:
                query = input("\nEingabe: ").strip()

                if query.lower() == 'quit':
                    break

                response = await self.process_query(query)
                print("\n" + response)

            except Exception as e:
                print(f"\nFehler: {str(e)}")

    async def cleanup(self):
        """Ressourcen aufräumen"""
        await self.exit_stack.aclose()
    ```

    ## Erläuterung der Kernkomponenten

    ### 1. Clientinitialisierung

    * Die Klasse `MCPClient` wird mit Sitzungsverwaltung und API-Clients initialisiert
    * Verwendet `AsyncExitStack` für eine korrekte Ressourcenverwaltung
    * Konfiguriert den Anthropic-Client für Interaktionen mit Claude

    ### 2. Serververbindung

    * Unterstützt sowohl Python- als auch Node.js-Server
    * Validiert den Skripttyp des Servers
    * Stellt ordnungsgemäße Kommunikationskanäle her
    * Initialisiert die Sitzung und listet verfügbare Werkzeuge auf

    ### 3. Abfrageverarbeitung

    * Erhält den Gesprächskontext
    * Verarbeitet Claudes Antworten und Werkzeugaufrufe
    * Verwaltet den Nachrichtenfluss zwischen Claude und Werkzeugen
    * Fasst Ergebnisse zu einer kohärenten Antwort zusammen

    ### 4. Interaktive Benutzeroberfläche

    * Stellt eine einfache Befehlszeilenschnittstelle bereit
    * Verarbeitet Benutzereingaben und zeigt Antworten an
    * Beinhaltet grundlegende Fehlerbehandlung
    * Ermöglicht einen sauberen Abschluss

    ### 5. Ressourcenmanagement

    * Ordnungsgemäße Freigabe von Ressourcen
    * Fehlerbehandlung bei Verbindungsstörungen
    * Geordnete Herunterfahrverfahren

    ## Häufige Anpassungspunkte

    1. **Umgang mit Werkzeugen**

    2. **Verarbeitungslogik für Antworten**
       * Anpassen, wie Werkzeug-Ergebnisse formatiert werden

    ## So funktioniert’s

    Wenn Sie eine Anfrage stellen:

    1. Der Client erhält vom Server die Liste der verfügbaren Werkzeuge
    2. Ihre Anfrage wird zusammen mit Werkzeugbeschreibungen an Claude übermittelt
    3. Claude entscheidet, welche Werkzeuge (falls überhaupt) verwendet werden
    4. Der Client führt alle angeforderten Werkzeugaufrufe über den Server aus
    5. Ergebnisse werden an Claude zurückgeschickt
    6. Claude liefert eine Antwort in natürlicher Sprache
    7. Die Antwort wird dir angezeigt

    ## Best Practices

    1. **Fehlerbehandlung**
       * Tool-Aufrufe immer in try-catch-Blöcke einschließen
       * Aussagekräftige Fehlermeldungen liefern
       * Verbindungsprobleme robust und benutzerfreundlich behandeln

    2. **Ressourcen-Management**
       * Verwenden Sie `AsyncExitStack` für eine saubere Aufräumung
       * Verbindungen nach Abschluss schließen
       * Servertrennungen behandeln

    3. **Sicherheit**
       * API-Schlüssel sicher in `.env` speichern
       * Serverantworten prüfen
       * Mit Werkzeugberechtigungen vorsichtig umgehen

    ## Problembehandlung

    ### Probleme mit Server-Pfaden

    * Prüfen Sie, ob der Pfad zu Ihrem Server-Skript korrekt ist
    * Verwenden Sie den absoluten Pfad, wenn der relative Pfad nicht funktioniert.
    * Für Windows-Nutzende: Achten Sie darauf, im Pfad Vorwärtsschrägstriche (/) oder maskierte Backslashes (\) zu verwenden.
    * Verifizieren Sie, dass die Serverdatei die richtige Erweiterung hat (.py für Python oder .js für Node.js)

    Beispiel für die korrekte Verwendung eines Pfads:

    ```bash
    # Relativer Pfad
    uv run client.py ./server/weather.py

    # Absoluter Pfad
    uv run client.py /Users/username/projects/mcp-server/weather.py

    # Windows-Pfad (beide Formate funktionieren)
    uv run client.py C:/projects/mcp-server/weather.py
    uv run client.py C:\\projects\\mcp-server\\weather.py
    ```

    ### Antwort-Timing

    * Die erste Antwort kann bis zu 30 Sekunden benötigen
    * Das ist normal und tritt auf während:
      * Der Server wird gestartet
      * Claude verarbeitet die Anfrage
      * Werkzeuge werden ausgeführt
    * Spätere Antworten sind in der Regel schneller
    * Unterbrechen Sie den Prozess nicht während dieser anfänglichen Wartezeit

    ### Häufige Fehlermeldungen

    Wenn Sie Folgendes sehen:

    * `FileNotFoundError`: Prüfen Sie Ihren Serverpfad
    * `Connection refused`: Stellen Sie sicher, dass der Server läuft und der Pfad korrekt ist
    * `Tool execution failed`: Überprüfen Sie, ob die für das Werkzeug erforderlichen Umgebungsvariablen gesetzt sind
    * `Timeout error`: Erwägen Sie, das Timeout in Ihrer Client-Konfiguration zu erhöhen

    ### Verwaltung der Serververbindung

    Als Nächstes implementieren wir die Methode, um eine Verbindung zu einem MCP-Server herzustellen:

    ```typescript
    async connectToServer(serverScriptPath: string) {
      try {
        const isJs = serverScriptPath.endsWith(".js");
        const isPy = serverScriptPath.endsWith(".py");
        if (!isJs && !isPy) {
          throw new Error("Serverskript muss eine .js- oder .py-Datei sein");
        }
        const command = isPy
          ? process.platform === "win32"
            ? "python"
            : "python3"
          : process.execPath;

        this.transport = new StdioClientTransport({
          command,
          args: [serverScriptPath],
        });
        await this.mcp.connect(this.transport);

        const toolsResult = await this.mcp.listTools();
        this.tools = toolsResult.tools.map((tool) => {
          return {
            name: tool.name,
            description: tool.description,
            input_schema: tool.inputSchema,
          };
        });
        console.log(
          "Mit Server verbunden. Verfügbare Werkzeuge:",
          this.tools.map(({ name }) => name)
        );
      } catch (e) {
        console.log("Fehler beim Verbinden mit dem MCP-Server: ", e);
        throw e;
      }
    }
    ```

    ### Logik der Abfrageverarbeitung

    Fügen wir nun die Kernfunktionalität für die Verarbeitung von Abfragen und die Behandlung von Werkzeugaufrufen hinzu:

    ```typescript
    async processQuery(query: string) {
      const messages: MessageParam[] = [
        {
          role: "user",
          content: query,
        },
      ];

      const response = await this.anthropic.messages.create({
        model: "claude-3-5-sonnet-20241022",
        max_tokens: 1000,
        messages,
        tools: this.tools,
      });

      const finalText = [];

      for (const content of response.content) {
        if (content.type === "text") {
          finalText.push(content.text);
        } else if (content.type === "tool_use") {
          const toolName = content.name;
          const toolArgs = content.input as { [x: string]: unknown } | undefined;

          const result = await this.mcp.callTool({
            name: toolName,
            arguments: toolArgs,
          });
          finalText.push(
            `[Rufe Werkzeug ${toolName} mit Argumenten ${JSON.stringify(toolArgs)} auf]`
          );

          messages.push({
            role: "user",
            content: result.content as string,
          });

          const response = await this.anthropic.messages.create({
            model: "claude-3-5-sonnet-20241022",
            max_tokens: 1000,
            messages,
          });

          finalText.push(
            response.content[0].type === "text" ? response.content[0].text : ""
          );
        }
      }

      return finalText.join("\n");
    }
    ```

    ### Interaktive Chat-Oberfläche

    Nun fügen wir die Chat-Schleife und die Bereinigungsfunktionen hinzu:

    ```typescript
    async chatLoop() {
      const rl = readline.createInterface({
        input: process.stdin,
        output: process.stdout,
      });

      try {
        console.log("\nMCP-Client gestartet!");
        console.log("Geben Sie Ihre Anfrage ein oder „quit“ zum Beenden.");

        while (true) {
          const message = await rl.question("\nEingabe: ");
          if (message.toLowerCase() === "quit") {
            break;
          }
          const response = await this.processQuery(message);
          console.log("\n" + response);
        }
      } finally {
        rl.close();
      }
    }

    async cleanup() {
      await this.mcp.close();
    }
    ```

    ### Verwaltung der Serververbindung

    Als Nächstes implementieren wir die Methode, um eine Verbindung zu einem MCP-Server herzustellen:

    ```kotlin
    suspend fun connectToServer(serverScriptPath: String) {
        try {
            val command = buildList {
                when (serverScriptPath.substringAfterLast(".")) {
                    "js" -> add("node")
                    "py" -> add(if (System.getProperty("os.name").lowercase().contains("win")) "python" else "python3")
                    "jar" -> addAll(listOf("java", "-jar"))
                    else -> throw IllegalArgumentException("Serverskript muss eine .js-, .py- oder .jar-Datei sein")
                }
                add(serverScriptPath)
            }

            val process = ProcessBuilder(command).start()
            val transport = StdioClientTransport(
                input = process.inputStream.asSource().buffered(),
                output = process.outputStream.asSink().buffered()
            )

            mcp.connect(transport)

            val toolsResult = mcp.listTools()
            tools = toolsResult?.tools?.map { tool ->
                ToolUnion.ofTool(
                    Tool.builder()
                        .name(tool.name)
                        .description(tool.description ?: "")
                        .inputSchema(
                            Tool.InputSchema.builder()
                                .type(JsonValue.from(tool.inputSchema.type))
                                .properties(tool.inputSchema.properties.toJsonValue())
                                .putAdditionalProperty("required", JsonValue.from(tool.inputSchema.required))
                                .build()
                        )
                        .build()
                )
            } ?: emptyList()
            println("Mit Server verbunden; Werkzeuge: ${tools.joinToString(", ") { it.tool().get().name() }}")
        } catch (e: Exception) {
            println("Verbindung zum MCP-Server fehlgeschlagen: $e")
            throw e
        }
    }
    ```

    Erstellen Sie außerdem eine Hilfsfunktion, die für Anthropic ein `JsonObject` in ein `JsonValue` konvertiert:

    ```kotlin
    private fun JsonObject.toJsonValue(): JsonValue {
        val mapper = ObjectMapper()
        val node = mapper.readTree(this.toString())
        return JsonValue.fromJsonNode(node)
    }
    ```

    ### Logik der Anfrageverarbeitung

    Fügen wir nun die Kernfunktionalität für die Verarbeitung von Abfragen und das Handling von Tool-Aufrufen hinzu:

    ```kotlin
    private val messageParamsBuilder: MessageCreateParams.Builder = MessageCreateParams.builder()
        .model(Model.CLAUDE_3_5_SONNET_20241022)
        .maxTokens(1024)

    suspend fun processQuery(query: String): String {
        val messages = mutableListOf(
            MessageParam.builder()
                .role(MessageParam.Role.USER)
                .content(query)
                .build()
        )

        val response = anthropic.messages().create(
            messageParamsBuilder
                .messages(messages)
                .tools(tools)
                .build()
        )

        val finalText = mutableListOf<String>()
        response.content().forEach { content ->
            when {
                content.isText() -> finalText.add(content.text().getOrNull()?.text() ?: "")

                content.isToolUse() -> {
                    val toolName = content.toolUse().get().name()
                    val toolArgs =
                        content.toolUse().get()._input().convert(object : TypeReference<Map<String, JsonValue>>() {})

                    val result = mcp.callTool(
                        name = toolName,
                        arguments = toolArgs ?: emptyMap()
                    )
                    finalText.add("[Werkzeug $toolName wird mit Argumenten $toolArgs aufgerufen]")

                    messages.add(
                        MessageParam.builder()
                            .role(MessageParam.Role.USER)
                            .content(
                                """
                                    "type": "tool_result",
                                    "tool_name": $toolName,
                                    "result": ${result?.content?.joinToString("\n") { (it as TextContent).text ?: "" }}
                                """.trimIndent()
                            )
                            .build()
                    )

                    val aiResponse = anthropic.messages().create(
                        messageParamsBuilder
                            .messages(messages)
                            .build()
                    )

                    finalText.add(aiResponse.content().first().text().getOrNull()?.text() ?: "")
                }
            }
        }

        return finalText.joinToString("\n", prefix = "", postfix = "")
    }
    ```

    ## Funktionsweise

    Hier ist ein grobes Workflow-Schema:

    ```mermaid
    ---
    config:
        theme: neutral
    ---
    sequenceDiagram
        actor User
        participant Client
        participant Claude
        participant MCP_Server as MCP Server
        participant Tools

        User->>Client: Anfrage senden
        Client<<->>MCP_Server: Verfügbare Werkzeuge abrufen
        Client->>Claude: Anfrage mit Werkzeugbeschreibungen senden
        Claude-->>Client: Entscheidung zur Werkzeugausführung
        Client->>MCP_Server: Ausführung des Werkzeugs anfordern
        MCP_Server->>Tools: Ausgewählte Werkzeuge ausführen
        Tools-->>MCP_Server: Ergebnisse zurückgeben
        MCP_Server-->>Client: Ergebnisse senden
        Client->>Claude: Werkzeugergebnisse senden
        Claude-->>Client: Finale Antwort bereitstellen
        Client-->>User: Antwort anzeigen
    ```

    Wenn Sie eine Abfrage senden:

    1. Der Client ruft die Liste der verfügbaren Werkzeuge vom Server ab
    2. Ihre Anfrage wird zusammen mit Beschreibungen der Werkzeuge an Claude gesendet
    3. Claude entscheidet, welche Werkzeuge (falls überhaupt) verwendet werden
    4. Der Client führt alle angeforderten Werkzeuge über den Server aus
    5. Ergebnisse werden an Claude zurückgesendet.
    6. Claude gibt eine Antwort in natürlicher Sprache
    7. Die Antwort wird Ihnen angezeigt

    ```csharp
    });

    await using var mcpClient = await McpClientFactory.CreateAsync(clientTransport);

    var tools = await mcpClient.ListToolsAsync();
    foreach (var tool in tools)
    {
        Console.WriteLine($"Mit Server verbunden, verfügbare Werkzeuge: {tool.Name}")
    }
    ```

    Fügen Sie die folgende Funktion am Ende der Datei `Program.cs` hinzu:

    ```csharp
    static (string command, string[] arguments) GetCommandAndArguments(string[] args)
    {
        return args switch
        {
            [var script] when script.EndsWith(".py") => ("python", args),
            [var script] when script.EndsWith(".js") => ("node", args),
            [var script] when Directory.Exists(script) || (File.Exists(script) && script.EndsWith(".csproj")) => ("dotnet", ["run", "--project", script, "--no-build"]),
            _ => throw new NotSupportedException("Ein nicht unterstütztes Server-Skript wurde angegeben. Unterstützte Skripte sind .py, .js oder .csproj")
        };
    }
    ```

    Dies erzeugt einen MCP-Client, der sich mit einem Server verbindet, der als Befehlszeilenargument angegeben wird. Anschließend listet er die verfügbaren Werkzeuge des verbundenen Servers auf.

    ### Logik der Anfrageverarbeitung

    Fügen wir nun die Kernfunktionalität zum Verarbeiten von Abfragen und zum Ausführen von Werkzeugaufrufen hinzu:

    ```csharp
    using var anthropicClient = new AnthropicClient(new APIAuthentication(builder.Configuration["ANTHROPIC_API_KEY"]))
        .Messages
        .AsBuilder()
        .UseFunctionInvocation()
        .Build();

    var options = new ChatOptions
    {
        MaxOutputTokens = 1000,
        ModelId = "claude-3-5-sonnet-20241022",
        Tools = [.. tools]
    };

    Console.ForegroundColor = ConsoleColor.Green;
    Console.WriteLine("MCP-Client gestartet!");
    Console.ResetColor();

    PromptForInput();
    while(Console.ReadLine() is string query && !"exit".Equals(query, StringComparison.OrdinalIgnoreCase))
    {
        if (string.IsNullOrWhiteSpace(query))
        {
            PromptForInput();
            continue;
        }

        await foreach (var message in anthropicClient.GetStreamingResponseAsync(query, options))
        {
            Console.Write(message);
        }
        Console.WriteLine();

        PromptForInput();
    }

    static void PromptForInput()
    {
        Console.WriteLine("Geben Sie einen Befehl ein (oder 'exit' zum Beenden):");
        Console.ForegroundColor = ConsoleColor.Cyan;
        Console.Write("> ");
        Console.ResetColor();
    }
    ```

    ## Wichtige Komponenten im Überblick

    ### 1. Client-Initialisierung

    * Der Client wird mit `McpClientFactory.CreateAsync()` initialisiert; dabei werden der Transporttyp und der auszuführende Befehl zum Starten des Servers festgelegt.

    ### 2. Server-Verbindung

    * Unterstützt Python-, Node.js- und .NET-Server.
    * Der Server wird mit dem in den Argumenten angegebenen Befehl gestartet.
    * Konfiguriert die Verwendung von STDIO für die Kommunikation mit dem Server.
    * Initialisiert die Sitzung und die verfügbaren Werkzeuge.

    ### 3. Anfrageverarbeitung

    * Verwendet [Microsoft.Extensions.AI](https://learn.microsoft.com/dotnet/ai/ai-extensions) für den Chat-Client.
    * Konfiguriert den `IChatClient` für die automatische Ausführung von Werkzeugen (Funktionen).
    * Der Client liest die Eingabe des Benutzers und sendet sie an den Server.
    * Der Server verarbeitet die Anfrage und gibt eine Antwort zurück.
    * Die Antwort wird dem Nutzer angezeigt.

    ## Den Client ausführen

    So starten Sie Ihren Client mit einem beliebigen MCP-Server:

    ```bash
    dotnet run -- path/to/server.csproj # .NET-Server
    dotnet run -- path/to/server.py # Python-Server
    dotnet run -- path/to/server.js # Node.js-Server
    ```

    <Note>
      Wenn Sie das Wetter-Tutorial aus dem Server-Quickstart fortsetzen, könnte Ihr Befehl etwa so aussehen: `dotnet run -- path/to/QuickstartWeatherServer`.
    </Note>

    Der Client wird:

    1. Mit dem angegebenen Server verbinden
    2. Verfügbare Werkzeuge anzeigen
    3. Starten Sie eine interaktive Chat-Sitzung, in der Sie Folgendes tun können:
       * Anfragen eingeben
       * Werkzeugausführungen ansehen
       * Antworten von Claude erhalten
    4. Beenden Sie die Sitzung, wenn Sie fertig sind.
