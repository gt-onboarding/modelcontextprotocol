title: 仕様
---

[Model Context Protocol](https://modelcontextprotocol.io)（MCP）は、LLMアプリケーションと外部のデータソースやツールをシームレスに統合するためのオープンなプロトコルです。AI対応のIDEを構築する場合、チャットインターフェースを強化する場合、カスタムのAIワークフローを作成する場合など、MCPはLLMを必要なコンテキストへ接続するための標準化された方法を提供します。

本仕様は、TypeScriptのスキーマ（
[schema.ts](https://github.com/modelcontextprotocol/specification/blob/main/schema/2025-03-26/schema.ts)
）に基づく正準的なプロトコル要件を定義します。

実装ガイドやサンプルについては、
[modelcontextprotocol.io](https://modelcontextprotocol.io) を参照してください。

本ドキュメントにおける「MUST」「MUST NOT」「REQUIRED」「SHALL」「SHALL NOT」「SHOULD」「SHOULD NOT」「RECOMMENDED」「NOT RECOMMENDED」「MAY」「OPTIONAL」というキーワードは、[BCP 14](https://datatracker.ietf.org/doc/html/bcp14)
[[RFC2119](https://datatracker.ietf.org/doc/html/rfc2119)]
[[RFC8174](https://datatracker.ietf.org/doc/html/rfc8174)] に記載されているとおりに解釈されます。ただし、その場合は、ここで示すようにすべて大文字で表記されている場合に限ります。

<div id="overview">
  ## 概要
</div>

MCP はアプリケーションが次のことを行うための標準化された方法を提供します:

* 言語モデルとコンテキスト情報を共有する
* AI システムにツールや機能を公開する
* 組み合わせ可能な連携やワークフローを構築する

このプロトコルは [JSON-RPC](https://www.jsonrpc.org/) 2.0 メッセージを用いて、次の間で
通信を確立します:

* **ホスト**: 接続を開始する LLM アプリケーション
* **クライアント**: ホストアプリケーション内のコネクタ
* **サーバー**: コンテキストと機能を提供するサービス

MCP は
[Language Server Protocol](https://microsoft.github.io/language-server-protocol/) から着想を得ており、これは
エコシステム全体の開発ツールにわたってプログラミング言語のサポートを追加する方法を標準化しています。同様に、MCP は AI アプリケーションのエコシステムに追加のコンテキストや
ツールを統合する方法を標準化します。

<div id="key-details">
  ## 重要なポイント
</div>

<div id="base-protocol">
  ### ベースプロトコル
</div>

* [JSON-RPC](https://www.jsonrpc.org/) のメッセージフォーマット
* ステートフルな接続
* サーバーとクライアントによる機能交渉

<div id="features">
  ### 機能
</div>

サーバーはクライアントに対して次の機能のいずれか（またはいくつか）を提供します:

* **リソース**: ユーザーまたはAIモデルが利用できるコンテキストやデータ
* **プロンプト**: ユーザー向けのテンプレート化されたメッセージやワークフロー
* **ツール**: AIモデルが実行するための機能

クライアントはサーバーに対して次の機能を提供する場合があります:

* **サンプリング**: サーバー起点のエージェント的な振る舞いと再帰的なLLMとのやり取り

<div id="additional-utilities">
  ### 追加ユーティリティ
</div>

* 設定
* 進捗の追跡
* キャンセル
* エラー報告
* ログ記録

<div id="security-and-trust-safety">
  ## セキュリティとトラスト&セーフティ
</div>

Model Context Protocol（MCP）は、任意のデータへのアクセスやコード実行経路を通じて強力な機能を提供します。その一方で、重要なセキュリティおよびトラストに関する考慮事項が伴うため、すべての実装者はこれらに細心の注意を払って対処する必要があります。

<div id="key-principles">
  ### 重要な原則
</div>

1. **ユーザーの同意とコントロール**
   * すべてのデータアクセスと操作について、ユーザーは明示的に同意し、内容を理解していなければならない
   * どのデータを共有し、どのアクションを実行するかについて、ユーザーがコントロールを保持しなければならない
   * 実装者は、アクティビティの確認と承認のために明確なUIを提供すべきである

2. **データプライバシー**
   * ホストは、ユーザーデータをサーバーに開示する前に、明示的なユーザー同意を取得しなければならない
   * ホストは、ユーザーの同意なしにリソースデータを他所へ送信してはならない
   * ユーザーデータは、適切なアクセス制御で保護しなければならない

3. **ツールの安全性**
   * ツールは任意のコード実行を伴う可能性があり、相応の注意をもって扱わなければならない。
     * とりわけ、アノテーションなどのツール挙動の説明は、信頼できるサーバーから取得された場合を除き、信頼できないものとして扱うべきである。
   * ホストは、いかなるツールを呼び出す前にも、明示的なユーザー同意を取得しなければならない
   * ユーザーは、その使用を承認する前に、各ツールの機能と挙動を理解しておくべきである

4. **LLMサンプリングの制御**
   * すべてのLLMサンプリング要求は、ユーザーが明示的に承認しなければならない
   * ユーザーは次の点をコントロールすべきである:
     * そもそもサンプリングを行うかどうか
     * 送信される実際のプロンプト
     * サーバーが閲覧できる結果
   * このプロトコルは意図的に、プロンプトに対するサーバーの可視性を制限している

<div id="implementation-guidelines">
  ### 実装ガイドライン
</div>

MCP 自体はプロトコルレベルでこれらのセキュリティ原則を強制できないものの、
実装者は**従うべきです（SHOULD）**:

1. アプリケーションに堅牢な同意および認可フローを組み込むこと
2. セキュリティ上の影響を明確にドキュメント化すること
3. 適切なアクセス制御とデータ保護を実装すること
4. 統合においてセキュリティのベストプラクティスに従うこと
5. 機能設計においてプライバシーへの影響を考慮すること
