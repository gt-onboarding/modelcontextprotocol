<div id="enable-section-numbers" />

[Model Context Protocol](https://modelcontextprotocol.io)（MCP）は、LLMアプリケーションと外部のデータソースおよび
ツールをシームレスに統合するためのオープンプロトコルです。AI対応のIDEを構築する場合、チャットインターフェースを強化する場合、あるいは
カスタムAIワークフローを作成する場合でも、MCPはLLMを必要なコンテキストに接続するための標準化された手段を提供します。

この仕様は、TypeScriptのスキーマ
[schema.ts](https://github.com/modelcontextprotocol/specification/blob/main/schema/2025-06-18/schema.ts)
に基づく権威あるプロトコル要件を定義します。

実装ガイドやサンプルについては、
[modelcontextprotocol.io](https://modelcontextprotocol.io) を参照してください。

本ドキュメントにおける「MUST」「MUST NOT」「REQUIRED」「SHALL」「SHALL NOT」「SHOULD」、
「SHOULD NOT」「RECOMMENDED」「NOT RECOMMENDED」「MAY」「OPTIONAL」というキーワードは、
[BCP 14](https://datatracker.ietf.org/doc/html/bcp14)
[[RFC2119](https://datatracker.ietf.org/doc/html/rfc2119)]
[[RFC8174](https://datatracker.ietf.org/doc/html/rfc8174)]に記載されているとおり、
ここに示すようにすべて大文字で表記されている場合に限り、その意味で解釈されます。

<div id="overview">
  ## 概要
</div>

MCPは、アプリケーションが次のことを行える標準化された方法を提供します：

* 言語モデルと文脈情報を共有する
* AIシステムにツールや機能を公開する
* 組み合わせ可能な統合やワークフローを構築する

このプロトコルは、[JSON-RPC](https://www.jsonrpc.org/) 2.0 メッセージを用いて、次の間で通信を確立します：

* **ホスト**：接続を開始するLLMアプリケーション
* **クライアント**：ホストアプリケーション内のコネクタ
* **サーバー**：文脈と機能を提供するサービス

MCPは
[Language Server Protocol](https://microsoft.github.io/language-server-protocol/) からも着想を得ており、これはエコシステム全体の開発ツールにわたってプログラミング言語のサポートを追加する方法を標準化しています。これと同様に、MCPはAIアプリケーションのエコシステムに追加の文脈やツールを統合する方法を標準化します。

<div id="key-details">
  ## 主なポイント
</div>

<div id="base-protocol">
  ### ベースプロトコル
</div>

* [JSON-RPC](https://www.jsonrpc.org/) メッセージ形式
* 状態管理ありの接続
* サーバーとクライアント間の機能交渉

<div id="features">
  ### 機能
</div>

サーバーはクライアントに対して次の機能を提供できます：

* **リソース**: ユーザーまたはAIモデルが利用するためのコンテキストやデータ
* **プロンプト**: ユーザー向けのテンプレート化されたメッセージやワークフロー
* **ツール**: AIモデルが実行するための関数

クライアントはサーバーに対して次の機能を提供できる場合があります：

* **サンプリング**: サーバー主導のエージェント的な挙動と再帰的なLLMとのやり取り
* **ルーツ**: サーバーが処理を行う範囲としてのURIまたはファイルシステムの境界に関する問い合わせ
* **エリシテーション**: ユーザーに追加情報を求めるサーバー主導のリクエスト

<div id="additional-utilities">
  ### 追加ユーティリティ
</div>

* 設定
* 進捗の追跡
* キャンセル
* エラー報告
* ログ記録

<div id="security-and-trust-safety">
  ## セキュリティとトラスト＆セーフティ
</div>

Model Context Protocol（MCP）は、任意のデータアクセスやコード実行経路を通じて強力な機能を提供します。この強力さには重要なセキュリティおよびトラストに関する考慮事項が伴うため、実装者はすべて、細心の注意を払って対処する必要があります。

<div id="key-principles">
  ### 重要な原則
</div>

1. **ユーザーの同意とコントロール**
   * すべてのデータアクセスと操作について、ユーザーは明示的に同意し、内容を理解していなければならない
   * 共有するデータと実行されるアクションについてのコントロールは、ユーザーが保持し続けなければならない
   * 実装者は、活動の確認と承認のための明確なUIを提供すべきである

2. **データプライバシー**
   * ホストは、ユーザーデータをサーバーに開示する前に、明示的なユーザーの同意を得なければならない
   * ホストは、ユーザーの同意なしにリソースデータを他所へ送信してはならない
   * ユーザーデータは、適切なアクセス制御によって保護されるべきである

3. **ツールの安全性**
   * ツールは任意コードの実行を伴い、適切な注意をもって扱わなければならない
     * 特に、アノテーションなどのツール動作の記述は、信頼できるサーバーから取得された場合を除き、信頼せずに扱うべきである
   * ホストは、いかなるツールを呼び出す前にも、明示的なユーザーの同意を得なければならない
   * ユーザーは、その使用を承認する前に、各ツールの機能と挙動を理解しておくべきである

4. **LLMサンプリングの制御**
   * すべてのLLMサンプリング要求は、ユーザーが明示的に承認しなければならない
   * ユーザーは次の点をコントロールすべきである:
     * サンプリングを行うかどうか
     * 送信される実際のプロンプト
     * サーバーが参照できる結果
   * 本プロトコルは、意図的にサーバーからプロンプトの可視性を制限している

<div id="implementation-guidelines">
  ### 実装ガイドライン
</div>

Model Context Protocol（MCP）自体はプロトコルレベルでこれらのセキュリティ原則を強制できないため、
実装者は次の事項を遵守することが**望ましい**とされます。

1. アプリケーションに堅牢な同意および認可フローを組み込む
2. セキュリティ上の影響を明確にドキュメント化する
3. 適切なアクセス制御とデータ保護を実装する
4. 連携においてセキュリティのベストプラクティスに従う
5. 機能設計においてプライバシーへの影響を考慮する
