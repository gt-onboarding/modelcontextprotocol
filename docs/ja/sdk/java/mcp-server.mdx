<div id="server-features">
  ## サーバー機能
</div>

MCPサーバーは、Model Context Protocol（MCP）アーキテクチャにおける基盤コンポーネントで、クライアントにツール、リソース、各種機能を提供します。プロトコルのサーバー側を実装し、以下を担います：

* クライアントが探索・実行できるツールの公開。
  入出力スキーマをサポートし、構造化・非構造化のコンテンツタイプを返します。
* URIベースのアクセスパターンによるリソース管理
* プロンプトテンプレートの提供とプロンプト要求の処理
* クライアントとの機能交渉のサポート
* サーバー側のプロトコル操作の実装
* 複数クライアントからの同時接続の管理
* 構造化ロギング、進捗トラッキング、通知の提供
* 外部のWebフレームワークを必要とせず、`STDIO`、`Streamable-HTTP`、`SSE` のサーバートランスポート実装を提供
* （任意）[Spring AI](https://docs.spring.io/spring-ai/reference/api/mcp/mcp-server-boot-starter-docs.html) ユーザー向けの、Spring専用トランスポート依存関係 `io.modelcontextprotocol.sdk:mcp-spring-webflux`、`io.modelcontextprotocol.sdk:mcp-spring-webmvc`

<Tip>
  Spring AI MCP に基づくこの[クイックスタートデモ](/ja/quickstart/server)では、
  MCPサーバーの構築方法を紹介します。
</Tip>

サーバーは同期・非同期の両APIをサポートし、さまざまなアプリケーション文脈で柔軟に統合できます。

<Tabs>
  <Tab title="Sync API">
    ```java
    // Create a server with custom configuration
    McpSyncServer syncServer = McpServer.sync(transportProvider)
        .serverInfo("my-server", "1.0.0")
        .capabilities(ServerCapabilities.builder()
            .resources(false, true)  // Enable resource support
            .tools(true)             // Enable tool support
            .prompts(true)           // Enable prompt support
            .logging()               // Enable logging support
            .completions()           // Enable completions support
            .build())
        .build();

    // Register tools, resources, and prompts
    syncServer.addTool(syncToolSpecification);
    syncServer.addResource(syncResourceSpecification);
    syncServer.addPrompt(syncPromptSpecification);

    // Close the server when done
    syncServer.close();
    ```

    ### ハンドラーでの thread-local の保持

    `McpSyncServer` は、実行を基盤の `McpAsyncServer` に委譲します。
    ツールやリソースなどのハンドラーの実行は、メソッドを呼び出したスレッド上で行われない場合があります。
    その場合、thread-local は失われ、スレッドに束縛された動作に依存するフレームワークの機能が壊れる可能性があります。
    呼び出し元スレッドで実行し、thread-local を利用可能なままに保つには、`McpServer.sync(...).immediateExecution(true)` を設定してください。

    <Note>
      これは同期サーバーにのみ関係します。非同期サーバーはリアクティブスタックを使用し、thread-local に依存すべきではありません。
    </Note>
  </Tab>

  <Tab title="Async API">
    ```java
    // Create an async server with custom configuration
    McpAsyncServer asyncServer = McpServer.async(transportProvider)
        .serverInfo("my-server", "1.0.0")
        .capabilities(ServerCapabilities.builder()
            .resources(false, true)     // Enable resource support
            .tools(true)                // Enable tool support
            .prompts(true)              // Enable prompt support
            .logging()                  // Enable logging support
            .completions()              // Enable completions support
            .build())
        .build();

    // Register tools, resources, and prompts
    asyncServer.addTool(asyncToolSpecification)
        .doOnSuccess(v -> logger.info("Tool registered"))
        .subscribe();

    asyncServer.addResource(asyncResourceSpecification)
        .doOnSuccess(v -> logger.info("Resource registered"))
        .subscribe();

    asyncServer.addPrompt(asyncPromptSpecification)
        .doOnSuccess(v -> logger.info("Prompt registered"))
        .subscribe();

    // Close the server when done
    asyncServer.close()
        .doOnSuccess(v -> logger.info("Server closed"))
        .subscribe();
    ```
  </Tab>
</Tabs>

<div id="server-transport-providers">
  ## サーバー向けトランスポートプロバイダー
</div>

MCP SDKのトランスポート層は、クライアントとサーバー間の通信を担います。
さまざまな通信プロトコルやパターンをサポートするための実装を提供します。
SDKには、いくつかの組み込みのトランスポートプロバイダー実装が含まれています。

<Tabs>
  <Tab title="STDIO">
    プロセス内ベースのトランスポートを作成:

    ```java
    StdioServerTransportProvider transportProvider = new StdioServerTransportProvider(new ObjectMapper());
    ```

    標準入力/出力ストリーム上で双方向のJSON-RPCメッセージを扱い、ノンブロッキング処理、シリアル化/デシリアル化、および安全なシャットダウンをサポートします。

    主な特長:

    <ul>
      <li>stdin/stdout を用いた双方向通信</li>
      <li>プロセスベースの統合に対応</li>
      <li>簡単なセットアップと設定</li>
      <li>軽量な実装</li>
    </ul>
  </Tab>

  <Tab title="WebFlux">
    <p>WebFlux-based SSE and Streamable-HTTP server transport. [Requires the `mcp-spring-webflux` dependency](/ja/sdk/java/mcp-overview#dependencies).</p>

    <Tabs>
      <Tab title="Streamable-HTTP (WebFlux)">
        ```java
        @Configuration
        class McpConfig {
            @Bean
            WebFluxStreamableServerTransportProvider webFluxStreamableServerTransportProvider(ObjectMapper mapper) {
              	return this.mcpStreamableServerTransportProvider = WebFluxStreamableServerTransportProvider.builder()
                        .objectMapper(mapper)
                        .messageEndpoint("/mcp")
                        .build();
            }

            @Bean
            RouterFunction<?> mcpRouterFunction(WebFluxStreamableServerTransportProvider transportProvider) {
                return transportProvider.getRouterFunction();
            }
        }
        ```

        <p>ストリーム対応HTTPのトランスポート仕様を実装し、以下を提供します:</p>

        <ul>
          <li>ストリーム対応HTTPによる双方向通信</li>
          <li>WebFluxによるリアクティブHTTPストリーミング</li>
          <li>ストリーム対応HTTPエンドポイントによる同時クライアント接続</li>
          <li>メッセージルーティングとセッション管理</li>
          <li>グレースフルシャットダウン機能</li>
        </ul>

        <Note>
          現在の実装はセッションストレージがないため、再開（レジューム）には未対応です。
        </Note>

        <Tip>
          複数のMCPサーバーインスタンスを運用する分散環境では、適切なメッセージルーティングが必要です。
          大きなインフラ負荷をかけずにスケールさせるには、`Stateless Streamable-HTTP` サーバートランスポート実装の利用を検討してください。
        </Tip>
      </Tab>

      <Tab title="Stateless Streamable-HTTP (WebFlux)">
        ステートレスなMCPサーバーは、リクエスト間でセッション状態を保持しないシンプルなデプロイ向けに設計されています。
        `Streamable-HTTP` 仕様のサブセットを実装し、`application/json` のレスポンスを返します。
        標準のストリーム対応HTTPトランスポートと異なり、リクエスト間でセッション状態を保持しないためステートレスと呼ばれます。
        これらのサーバーはマイクロサービスアーキテクチャやクラウドネイティブなデプロイに最適です。

        ```java

            @Bean
            WebFluxStatelessServerTransport webFluxStatelessServerTransport(ObjectMapper mapper) {
              	return this.mcpStatelessServerTransport = WebFluxStatelessServerTransport.builder()
                        .objectMapper(mapper)
                        .messageEndpoint("/mcp")
                        .build();
            }

            @Bean
            RouterFunction<?> mcpRouterFunction(WebFluxStatelessServerTransport transportProvider) {
                return transportProvider.getRouterFunction();
            }

            @Bean
            public McpStatelessSyncServer prepareSyncServerBuilder(WebFluxStatelessServerTransport statelessServerTransport) {
              return McpServer.sync(statelessServerTransport)
                      //...
                      .build();
            }

        }
        ```

        <p>MCPのステートレスなストリーム対応HTTPトランスポート仕様を実装し、以下を提供します:</p>

        <ul>
          <li>ストリーム対応HTTPによる単方向（クライアント→サーバー）通信</li>
          <li>WebFluxによるリアクティブHTTPストリーミング</li>
          <li>ステートレスなエンドポイントによる同時クライアント接続</li>
          <li>メッセージルーティングとセッション管理</li>
          <li>グレースフルシャットダウン機能</li>

        <Note>
          現在の実装はクライアントへの通知送信をサポートしていません。

<div id="server-capabilities">
  ## サーバーの機能
</div>

サーバーでは、さまざまな機能を設定できます:

```java
var capabilities = ServerCapabilities.builder()
    .resources(false, true)  // リソース対応（リスト変更の通知あり）
    .tools(true)             // ツール対応（リスト変更の通知あり）
    .prompts(true)           // プロンプト対応（リスト変更の通知あり）
    .logging()               // ログ出力を有効化（デフォルトで有効、ログレベルは INFO）
    .build();
```

<div id="tool-specification">
  ### ツール仕様
</div>

Model Context Protocol（MCP）は、サーバーが言語モデルから呼び出せる[ツールを公開](/ja/specification/2024-11-05/server/tools/)できるようにします。
Java SDK では、ハンドラー関数付きのツール仕様を実装できます。
ツールにより、AIモデルは計算、外部APIへのアクセス、データベースのクエリ、ファイル操作などを実行できます:

<Tabs>
  <Tab title="Sync">
    ```java
    // Sync tool specification
    var schema = """
                {
                  "type" : "object",
                  "id" : "urn:jsonschema:Operation",
                  "properties" : {
                    "operation" : {
                      "type" : "string"
                    },
                    "a" : {
                      "type" : "number"
                    },
                    "b" : {
                      "type" : "number"
                    }
                  }
                }
                """;
    var syncToolSpecification = new McpServerFeatures.SyncToolSpecification(
        new Tool("calculator", "Basic calculator", schema),
        (exchange, arguments) -> {
            // Tool implementation
            return new CallToolResult(result, false);

    );
    ```
  </Tab>

  <Tab title="Async">

    ```java
    // Async tool specification

    ```
  </Tab>
</Tabs>

ツール仕様には、`name`、`description`、`parameter schema` を含むツール定義と、そのロジックを実装する呼び出しハンドラーが含まれます。
関数の第1引数はクライアントとのやり取り用の `McpAsyncServerExchange`、第2引数はツール引数のマップです。

リソース仕様は、リソース定義とリソース読み取りハンドラーで構成されます。

  </Tab>

  <Tab title="Async">

  </Tab>

プロンプト定義には、name（プロンプトの識別子）、description（プロンプトの目的）、および引数のリスト（テンプレート用のパラメータ）が含まれます。
ハンドラー関数はリクエストを処理し、整形済みテンプレートを返します。

<div id="completion-specification">
  ### 補完仕様
</div>

[Completion capabilities](/ja/specification/2025-03-26/server/utilities/completion) の一部として、MCP はサーバーがプロンプトやリソースURIの引数に対するオートコンプリート候補を提供できるよう、標準化された方法を提供します。

<Tabs>
  <Tab title="Sync">
    ```java
    // 同期補完仕様
    var syncCompletionSpecification = new McpServerFeatures.SyncCompletionSpecification(
    			new McpSchema.PromptReference("code_review"), (exchange, request) -> {

            // 補完の実装 ...

            return new McpSchema.CompleteResult(
                new CompleteResult.CompleteCompletion(
                  List.of("python", "pytorch", "pyside"),
                  10, // total
                  false // hasMore
                ));
          }
    );

    // 補完機能を備えた同期サーバーを作成
    var mcpServer = McpServer.sync(mcpServerTransportProvider)
      .capabilities(ServerCapabilities.builder()
        .completions() // 補完サポートを有効化
          // ...
        .build())
      // ...
      .completions(new McpServerFeatures.SyncCompletionSpecification( // 補完仕様を登録
          new McpSchema.PromptReference("code_review"), syncCompletionSpecification))
      .build();

    ```
  </Tab>

  <Tab title="Async">
    ```java
    // 非同期補完仕様

    			new McpSchema.PromptReference("code_review"), (exchange, request) -> {

            // 補完の実装 ...
    ```

  </Tab>
</Tabs>

`McpSchema.CompletionReference` の定義では、補完仕様の種類（`PromptReference` または `ResourceReference`）と、その識別子（例: ハンドラー）を指定します。
ハンドラー関数はリクエストを処理し、補完レスポンスを返します。
第1引数はクライアントとのやり取り用の `McpAsyncServerExchange`、第2引数は `CompleteRequest` インスタンスです。

クライアント側での補完機能の使い方は、[using completion](/ja/sdk/java/mcp-client#using-completion) を参照してください。

<div id="using-sampling-from-a-server">
  ### サーバーからサンプリングを利用する
</div>

[サンプリング機能](/ja/specification/2024-11-05/client/sampling/)を使うには、サンプリングをサポートする互換クライアントが必要です。
サーバー側で特別な設定は不要ですが、リクエストする前にクライアントがサンプリングに対応しているか確認してください。
[クライアントのサンプリング対応](ja/./mcp-client#sampling-support)について学ぶ。

互換クライアントがステートフルなサーバーに接続すると、サーバーは言語モデルによる生成を要求できます:

<Tabs>
  <Tab title="Sync API">
    ```java
    // Create a server
    McpSyncServer server = McpServer.sync(transportProvider)
        .serverInfo("my-server", "1.0.0")
        .build();

    // Define a tool that uses sampling
    var calculatorTool = new McpServerFeatures.SyncToolSpecification(
        new Tool("ai-calculator", "Performs calculations using AI", schema),
        (exchange, arguments) -> {
            // Check if client supports sampling
            if (exchange.getClientCapabilities().sampling() == null) {
                return new CallToolResult("Client does not support AI capabilities", false);
            }

            // Create a sampling request
            McpSchema.CreateMessageRequest request = McpSchema.CreateMessageRequest.builder()
                .messages(List.of(new McpSchema.SamplingMessage(McpSchema.Role.USER,
                    new McpSchema.TextContent("Calculate: " + arguments.get("expression")))
                .modelPreferences(McpSchema.ModelPreferences.builder()
                    .hints(List.of(
                        McpSchema.ModelHint.of("claude-3-sonnet"),
                        McpSchema.ModelHint.of("claude")
                    ))
                    .intelligencePriority(0.8)  // 知能重視の度合い
                    .speedPriority(0.5)         // 速度重視の度合い（中程度）
                    .build())
                .systemPrompt("You are a helpful calculator assistant. Provide only the numerical answer.")
                .maxTokens(100)
                .build();

            // Request sampling from the client
            McpSchema.CreateMessageResult result = exchange.createMessage(request);

            // Process the result
            String answer = result.content().text();
            return new CallToolResult(answer, false);
        }
    );

    // Add the tool to the server
    server.addTool(calculatorTool);
    ```
  </Tab>

  <Tab title="Async API">
    ```java
    // Create a server
    McpAsyncServer server = McpServer.async(transportProvider)
        .serverInfo("my-server", "1.0.0")
        .build();

    // Define a tool that uses sampling
    var calculatorTool = new McpServerFeatures.AsyncToolSpecification(
        new Tool("ai-calculator", "Performs calculations using AI", schema),
        (exchange, arguments) -> {
            // Check if client supports sampling
            if (exchange.getClientCapabilities().sampling() == null) {
                return Mono.just(new CallToolResult("Client does not support AI capabilities", false));
            }

            // Create a sampling request
            McpSchema.CreateMessageRequest request = McpSchema.CreateMessageRequest.builder()
                .content(new McpSchema.TextContent("Calculate: " + arguments.get("expression")))
                .modelPreferences(McpSchema.ModelPreferences.builder()
                    .hints(List.of(
                        McpSchema.ModelHint.of("claude-3-sonnet"),
                        McpSchema.ModelHint.of("claude")
                    ))
                    .intelligencePriority(0.8)  // 知能重視の度合い
                    .speedPriority(0.5)         // 速度重視の度合い（中程度）
                    .build())
                .systemPrompt("You are a helpful calculator assistant. Provide only the numerical answer.")
                .maxTokens(100)
                .build();

            // Request sampling from the client
            return exchange.createMessage(request)
                .map(result -> {
                    // Process the result
                    String answer = result.content().text();
                    return new CallToolResult(answer, false);
                });
        }
    );

    // Add the tool to the server
    server.addTool(calculatorTool)
        .subscribe();
    ```
  </Tab>
</Tabs>

`CreateMessageRequest` オブジェクトでは、次の項目を指定できます: `Content` - モデルへの入力テキストまたは画像、`Model Preferences` - モデル選択のためのヒントと優先度、`System Prompt` - モデルの動作に関する指示、`Max Tokens` - 生成される応答の最大長。

<div id="logging-support">
  ### ログ機能のサポート
</div>

サーバーは構造化ログ機能を提供しており、クライアントへ重大度の異なるログメッセージを送信できます。ログ通知は、ツール、リソース、プロンプトの呼び出しなど、既存のクライアントセッション内からのみ送信可能です。

たとえば、ツールのハンドラー関数内からログメッセージを送信できます。クライアント側では、サーバーからのログメッセージを受け取るロギングコンシューマを登録し、フィルタリングのための最小ログレベルを設定できます。

```java
var mcpClient = McpClient.sync(transport)
        .loggingConsumer(notification -> {
            System.out.println("Received log message: " + notification.data());
        })
        .build();

mcpClient.initialize();

mcpClient.setLoggingLevel(McpSchema.LoggingLevel.INFO);

// ログ通知を送るツールを呼び出す
CallToolResult result = mcpClient.callTool(new McpSchema.CallToolRequest("logging-test", Map.of()));
```

サーバーは、ツール／リソース／プロンプトのハンドラー関数内で `McpAsyncServerExchange`／`McpSyncServerExchange` オブジェクトを使用してログメッセージを送信できます。

```java
var tool = new McpServerFeatures.AsyncToolSpecification(
    new McpSchema.Tool("logging-test", "Test logging notifications", emptyJsonSchema),
    (exchange, request) -> {

      exchange.loggingNotification( // exchange を使ってログメッセージを送信
          McpSchema.LoggingMessageNotification.builder()
            .level(McpSchema.LoggingLevel.DEBUG)
            .logger("test-logger")
            .data("Debug message")
            .build())
        .block();

      return Mono.just(new CallToolResult("Logging test completed", false));
    });

var mcpServer = McpServer.async(mcpServerTransportProvider)
  .serverInfo("test-server", "1.0.0")
  .capabilities(
    ServerCapabilities.builder()
      .logging() // ログ機能を有効化
      .tools(true)
      .build())
  .tools(tool)
  .build();
```

クライアントは `mcpClient.setLoggingLevel(level)` リクエストで、受信する最小ログレベルを制御できます。設定したレベル未満のメッセージはフィルタリングされます。
サポートされているログレベル（重大度の低い順）：DEBUG（0）、INFO（1）、NOTICE（2）、WARNING（3）、ERROR（4）、CRITICAL（5）、ALERT（6）、EMERGENCY（7）

<div id="error-handling">
  ## エラーハンドリング
</div>

SDK は McpError クラスを通じて包括的なエラーハンドリングを提供し、プロトコル互換性、トランスポート通信、JSON-RPC 2.0 メッセージング、ツールの実行、リソース管理、プロンプトの取り扱い、タイムアウト、接続に関する問題を網羅します。この統一的なエラーハンドリングにより、同期処理・非同期処理のいずれにおいても一貫性と信頼性の高いエラー管理が実現します。
