<div id="core-mcp-concepts">
  ### MCPのコア概念
</div>

MCPサーバーは、主に次の3種類の機能を提供できます。

1. **[リソース](/ja/docs/learn/server-concepts#resources)**: クライアントが読み取れるファイルのようなデータ（APIレスポンスやファイル内容など）
2. **[ツール](/ja/docs/learn/server-concepts#tools)**: LLMが呼び出せる関数（ユーザーの承認が必要）
3. **[プロンプト](/ja/docs/learn/server-concepts#prompts)**: 特定のタスク達成を支援するための事前作成済みテンプレート

このチュートリアルでは、主にツールに焦点を当てます。

    ### MCPサーバーでのログ出力

    MCPサーバーを実装する際は、ログの取り扱いに注意してください。

    **STDIOベースのサーバーの場合：** 標準出力（stdout）には決して書き込まないでください。これには次が含まれます：

    * Python における `print()` 文
    * JavaScriptの `console.log()`
    * Goの `fmt.Println()`
    * 他の言語における類似の stdout 関数

    標準出力に書き込むと、JSON-RPC 2.0 のメッセージが破損し、サーバーが壊れます。

    **HTTPベースのサーバーの場合：** 標準出力でのログ出力で問題ありません。HTTPレスポンスの送信を妨げないためです。

    ### ベストプラクティス

    1. stderr またはファイルに出力するロギングライブラリを使用してください。

    ### クイック例

    ```python

    print("リクエストを処理中")

    # ✅ 良い例（STDIO）
    import logging
    logging.info("リクエストを処理中")
    ```

    ## サーバーのビルド

    ### パッケージのインポートとインスタンスの設定

    `weather.py` の先頭に次を追加してください：

    ```python
    from typing import Any
    import httpx
    from mcp.server.fastmcp import FastMCP

    # FastMCPサーバーを初期化する
    mcp = FastMCP("weather")

    # 定数
    NWS_API_BASE = "https://api.weather.gov"
    USER_AGENT = "weather-app/1.0"
    ```

    FastMCPクラスはPythonの型ヒントとドキュメンテーション文字列（docstring）を用いてツール定義を自動生成し、MCPのツールを簡単に作成・保守できるようにします。

    ### ヘルパー関数

    次に、National Weather Service APIからデータを取得して整形するためのヘルパー関数を追加しましょう。

    ```python
    async def make_nws_request(url: str) -> dict[str, Any] | None:
        """適切なエラー処理を行い、NWS API にリクエストを送信します。"""
        headers = {
            "User-Agent": USER_AGENT,
            "Accept": "application/geo+json"
        }
        async with httpx.AsyncClient() as client:
            try:
                response = await client.get(url, headers=headers, timeout=30.0)
                response.raise_for_status()
                return response.json()
            except Exception:
                return None

    def format_alert(feature: dict) -> str:
        """警報のフィーチャを読みやすい文字列に整形します。"""
        props = feature["properties"]
        return f"""
    Event: {props.get('event', '不明')}
    Area: {props.get('areaDesc', '不明')}
    Severity: {props.get('severity', '不明')}
    Description: {props.get('description', '説明はありません')}
    Instructions: {props.get('instruction', '特別な指示はありません')}
    """
    ```

    ### ツール実行の実装方法

    各ツールのロジックを実際に実行するのはツール実行ハンドラの役割です。追加しましょう：

    ```python
    @mcp.tool()
    async def get_alerts(state: str) -> str:
        """米国の各州に発表されている気象警報を取得します。

        引数:
            state: 2文字の米国州コード（例：CA、NY）
        """
        url = f"{NWS_API_BASE}/alerts/active/area/{state}"
        data = await make_nws_request(url)

        if not data or "features" not in data:
            return "警報を取得できないか、警報が見つかりませんでした。"

        if not data["features"]:
            return "この州に現在有効な警報はありません。"

        alerts = [format_alert(feature) for feature in data["features"]]
        return "\n---\n".join(alerts)

    @mcp.tool()
    async def get_forecast(latitude: float, longitude: float) -> str:
        """指定地点の天気予報を取得します。

        引数:
            latitude: 地点の緯度
            longitude: 地点の経度
        """
        # First get the forecast grid endpoint
        points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
        points_data = await make_nws_request(points_url)

        if not points_data:
            return "この地点の予報データを取得できません。"

        # Get the forecast URL from the points response
        forecast_url = points_data["properties"]["forecast"]
        forecast_data = await make_nws_request(forecast_url)

        if not forecast_data:
            return "詳細な予報を取得できません。"

        # 期間ごとの情報を読みやすい予報に整形
        periods = forecast_data["properties"]["periods"]
        forecasts = []
        for period in periods[:5]:  # 直近5期間のみ表示
            forecast = f"""
    {period['name']}:
    Temperature: {period['temperature']}°{period['temperatureUnit']}
    Wind: {period['windSpeed']} {period['windDirection']}
    Forecast: {period['detailedForecast']}
    """
            forecasts.append(forecast)

        return "\n---\n".join(forecasts)
    ```

    ### MCPサーバーのログ記録

    MCPサーバーを実装する際は、ログの取り扱いに注意してください。

    **STDIOベースのサーバーの場合:** 標準出力（stdout）には絶対に書き込まないでください。これには次のものが含まれます：

    * Python における `print()` 文
    * JavaScript の `console.log()`
    * Go の `fmt.Println()`
    * 他の言語における類似のstdout関数

    stdout に書き込むと JSON-RPC メッセージが損傷し、サーバーが故障します。

    **HTTPベースのサーバーの場合：** 標準出力へのロギングで構いません。HTTPレスポンスに影響しないためです。

    ### ベストプラクティス

    1. Pythonの`logging`など、stderrまたはファイルに出力するロギングライブラリを使用してください。
    2. JavaScriptの場合は特に注意してください—`console.log()`はデフォルトで標準出力（stdout）に書き込みます

    ## サーバーの構築

    ### パッケージのインポートとインスタンスの設定

    `src/index.ts` の先頭に以下を追加します：

    ```typescript
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
    import { z } from "zod";

    const NWS_API_BASE = "https://api.weather.gov";
    const USER_AGENT = "weather-app/1.0";

    // サーバーのインスタンスを作成
    const server = new McpServer({
      name: "weather",
      version: "1.0.0",
      capabilities: {
        resources: {},
        tools: {},
      },
    });
    ```

    ### ヘルパー関数

    次に、National Weather Service の API からデータを取得して整形するためのヘルパー関数を追加しましょう。

    ```typescript
    // NWS API リクエスト用のヘルパー関数
    async function makeNWSRequest<T>(url: string): Promise<T | null> {
      const headers = {
        "User-Agent": USER_AGENT,
        Accept: "application/geo+json",
      };

      try {
        const response = await fetch(url, { headers });
        if (!response.ok) {
          throw new Error(`HTTPエラーが発生しました。ステータス: ${response.status}`);
        }
        return (await response.json()) as T;
      } catch (error) {
        console.error("NWS リクエストの実行中にエラーが発生しました:", error);
        return null;
      }
    }

    interface AlertFeature {
      properties: {
        event?: string;
        areaDesc?: string;
        severity?: string;
        status?: string;
        headline?: string;
      };
    }

    // アラートデータの整形
    function formatAlert(feature: AlertFeature): string {
      const props = feature.properties;
      return [
        `イベント: ${props.event || "不明"}`,
        `地域: ${props.areaDesc || "不明"}`,
        `深刻度: ${props.severity || "不明"}`,
        `ステータス: ${props.status || "不明"}`,
        `見出し: ${props.headline || "見出しなし"}`,
        "---",
      ].join("\n");
    }

    interface ForecastPeriod {
      name?: string;
      temperature?: number;
      temperatureUnit?: string;
      windSpeed?: string;
      windDirection?: string;
      shortForecast?: string;
    }

    interface AlertsResponse {
      features: AlertFeature[];
    }

    interface PointsResponse {
      properties: {
        forecast?: string;
      };
    }

    interface ForecastResponse {
      properties: {
        periods: ForecastPeriod[];
      };
    }
    ```

    ### ツール実行の実装方法

    ツール実行ハンドラは、各ツールのロジックを実際に実行する役割を担います。追加しましょう：

    ```typescript
    // 天気ツールを登録
    server.tool(
      "get_alerts",
      "州の気象警報を取得"
      {
        state: z.string().length(2).describe("2文字の州コード（例：CA、NY）"),
      },
      async ({ state }) => {
        const stateCode = state.toUpperCase();
        const alertsUrl = `${NWS_API_BASE}/alerts?area=${stateCode}`;
        const alertsData = await makeNWSRequest<AlertsResponse>(alertsUrl);

        if (!alertsData) {
          return {
            content: [
              {
                type: "text",
                text: "警報データを取得できませんでした",
              },
            ],
          };
        }

        const features = alertsData.features || [];
        if (features.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: `${stateCode} に有効な警報はありません`,
              },
            ],
          };
        }

        const formattedAlerts = features.map(formatAlert);
        const alertsText = `${stateCode} の現在の警報:\n\n${formattedAlerts.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: alertsText,
            },
          ],
        };
      },
    );

    server.tool(
      "get_forecast",
      "地点の天気予報を取得"
      {
        latitude: z.number().min(-90).max(90).describe("地点の緯度"),
        longitude: z
          .number()
          .min(-180)
          .max(180)
          .describe("地点の経度"),
      },
      async ({ latitude, longitude }) => {
        // グリッドポイントデータを取得
        const pointsUrl = `${NWS_API_BASE}/points/${latitude.toFixed(4)},${longitude.toFixed(4)}`;
        const pointsData = await makeNWSRequest<PointsResponse>(pointsUrl);

        if (!pointsData) {
          return {
            content: [
              {
                type: "text",
                text: `座標 ${latitude}, ${longitude} のグリッドポイントデータを取得できませんでした。NWS API でサポートされていない可能性があります（米国内の地点のみ対応）。`,
              },
            ],
          };
        }

        const forecastUrl = pointsData.properties?.forecast;
        if (!forecastUrl) {
          return {
            content: [
              {
                type: "text",
                text: "グリッドポイントデータから予報URLを取得できませんでした",
              },
            ],
          };
        }

        // 予報データを取得
        const forecastData = await makeNWSRequest<ForecastResponse>(forecastUrl);
        if (!forecastData) {
          return {
            content: [
              {
                type: "text",
                text: "予報データを取得できませんでした",
              },
            ],
          };
        }

        const periods = forecastData.properties?.periods || [];
        if (periods.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: "利用可能な予報期間がありません",
              },
            ],
          };
        }

        // 予報期間を整形
        const formattedForecast = periods.map((period: ForecastPeriod) =>
          [
            `${period.name || "不明"}:`,
            `気温: ${period.temperature || "不明"}°${period.temperatureUnit || "F"}`,
            `風: ${period.windSpeed || "不明"} ${period.windDirection || ""}`,
            `${period.shortForecast || "予報はありません"}`,
            "---",
          ].join("\n"),
        );

        const forecastText = `${latitude}, ${longitude} の予報:\n\n${formattedForecast.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: forecastText,
            },
          ],
        };
      },
    );
    ```

    ### サーバーの起動

    最後に、サーバーを起動するメイン関数を実装します：

    ```typescript
    async function main() {
      const transport = new StdioServerTransport();
      await server.connect(transport);
      console.error("Weather MCPサーバーがstdioで起動しました")
    }

    main().catch((error) => {
      console.error("main()で致命的なエラーが発生しました:", error)
      process.exit(1);
    });
    ```

    ### MCPサーバーのロギング

    MCPサーバーを実装する際は、ログの取り扱いに注意してください。

    **STDIOベースのサーバーの場合:** 標準出力（stdout）には一切書き込まないでください。対象には次が含まれます：

    * Python における `print()` 文
    * JavaScript における `console.log()`
    * Go における `fmt.Println()`
    * 他の言語における同様の標準出力（stdout）関数

    stdout へ出力すると JSON-RPC 2.0 のメッセージが破損し、サーバーが動作しなくなります。

    **HTTPベースのサーバーの場合：** 標準出力へのロギングで問題ありません。HTTPレスポンスに干渉しないためです。

    ### ベストプラクティス

    1. stderr またはファイルに出力するロギングライブラリを使用してください。
    2. 構成したロギングライブラリが STDOUT に出力しないようにする

    ```java

    		SpringApplication.run(McpServerApplication.class, args);

    	@Bean
    	public ToolCallbackProvider weatherTools(WeatherService weatherService) {
    		return MethodToolCallbackProvider.builder().toolObjects(weatherService).build();
    ```

    ### インスタンスの設定

    サーバーの初期化関数を追加する：

    ```kotlin
    // MCPサーバーを起動するメイン関数
    fun `run mcp server`() {
        // シンプルな実装でMCPサーバーのインスタンスを作成する
        val server = Server(
            Implementation(
                name = "weather", // ツール名は「weather」
                version = "1.0.0" // 実装のバージョン
            ),
            ServerOptions(
                capabilities = ServerCapabilities(tools = ServerCapabilities.Tools(listChanged = true))
            )
        )

        // サーバー間通信のために標準I/Oを使ったトランスポートを作成する
        val transport = StdioServerTransport(
            System.`in`.asInput(),
            System.out.asSink().buffered()
        )

        runBlocking {
            server.connect(transport)
            val done = Job()
            server.onClose {
                done.complete()
            }
            done.join()
        }
    }
    ```

    ### 天気API向けヘルパー関数

    次に、National Weather Service の API に問い合わせて応答を変換するための関数とデータクラスを追加しましょう。

    ```kotlin
    // 指定した緯度・経度の予報を取得する拡張関数
    suspend fun HttpClient.getForecast(latitude: Double, longitude: Double): List<String> {
        val points = this.get("/points/$latitude,$longitude").body<Points>()
        val forecast = this.get(points.properties.forecast).body<Forecast>()
        return forecast.properties.periods.map { period ->
            """
                ${period.name}:
                気温：${period.temperature} ${period.temperatureUnit}
                風：${period.windSpeed} ${period.windDirection}
                予報：${period.detailedForecast}
            """.trimIndent()
        }
    }

    // 指定した州の気象警報を取得する拡張関数
    suspend fun HttpClient.getAlerts(state: String): List<String> {
        val alerts = this.get("/alerts/active/area/$state").body<Alert>()
        return alerts.features.map { feature ->
            """
                事象：${feature.properties.event}
                対象地域：${feature.properties.areaDesc}
                重大度：${feature.properties.severity}
                説明：${feature.properties.description}
                指示：${feature.properties.instruction}
            """.trimIndent()
        }
    }

    @Serializable
    data class Points(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val forecast: String)
    }

    @Serializable
    data class Forecast(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val periods: List<Period>)

        @Serializable
        data class Period(
            val number: Int, val name: String, val startTime: String, val endTime: String,
            val isDaytime: Boolean, val temperature: Int, val temperatureUnit: String,
            val temperatureTrend: String, val probabilityOfPrecipitation: JsonObject,
            val windSpeed: String, val windDirection: String,
            val shortForecast: String, val detailedForecast: String,
        )
    }

    @Serializable
    data class Alert(
        val features: List<Feature>
    ) {
        @Serializable
        data class Feature(
            val properties: Properties
        )

        @Serializable
        data class Properties(
            val event: String, val areaDesc: String, val severity: String,
            val description: String, val instruction: String?,
        )
    }
    ```

    ### ツール実行の実装

    各ツールのロジックを実際に実行するのがツール実行ハンドラーです。追加しましょう：

    ```kotlin
    // 既定のリクエスト設定とJSONのコンテンツネゴシエーションを備えたHTTPクライアントを作成する
    val httpClient = HttpClient {
        defaultRequest {
            url("https://api.weather.gov")
            headers {
                append("Accept", "application/geo+json")
                append("User-Agent", "WeatherApiClient/1.0")
            }
            contentType(ContentType.Application.Json)
        }
        // JSONのシリアル化／デシリアル化のためにコンテンツネゴシエーションプラグインをインストールする
        install(ContentNegotiation) { json(Json { ignoreUnknownKeys = true }) }
    }

    // 州別の気象警報を取得するツールを登録する
    server.addTool(
        name = "get_alerts",
        description = """
            米国の州の気象警報を取得します。入力は2文字の州コード（例：CA、NY）です
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("state") {
                    put("type", "string")
                    put("description", "2文字の米国州コード（例：CA、NY）")
                }
            },
            required = listOf("state")
        )
    ) { request ->
        val state = request.arguments["state"]?.jsonPrimitive?.content
        if (state == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("‘state’ パラメータは必須です。"))
            )
        }

        val alerts = httpClient.getAlerts(state)

        CallToolResult(content = alerts.map { TextContent(it) })
    }

    // 緯度・経度で天気予報を取得するツールを登録する
    server.addTool(
        name = "get_forecast",
        description = """
            指定した緯度・経度の天気予報を取得します
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("latitude") { put("type", "number") }
                putJsonObject("longitude") { put("type", "number") }
            },
            required = listOf("latitude", "longitude")
        )
    ) { request ->
        val latitude = request.arguments["latitude"]?.jsonPrimitive?.doubleOrNull
        val longitude = request.arguments["longitude"]?.jsonPrimitive?.doubleOrNull
        if (latitude == null || longitude == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("‘latitude’ と ‘longitude’ パラメータは必須です。"))
            )
        }

        val forecast = httpClient.getForecast(latitude, longitude)

        CallToolResult(content = forecast.map { TextContent(it) })
    }
    ```

    ### MCPサーバーのログ記録

    MCPサーバーを実装する際は、ログの取り扱いに注意してください：

    **STDIO ベースのサーバーの場合:** 標準出力（stdout）へは絶対に書き込まないでください。対象には次が含まれます:

    * Python における `print()` 文
    * JavaScript の `console.log()`
    * Go の `fmt.Println()`
    * 他言語における同様の stdout 関数

    stdout に書き込むと、JSON-RPC 2.0 のメッセージが破損し、サーバーが故障します。

    **HTTPベースのサーバーの場合：** 標準出力へのログ出力で問題ありません。HTTPレスポンスに干渉しないためです。

    ## サーバーの構築

    プロジェクトの `Program.cs` ファイルを開き、内容を次のコードに置き換えます。

    ```csharp
    using Microsoft.Extensions.DependencyInjection;
    using Microsoft.Extensions.Hosting;
    using ModelContextProtocol;
    using System.Net.Http.Headers;

    var builder = Host.CreateEmptyApplicationBuilder(settings: null);

    builder.Services.AddMcpServer()
        .WithStdioServerTransport()
        .WithToolsFromAssembly();

    builder.Services.AddSingleton(_ =>
    {
        var client = new HttpClient() { BaseAddress = new Uri("https://api.weather.gov") };
        client.DefaultRequestHeaders.UserAgent.Add(new ProductInfoHeaderValue("weather-tool", "1.0"));
        return client;
    });

    var app = builder.Build();

    await app.RunAsync();
    ```

    <Note>
      `ApplicationHostBuilder` を作成する際は、`CreateDefaultBuilder` ではなく `CreateEmptyApplicationBuilder` を使用してください。これにより、サーバーがコンソールに余分なメッセージを書き込まなくなります。これは STDIO トランスポートを使用するサーバーの場合にのみ必要です。
    </Note>

    このコードは、Model Context Protocol（MCP）SDK を使用し、標準入出力のトランスポートで動作する MCPサーバーを作成するための基本的なコンソールアプリケーションをセットアップします。

    ### 天気API用ヘルパー関数

    `HttpClient` 用の拡張クラスを作成し、JSON リクエスト処理を簡単にします：

    ```csharp
    using System.Text.Json;

    internal static class HttpClientExt
    {
        public static async Task<JsonDocument> ReadJsonDocumentAsync(this HttpClient client, string requestUri)
        {
            using var response = await client.GetAsync(requestUri);
            response.EnsureSuccessStatusCode();
            return await JsonDocument.ParseAsync(await response.Content.ReadAsStreamAsync());
        }
    }
    ```

    次に、National Weather Service APIに問い合わせ、応答を変換するためのツール実行ハンドラーを備えたクラスを定義します。

    ```csharp
    using ModelContextProtocol.Server;
    using System.ComponentModel;
    using System.Globalization;
    using System.Text.Json;

    namespace QuickstartWeatherServer.Tools;

    [McpServerToolType]
    public static class WeatherTools
    {
        [McpServerTool, Description("米国の各州の気象警報を取得します。")]
        public static async Task<string> GetAlerts(
            HttpClient client,
            [Description("警報を取得する対象の米国の州。")] string state)
        {
            using var jsonDocument = await client.ReadJsonDocumentAsync($"/alerts/active/area/{state}");
            var jsonElement = jsonDocument.RootElement;
            var alerts = jsonElement.GetProperty("features").EnumerateArray();

            if (!alerts.Any())
            {
                return "この州には現在有効な警報はありません。";
            }

            return string.Join("\n--\n", alerts.Select(alert =>
            {
                JsonElement properties = alert.GetProperty("properties");
                return $"""
                        Event: {properties.GetProperty("event").GetString()}
                        Area: {properties.GetProperty("areaDesc").GetString()}
                        Severity: {properties.GetProperty("severity").GetString()}
                        Description: {properties.GetProperty("description").GetString()}
                        Instruction: {properties.GetProperty("instruction").GetString()}
                        """;
            }));
        }

        [McpServerTool, Description("指定した地点の天気予報を取得します。")]
        public static async Task<string> GetForecast(
            HttpClient client,
            [Description("地点の緯度。")] double latitude,
            [Description("地点の経度。")] double longitude)
        {
            var pointUrl = string.Create(CultureInfo.InvariantCulture, $"/points/{latitude},{longitude}");
            using var jsonDocument = await client.ReadJsonDocumentAsync(pointUrl);
            var forecastUrl = jsonDocument.RootElement.GetProperty("properties").GetProperty("forecast").GetString()
                ?? throw new Exception($"予報URLが {client.BaseAddress}points/{latitude},{longitude} から提供されていません")

            using var forecastDocument = await client.ReadJsonDocumentAsync(forecastUrl);
            var periods = forecastDocument.RootElement.GetProperty("properties").GetProperty("periods").EnumerateArray();

            return string.Join("\n---\n", periods.Select(period => $"""
                    {period.GetProperty("name").GetString()}
                    Temperature: {period.GetProperty("temperature").GetInt32()}°F
                    Wind: {period.GetProperty("windSpeed").GetString()} {period.GetProperty("windDirection").GetString()}
                    Forecast: {period.GetProperty("detailedForecast").GetString()}
                    """));
        }
    }
    ```

<div id="whats-happening-under-the-hood">
  ## 内部で何が起きているのか
</div>

質問すると:

1. クライアントがあなたの質問をClaudeに送信します
2. Claudeが利用可能なツールを分析し、どれを使うか判断します
3. クライアントが選ばれたツールをMCPサーバー経由で実行します
4. 結果がClaudeに返されます
5. Claudeが自然言語での応答を生成します
6. 応答が表示されます！

<div id="troubleshooting">
  ## トラブルシューティング
</div>

<AccordionGroup>
  <Accordion title="Claude for Desktop の統合に関する問題">
    **Claude for Desktop のログを取得する**

    MCP に関連する Claude.app のログは、`~/Library/Logs/Claude` のログファイルに出力されます：

    * `mcp.log` には、MCP の接続や接続失敗に関する一般的なログが含まれます。
    * `mcp-server-SERVERNAME.log` という名前のファイルには、指定したサーバーからのエラー（stderr）ログが含まれます。

    最近のログを一覧表示し、新しいログを追いかけるには、次のコマンドを実行します：

    ```bash
    # Claude のログにエラーがないか確認
    tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
    ```

    **Claude にサーバーが表示されない**

    1. `claude_desktop_config.json` の文法を確認する
    2. プロジェクトへのパスが相対パスではなく絶対パスになっていることを確認する
    3. Claude for Desktop を完全に再起動する

    **ツール呼び出しが無言で失敗する**

    Claude がツールを使おうとするが失敗する場合：

    1. Claude のログにエラーがないか確認する
    2. サーバーがエラーなくビルド・実行できることを確認する
    3. Claude for Desktop を再起動してみる

    **何をしてもうまくいきません。どうすればいいですか？**

    より強力なデバッグ用ツールと詳しい手順については、[デバッグガイド](/ja/legacy/tools/debugging) を参照してください。
  </Accordion>

  <Accordion title="Weather API に関する問題">
    **エラー：グリッドポイントデータの取得に失敗しました**

    これは通常、次のいずれかを意味します：

    1. 座標が米国内ではない
    2. NWS API に不具合が発生している
    3. レート制限を受けている

    対処：

    * 米国内の座標を使用していることを確認する
    * リクエスト間に短い待機時間を入れる
    * NWS API のステータスページを確認する

    **エラー：[STATE] に有効な警報はありません**

    これはエラーではありません。該当の州に現在発令中の気象警報がないことを意味します。別の州を試すか、荒天時に確認してください。
  </Accordion>
</AccordionGroup>
