---
title: ローカルのMCPサーバーに接続する
description: ローカルのMCPサーバーでClaude Desktopを拡張し、ファイルシステムへのアクセスやその他の強力な連携を有効にする方法を学びます
---

Model Context Protocol（MCP）サーバーは、ローカルのリソースやツールへの安全かつ制御されたアクセスを提供することで、AIアプリケーションの機能を拡張します。多くのクライアントがMCPをサポートしており、さまざまなプラットフォームやアプリケーションで多様な連携が可能になります。

このガイドでは、MCPをサポートする[多数のクライアント](/ja/clients)のひとつであるClaude Desktopを例に、ローカルのMCPサーバーへ接続する方法を解説します。Claude Desktopの実装に焦点を当てますが、ここで扱う概念は他のMCP互換クライアントにも広く当てはまります。チュートリアルの最後には、あなたの明示的な許可のもとで、Claudeがコンピュータ上のファイルとやり取りし、新しいドキュメントを作成し、フォルダを整理し、ファイルシステムを検索できるようになります。

<div id="prerequisites">
  ## 前提条件
</div>

このチュートリアルを開始する前に、次のものがシステムにインストールされていることを確認してください。

<div id="claude-desktop">
  ### Claude Desktop
</div>

お使いのオペレーティングシステム向けに[Claude Desktop](https://claude.ai/download)をダウンロードしてインストールしてください。Claude Desktopは現在、macOSとWindowsに対応しています。Linux対応は近日公開予定です。

すでにClaude Desktopをインストール済みの場合は、Claudeメニューをクリックし、「Check for Updates...」を選択して最新バージョンか確認してください。

<div id="nodejs">
  ### Node.js
</div>

Filesystem Server と他の多くの MCPサーバーは、実行に Node.js を必要とします。ターミナルまたはコマンドプロンプトを開き、次のコマンドを実行して Node.js のインストールを確認してください:

```bash
node --version
```

Node.js がインストールされていない場合は、[nodejs.org](https://nodejs.org/) からダウンロードしてください。安定性のため、LTS（Long Term Support）版の使用を推奨します。

<div id="understanding-mcp-servers">
  ## MCPサーバーを理解する
</div>

MCPサーバーは、標準化されたプロトコルを通じて Claude Desktop に特定の機能を提供する、あなたのコンピュータ上で動作するプログラムです。各サーバーは、あなたの承認のもとで Claude が操作を実行するために利用できるツールを公開します。これからインストールする Filesystem Server は、次のツールを提供します:

* ファイル内容およびディレクトリ構造の読み取り
* 新規ファイルおよびディレクトリの作成
* ファイルの移動および名称変更
* 名前または内容によるファイル検索

すべての操作は実行前にあなたの明示的な承認が必要で、Claude がアクセス・変更できる範囲をあなたが完全に管理できます。

<div id="installing-the-filesystem-server">
  ## Filesystem Server のインストール
</div>

この手順では、アプリケーション起動時に Filesystem Server が自動的に開始されるよう、Claude Desktop を設定します。設定は JSON ファイルで行い、Claude Desktop に対して起動するサーバーと接続方法を指示します。

<Steps>
  <Step title="Claude Desktop の設定を開く">
    まず、Claude Desktop の設定を開きます。システムのメニューバーにある Claude メニュー（Claude ウィンドウ内の設定ではありません）をクリックし、「Settings...」を選択します。

    macOS では、上部のメニューバーに表示されます:

    <Frame style={{ textAlign: "center" }}>
      <img src="/images/quickstart-menu.png" width="400" alt="Settings オプションを示す Claude Desktop のメニュー" />
    </Frame>

    これにより、Claude アカウント設定とは別の、Claude Desktop 用の設定ウィンドウが開きます。
  </Step>

  <Step title="開発者向け設定にアクセスする">
    Settings ウィンドウで、左サイドバーの「Developer」タブに移動します。このセクションには、MCP サーバーやその他の開発者向け機能を設定するためのオプションが含まれています。

    「Edit Config」ボタンをクリックして設定ファイルを開きます:

    <Frame>
      <img src="/images/quickstart-developer.png" alt="Edit Config ボタンを示す開発者向け設定" />
    </Frame>

    この操作により、設定ファイルが存在しない場合は新規作成され、存在する場合はそのファイルが開かれます。ファイルの場所は次のとおりです:

    * **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
    * **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
  </Step>

  <Step title="Filesystem Server を構成する">
    設定ファイルの内容を、次の JSON 構造に置き換えます。この設定により、指定したディレクトリへのアクセス権を付与して Filesystem Server を起動するよう Claude Desktop に指示します:

    <CodeGroup>
      ```json macOS
      {
        "mcpServers": {
          "filesystem": {
            "command": "npx",
            "args": [
              "-y",
              "@modelcontextprotocol/server-filesystem",
              "/Users/username/Desktop",
              "/Users/username/Downloads"
            ]
          }
        }
      }
      ```

      ```json Windows
      {
        "mcpServers": {
          "filesystem": {
            "command": "npx",
            "args": [
              "-y",
              "@modelcontextprotocol/server-filesystem",
              "C:\\Users\\username\\Desktop",
              "C:\\Users\\username\\Downloads"
            ]
          }
        }
      }
      ```
    </CodeGroup>

    `username` を実際のユーザー名に置き換えてください。`args` 配列に記載されたパスは、Filesystem Server がアクセスできるディレクトリを指定します。必要に応じて、これらのパスを変更したり、ディレクトリを追加したりできます。

    <Tip>
      **構成のポイント**

      * `"filesystem"`: Claude Desktop に表示されるサーバーのわかりやすい名称
      * `"command": "npx"`: Node.js の npx ツールでサーバーを実行
      * `"-y"`: サーバーパッケージのインストールを自動承認
      * `"@modelcontextprotocol/server-filesystem"`: Filesystem Server のパッケージ名
      * 残りの引数: サーバーにアクセスを許可するディレクトリ
    </Tip>

    <Warning>
      **セキュリティ上の注意**

      Claude に読み取りや変更を許可しても問題ないディレクトリにのみアクセス権を与えてください。サーバーはあなたのユーザーアカウント権限で実行されるため、手動で行えるファイル操作はすべて実行可能です。
    </Warning>
  </Step>

  <Step title="Claude Desktop を再起動する">
    設定ファイルを保存したら、Claude Desktop を完全に終了し、再起動します。新しい設定を読み込み、MCP サーバーを起動するには再起動が必要です。

    再起動に成功すると、会話入力ボックスの右下に MCP サーバーのインジケーター <img src="/images/claude-desktop-mcp-slider.svg" style={{display: 'inline', margin: 0, height: '1.3em'}} /> が表示されます:

    <Frame>
      <img src="/images/quickstart-slider.png" alt="MCP サーバーのインジケーターを示す Claude Desktop のインターフェース" />
    </Frame>

    このインジケーターをクリックすると、Filesystem Server が提供するツールを確認できます:

    <Frame style={{ textAlign: "center" }}>
      <img src="/images/quickstart-tools.png" width="400" alt="Claude Desktop の利用可能なファイルシステム関連ツール" />
    </Frame>

    サーバーのインジケーターが表示されない場合は、[Troubleshooting](#troubleshooting) セクションのトラブルシューティング手順を参照してください。
  </Step>
</Steps>

<div id="using-the-filesystem-server">
  ## Filesystem Server の利用
</div>

Filesystem Server を接続すると、Claude がファイルシステムとやり取りできるようになります。以下のリクエスト例を試して、どのようなことができるか確かめてみてください。

<div id="file-management-examples">
  ### ファイル管理の例
</div>

* **「詩を書いてデスクトップに保存してくれる？」** - Claude が詩を作成し、デスクトップに新しいテキストファイルを保存します
* **「ダウンロードフォルダにある仕事関連のファイルは何？」** - Claude がダウンロードフォルダをスキャンし、仕事関連のドキュメントを特定します
* **「デスクトップ上のすべての画像を『Images』という新しいフォルダに整理してください」** - Claude がフォルダを作成し、画像ファイルをそこに移動します

<div id="how-approval-works">
  ### 承認の仕組み
</div>

ファイルシステムの操作を実行する前に、Claude は承認を求めます。これにより、すべての操作をあなたが制御できます。

<Frame style={{ textAlign: "center" }}>
  <img src="/images/quickstart-approve.png" width="500" alt="ファイル操作の実行について承認を求める Claude" />
</Frame>

承認する前に各リクエストを丁寧に確認してください。提案された操作に不安がある場合は、いつでも拒否できます。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

Filesystem Server のセットアップや使用時に問題が発生した場合は、次の対処法をお試しください。よくある問題に対応しています。

<AccordionGroup>
  <Accordion title="Claude にサーバーが表示されない／ハンマーアイコンが見当たらない">
    1. Claude Desktop を完全に再起動する
    2. `claude_desktop_config.json` の構文を確認する
    3. `claude_desktop_config.json` に記載したファイルパスが有効で、相対パスではなく絶対パスになっていることを確認する
    4. [ログ](#getting-logs-from-claude-for-desktop) を確認してサーバーが接続できない原因を調べる
    5. コマンドラインで、`claude_desktop_config.json` と同様に `username` を置き換えてサーバーを手動で実行し、エラーが出ないか確認する:

    <CodeGroup>
      ```bash macOS/Linux
      npx -y @modelcontextprotocol/server-filesystem /Users/username/Desktop /Users/username/Downloads
      ```

      ```powershell Windows
      npx -y @modelcontextprotocol/server-filesystem C:\Users\username\Desktop C:\Users\username\Downloads
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Claude Desktop のログ取得">
    MCP に関連する Claude.app のログは次の場所に出力されます:

    * macOS: `~/Library/Logs/Claude`

    * Windows: `%APPDATA%\Claude\logs`

    * `mcp.log` には MCP の接続状況や接続失敗に関する一般的なログが記録されます。

    * `mcp-server-SERVERNAME.log` というファイルには、該当サーバーからのエラー（stderr）ログが記録されます。

    最近のログを一覧表示し、新しいログを追跡するには次のコマンドを実行します（Windows では最近のログのみ表示されます）:

    <CodeGroup>
      ```bash macOS/Linux
      tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
      ```

      ```powershell Windows
      type "%APPDATA%\Claude\logs\mcp*.log"
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="ツール呼び出しが無言で失敗する">
    Claude がツールを使用しようとして失敗する場合:

    1. Claude のログでエラーを確認する
    2. サーバーがエラーなくビルド・実行できることを確認する
    3. Claude Desktop を再起動してみる
  </Accordion>

  <Accordion title="どれも上手くいきません。どうすればいいですか？">
    より強力なデバッグツールと詳しい手順については、[デバッグガイド](/ja/legacy/tools/debugging) を参照してください。
  </Accordion>

  <Accordion title="ENOENT エラーと Windows のパスにおける `${APPDATA}`">
    設定したサーバーが読み込まれず、ログ内のパスに `${APPDATA}` が含まれるエラーが表示される場合は、`claude_desktop_config.json` の `env` キーに `%APPDATA%` の展開後の値を追加する必要がある場合があります:

    ```json
    {
      "brave-search": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-brave-search"],
        "env": {
          "APPDATA": "C:\\Users\\user\\AppData\\Roaming\\",
          "BRAVE_API_KEY": "..."
        }
      }
    }
    ```

    この変更を反映したうえで、もう一度 Claude Desktop を起動してください。

    <Warning>
      **NPM はグローバルにインストールされている必要があります**

      `npx` コマンドは、NPM をグローバルにインストールしていない場合は引き続き失敗する可能性があります。NPM がすでにグローバルにインストールされていれば、システム上に `%APPDATA%\npm` が存在します。未インストールの場合は、次のコマンドで NPM をグローバルにインストールできます:

      ```bash
      npm install -g npm
      ```
    </Warning>
  </Accordion>
</AccordionGroup>
