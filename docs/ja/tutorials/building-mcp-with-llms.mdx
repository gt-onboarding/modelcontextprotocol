このガイドでは、LLMを活用してカスタムのModel Context Protocol（MCP）サーバーとクライアントを構築する方法を解説します。本チュートリアルではClaudeに重点を置きますが、他の最先端LLMでも同様に可能です。

<div id="preparing-the-documentation">
  ## ドキュメントの準備
</div>

開始する前に、Claude が MCP を理解できるよう、必要なドキュメントを集めます。

1. https://modelcontextprotocol.io/llms-full.txt にアクセスして、ドキュメント全文をコピーする
2. [MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) または [Python SDK リポジトリ](https://github.com/modelcontextprotocol/python-sdk) のいずれかに移動する
3. README ファイルおよびその他の関連ドキュメントをコピーする
4. これらのドキュメントを Claude との会話に貼り付ける

<div id="describing-your-server">
  ## サーバーの説明
</div>

ドキュメントを用意したら、どのようなサーバーを構築したいかをClaudeに明確に伝えてください。次の点を具体的に示しましょう:

- サーバーが公開するリソース
- 提供するツール
- 提供すべきプロンプト
- 連携が必要な外部システム

例:

```
次の要件を満たすMCPサーバーを構築してください:
- 自社のPostgreSQLデータベースに接続する
- テーブルスキーマをリソースとして公開する
- 読み取り専用のSQLクエリを実行するためのツールを提供する
- 一般的なデータ分析タスク向けのプロンプトを含める
```

<div id="working-with-claude">
  ## Claude を活用する
</div>

MCPサーバーで Claude を使用する際は、次の点に留意してください：

1. まずコア機能から着手し、反復的に機能を拡張する
2. 理解が曖昧なコード箇所は Claude に解説してもらう
3. 必要に応じて修正や改善を依頼する
4. サーバーのテストやエッジケース対応を Claude に支援してもらう

Claude は主要な MCP 機能の実装を幅広く支援できます：

- リソースの管理と公開
- ツールの定義と実装
- プロンプトのテンプレートとハンドラー
- エラー処理とロギング
- 接続およびトランスポートのセットアップ

<div id="best-practices">
  ## ベストプラクティス
</div>

Claude で MCPサーバーを構築する際は、次の点を心がけてください:

- 複雑なサーバーは小さな構成要素に分割する
- 各コンポーネントを、先に進む前に十分にテストする
- セキュリティを常に意識し、入力を検証して適切にアクセスを制限する
- 将来の保守に備えて、コードを適切にドキュメント化する
- MCPプロトコル仕様に忠実に従う

<div id="next-steps">
  ## 次のステップ
</div>

Claude がサーバーの構築を手伝ったら:

1. 生成コードを入念に見直す
2. MCP Inspector ツールでサーバーをテストする
3. Claude.app または他の MCPクライアント に接続する
4. 実運用での利用状況とフィードバックに基づいて改善を重ねる

要件が時間とともに変化しても、Claude はサーバーの修正・改善をサポートできます。

さらに詳しいガイダンスが必要ですか？MCP の機能実装や発生した問題のトラブルシューティングについて、Claude に具体的な質問をしてください。
