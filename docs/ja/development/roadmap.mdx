<div id="agents">
  ## エージェント
</div>

MCPがエージェント指向のワークフローにますます組み込まれる中、私たちは次の重要な改善に注力しています：

* **非同期処理**：長時間に及ぶ可能性のある処理をサポートし、切断や再接続が発生しても堅牢に対応できるようにする

<div id="authentication-and-security">
  ## 認証とセキュリティ
</div>

ユーザーの安全性を高め、より良い開発者体験を提供するために、認可とセキュリティに関するリソースを拡充しています。

* **ガイドとベストプラクティス**: 開発者がよくある落とし穴を避けられるよう、MCP を安全にデプロイするための具体的な手順や注意点を、ガイドおよびベストプラクティスとして整理・公開します。
* **Dynamic Client Registration（DCR）の代替**: スムーズなユーザー体験を損なわずに運用上の課題へ対処するため、DCR に代わる選択肢を検討します。
* **きめ細かな認可**: 機微な操作に対する最小限の権限付与を実現する仕組みとガイドラインを整備します。
* **エンタープライズ管理の認可**: シングルサインオン（SSO）を活用し、企業が MCPサーバーの認可を簡素化できる機能を追加します。
* **安全な認可のエリシテーション**: メインの MCPサーバーの認可とは別に、下流の API に対する安全な認可フローを統合できるようにします。

<div id="validation">
  ## 検証
</div>

強固な開発者エコシステムを育むため、次の領域に投資する計画です：

* **参照クライアント実装**：高品質なAIアプリケーションでプロトコルの機能をデモンストレーション
* **参照サーバー実装**：認証パターンとリモートデプロイのベストプラクティスを提示
* **適合性テストスイート**：クライアント、サーバー、SDKが仕様を正しく実装しているかを自動で検証

これらのツールは、エコシステム全体で一貫した動作を確保しつつ、開発者が自信を持ってModel Context Protocol（MCP）を実装できるよう支援します。

<div id="registry">
  ## レジストリ
</div>

MCP が真価を発揮するには、MCPサーバーを配布し発見するための効率的な手段が必要です。

私たちは、サーバーの集中的な発見とメタデータ管理を可能にする [**MCP Registry**](https://github.com/orgs/modelcontextprotocol/discussions/159) の開発を計画しています。このレジストリは主に、サードパーティのマーケットプレイスや探索サービスが活用できるAPIレイヤーとして機能します。

<div id="multimodality">
  ## マルチモーダリティ
</div>

MCP における AI 機能の全範囲をサポートします。以下を含みます:

* **追加のモダリティ**: 動画やその他のメディアタイプ
* **[ストリーミング](https://github.com/modelcontextprotocol/specification/issues/117)**: マルチパート、チャンク化メッセージ、双方向通信による対話的な体験
