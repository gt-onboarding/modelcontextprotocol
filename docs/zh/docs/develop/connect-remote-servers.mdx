远程 MCP 服务器可将 AI 应用的能力拓展到本地环境之外，提供对互联网上托管的工具、服务和数据源的访问。通过连接远程 MCP 服务器，你可以把 AI 助手从“有用的工具”升级为“见多识广的团队伙伴”，在实时访问外部资源的前提下处理复杂的多步项目。

许多客户端如今都支持远程 MCP 服务器，从而带来广泛的集成可能性。本指南以 [Claude](https://claude.ai/) 为例演示如何连接远程 MCP 服务器；它是[众多支持 MCP 的客户端之一](/zh/clients)。虽然我们重点介绍通过自定义连接器（Custom Connectors）集成 Claude，但这些概念同样适用于其他兼容 MCP 的客户端。

<div id="understanding-remote-mcp-servers">
  ## 了解远程 MCP 服务器
</div>

远程 MCP 服务器与本地 MCP 服务器的工作方式相似，但托管在互联网上，而不是你的本地设备上。它们对外提供工具、提示模板和资源，Claude 可以利用它们代你执行任务。这些服务器可与各种服务集成，例如项目管理工具、文档系统、代码仓库，以及任何其他支持 API 的服务。

远程 MCP 服务器的主要优势在于其可达性。与需要在每台设备上安装和配置的本地服务器不同，只要有互联网连接，任何 MCP 客户端都可以访问远程服务器。这使它们非常适合基于 Web 的 AI 应用、注重易用性的集成，以及需要服务器端处理或身份验证的服务。

<div id="what-are-custom-connectors">
  ## 什么是自定义连接器（Claude）？
</div>

自定义连接器（Claude）是 Claude 与远程 MCP 服务器之间的桥梁。它们使你可以将 Claude 直接连接到与你工作流程最相关的工具和数据源，让 Claude 在你常用的软件中工作，并从外部工具的完整上下文中获取洞见。

借助自定义连接器（Claude），你可以：

* [将 Claude 连接到由第三方开发者提供的现有远程 MCP 服务器](https://support.anthropic.com/en/articles/11175166-getting-started-with-custom-connectors-using-remote-mcp)
* [构建你自己的远程 MCP 服务器以连接任何工具](https://support.anthropic.com/en/articles/11503834-building-custom-connectors-via-remote-mcp-servers)

<div id="connecting-to-a-remote-mcp-server">
  ## 连接到远程 MCP 服务器
</div>

将 Claude 连接到远程 MCP 服务器的过程需要通过 [Claude 界面](https://claude.ai/)添加一个自定义连接器。这会在 Claude 与你选择的远程服务器之间建立一条安全连接。

<Steps>
  <Step title="进入连接器设置">
    在浏览器中打开 Claude，进入设置页面。点击个人头像，在下拉菜单中选择“Settings”。进入设置后，找到并点击侧边栏中的“Connectors”部分。

    这里会显示当前已配置的连接器，并提供添加新连接器的选项。
  </Step>

  <Step title="添加自定义连接器">
    在 Connectors 部分滚动到页面底部，找到“Add custom connector”按钮。点击该按钮开始连接流程。

    <Frame>
      <img src="/images/quickstart-remote/1-add-connector.png" alt="Claude 设置中的“Add custom connector”按钮" />
    </Frame>

    将弹出一个对话框，提示你输入远程 MCP 服务器的 URL。该 URL 应由服务器开发者或管理员提供。请输入完整的 URL，确保包含正确的协议（https://）以及必要的路径部分。

    <Frame>
      <img src="/images/quickstart-remote/2-connect.png" alt="输入远程 MCP 服务器 URL 的对话框" />
    </Frame>

    输入 URL 后，点击“Add”继续连接。
  </Step>

  <Step title="完成身份验证">
    大多数远程 MCP 服务器需要身份验证以确保对其资源的安全访问。具体流程因服务器实现而异，但通常涉及 OAuth、API 密钥或用户名/密码组合。

    <Frame>
      <img src="/images/quickstart-remote/3-auth.png" alt="远程 MCP 服务器的身份验证界面" />
    </Frame>

    按照服务器提供的身份验证提示进行操作。你可能会被重定向到第三方身份验证提供商，或在 Claude 内看到一个表单。完成身份验证后，Claude 将与远程服务器建立安全连接。
  </Step>

  <Step title="访问资源与提示模板">
    连接成功后，远程服务器的资源与提示模板会在你的 Claude 对话中可用。点击消息输入区域的回形针图标打开附件菜单即可访问。

    <Frame>
      <img src="/images/quickstart-remote/4-select-resources-menu.png" alt="显示可用资源的附件菜单" />
    </Frame>

    该菜单会显示来自已连接服务器的所有可用资源与提示模板。选择你希望包含在对话中的项目。这些资源将为 Claude 提供来自外部工具的上下文与信息。

    <Frame>
      <img src="/images/quickstart-remote/5-select-prompts-resources.png" alt="从菜单中选择特定资源与提示模板" />
    </Frame>
  </Step>

  <Step title="配置工具权限">
    远程 MCP 服务器通常会提供多个功能各异的工具。你可以在连接器设置中配置权限，控制 Claude 被允许使用哪些工具，从而确保它只执行你明确授权的操作。

    <Frame>
      <img src="/images/quickstart-remote/6-configure-tools.png" alt="工具权限配置界面" />
    </Frame>

    返回 Connectors 设置并点击你已连接的服务器。在此处，你可以启用或禁用特定工具、设置使用限制，并根据需要配置其他安全参数。
  </Step>
</Steps>

<div id="best-practices-for-using-remote-mcp-servers">
  ## 使用远程 MCP 服务器的最佳实践
</div>

在使用远程 MCP 服务器时，请参考以下建议，以确保安全、高效的使用体验：

**安全注意事项**：在连接之前务必验证远程 MCP 服务器的真实性。仅连接来自可信来源的服务器，并在认证过程中仔细审查其请求的权限。对授予敏感数据或系统的访问权限需格外谨慎。

**管理多个连接器**：你可以同时连接多个远程 MCP 服务器。按用途或项目对连接器进行分类管理，以保持清晰。定期审查并移除不再使用的连接器，保持工作区整洁与安全。
