模型上下文协议（MCP）提供一个可选的 ping 机制，使任一方都能检查对端是否仍然响应、连接是否保持畅通。

<div id="overview">
  ## 概览
</div>

ping 功能通过简单的请求/响应模式实现。客户端或服务器都可以通过发送 `ping` 请求来发起一次 ping。

<div id="message-format">
  ## 消息格式
</div>

ping 请求是一个不带参数的标准 JSON-RPC 请求：

```json
{
  "jsonrpc": "2.0",
  "id": "123",
  "method": "ping"
}
```

<div id="behavior-requirements">
  ## 行为要求
</div>

1. 接收方**必须（MUST）**及时返回一个空响应：

```json
{
  "jsonrpc": "2.0",
  "id": "123",
  "result": {}
}
```

2. 如果在合理的超时时间内未收到响应，发送方**可以（MAY）**：
   - 视连接为失效
   - 终止该连接
   - 尝试执行重连流程

<div id="usage-patterns">
  ## 使用模式
</div>

```mermaid
sequenceDiagram
    participant 发送方
    participant 接收方

    发送方->>接收方: ping 请求
    接收方->>发送方: 空响应
```

<div id="implementation-considerations">
  ## 实现注意事项
</div>

- 实现**应当**定期发送 ping 以检测连接状态
- ping 的频率**应当**可配置
- 超时**应当**根据网络环境合理设置
- 为降低网络开销，**应当**避免过于频繁的 ping

<div id="error-handling">
  ## 错误处理
</div>

- 超时**应**视为连接失败
- 多次 ping 失败**可**触发连接重置
- 实现**应**记录 ping 失败以便诊断
