<Info>**协议修订**：2025-03-26</Info>

模型上下文协议（MCP）由多个关键组件协同构成：

* **基础协议**：核心 JSON-RPC 消息类型
* **生命周期管理**：连接初始化、能力协商与会话控制
* **服务器功能**：MCP 服务器暴露的资源、提示模板和工具
* **客户端功能**：MCP 客户端提供的采样和根路径列表
* **通用实用程序**：日志记录、参数补全等跨领域关注点

所有实现**必须**支持基础协议和生命周期管理组件。其他组件**可以**根据应用的具体需求选择性实现。

这些协议层在实现丰富的客户端与服务器交互的同时，确立了清晰的关注点分离。模块化设计使实现可以精确支持所需的功能。

<div id="messages">
  ## 消息
</div>

MCP 客户端与服务器之间的所有消息**必须**遵循
[JSON-RPC 2.0](https://www.jsonrpc.org/specification) 规范。该协议定义了以下几类消息：

<div id="requests">
  ### 请求
</div>

请求可以由客户端发送到服务器，或由服务器发送到客户端，用于发起一次操作。

```typescript
{
  jsonrpc: "2.0";
  id: string | number;
  method: string;
  params?: {
    [key: string]: unknown;
  };
}
```

* 请求**必须**包含字符串或整数类型的 ID。
* 与基础 JSON-RPC 不同，ID **不得**为 `null`。
* 在同一会话中，请求方**不得**重复使用任何先前已使用的请求 ID。

<div id="responses">
  ### 响应
</div>

响应是对请求的回复，包含操作的结果或错误。

```typescript
{
  jsonrpc: "2.0";
  id: string | number;
  result?: {
    [key: string]: unknown;
  }
  error?: {
    code: number;
    message: string;
    data?: unknown;
  }
}
```

* 响应**必须**包含与其对应请求相同的 ID。
* **响应**进一步分为**成功结果**或**错误**两类。必须设置 `result` 或 `error` 之一。响应**不得**同时设置二者。
* 结果**可以**采用任意 JSON 对象结构，而错误**必须**至少包含错误代码和消息。
* 错误代码**必须**为整数。

<div id="notifications">
  ### 通知
</div>

通知可以由客户端发送至服务器，或相反方向发送，属于单向消息。
接收方**不得**发送响应。

```typescript
{
  jsonrpc: "2.0";
  method: string;
  params?: {
    [key: string]: unknown;
  };
}
```

* 通知**不得**包含 ID。

<div id="batching">
  ### 批处理
</div>

JSON-RPC 还定义了
[将多个请求和通知打包](https://www.jsonrpc.org/specification#batch)
在一个数组中发送的方式。MCP 实现**可以（MAY）**支持发送 JSON-RPC 批处理，但**必须（MUST）**支持接收 JSON-RPC 批处理。

<div id="auth">
  ## 身份验证
</div>

MCP 为基于 HTTP 的场景提供了一个[授权](/zh/specification/2025-03-26/basic/authorization)框架。
采用基于 HTTP 的传输方式的实现**应当**遵循此规范，
而采用 STDIO 传输方式的实现**不应**遵循此规范，
而是应从环境中获取凭据。

此外，客户端和服务器**可以**协商各自的自定义身份验证与
授权策略。

如需进一步讨论并为 MCP 身份验证机制的演进做出贡献，欢迎加入
[GitHub Discussions](https://github.com/modelcontextprotocol/specification/discussions)
共同塑造该协议的未来！

<div id="schema">
  ## 模式
</div>

该协议的完整规范定义为
[TypeScript 架构](https://github.com/modelcontextprotocol/specification/blob/main/schema/2025-03-26/schema.ts)。
这是所有协议消息和结构的权威来源。

另有一个
[JSON 架构](https://github.com/modelcontextprotocol/specification/blob/main/schema/2025-03-26/schema.json)，
它由上述 TypeScript 权威源自动生成，供各类自动化工具使用。
