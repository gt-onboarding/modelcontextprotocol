title: 提示模板

<div id="enable-section-numbers" />

<Info>**协议修订**：2025-06-18</Info>

模型上下文协议（MCP）为服务器向客户端公开提示模板提供了标准化方式。通过提示模板，服务器可以提供面向语言模型交互的结构化消息与指令。客户端可以发现可用的提示模板、获取其内容，并通过传入参数进行自定义。

<div id="user-interaction-model">
  ## 用户交互模型
</div>

提示模板被设计为由用户掌控，这意味着它们由服务器暴露给客户端，目的是让用户可以明确选择并使用它们。

通常，提示模板会通过用户在界面中主动发起的命令来触发，便于用户自然地发现并调用可用的提示模板。

例如，作为斜杠命令：

![作为斜杠命令暴露的提示模板示例](/specification/2025-06-18/server/slash-command.png)

不过，实现者可以根据自身需求，通过任何合适的界面模式来暴露提示模板——协议本身并不强制规定任何特定的用户交互模型。

<div id="capabilities">
  ## 功能
</div>

支持提示模板的服务器在
[初始化](/zh/specification/2025-06-18/basic/lifecycle#initialization)期间**必须**声明 `prompts` 功能：

```json
{
  "capabilities": {
    "prompts": {
      "listChanged": true
    }
  }
}
```

`listChanged` 表示当可用提示模板列表发生变化时，服务器是否会发送通知。

<div id="protocol-messages">
  ## 协议消息
</div>

<div id="listing-prompts">
  ### 列出提示模板
</div>

要获取可用的提示模板，客户端需发送 `prompts/list` 请求。此操作支持[分页](/zh/specification/2025-06-18/server/utilities/pagination)。

**请求：**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "prompts/list",
  "params": {
    "cursor": "optional-cursor-value"
  }
}
```

**响应：**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "prompts": [
      {
        "name": "code_review",
        "title": "请求代码评审",
        "description": "让 LLM 分析代码质量并提出改进建议",
        "arguments": [
          {
            "name": "code",
            "description": "待评审的代码",
            "required": true
          }
        ]
      }
    ],
    "nextCursor": "next-page-cursor"
  }
}
```

<div id="getting-a-prompt">
  ### 获取提示模板
</div>

要获取特定的提示模板，客户端需发送 `prompts/get` 请求。参数可通过[补全 API](/zh/specification/2025-06-18/server/utilities/completion)自动补全。

**请求：**

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "prompts/get",
  "params": {
    "name": "code_review",
    "arguments": {
      "code": "def hello():\n    print('world')"
    }
  }
}
```

**响应：**

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "description": "代码审查提示模板",
    "messages": [
      {
        "role": "user",
        "content": {
          "type": "text",
          "text": "请审查以下 Python 代码：\ndef hello():\n    print('world')"
        }
      }
    ]
  }
}
```

<div id="list-changed-notification">
  ### 列表变更通知
</div>

当可用的提示模板列表发生变化时，声明了 `listChanged`
能力的服务器**应**发送一条通知：

```json
{
  "jsonrpc": "2.0",
  "method": "notifications/prompts/list_changed"
}
```

<div id="data-types">
  ## 数据类型
</div>

<div id="prompt">
  ### 提示模板
</div>

提示模板的定义包括：

* `name`: 提示模板的唯一标识符
* `title`: 用于显示的可选的人类可读名称
* `description`: 可选的人类可读描述
* `arguments`: 可选的参数列表，用于自定义

<div id="promptmessage">
  ### PromptMessage
</div>

提示中的消息可以包含：

* `role`: 使用 &quot;user&quot; 或 &quot;assistant&quot; 表示发言者
* `content`: 以下任一内容类型：

<Note>
  提示消息中的所有内容类型均支持可选
  [annotations](/zh/specification/2025-06-18/server/resources#annotations)，用于标注受众、优先级和修改时间等元数据。
</Note>

<div id="text-content">
  #### 文本内容
</div>

文本内容表示纯文本消息：

```json
{
  "type": "text",
  "text": "The text content of the message"
}
```

这是自然语言交互中最常用的内容类型。

<div id="image-content">
  #### 图像内容
</div>

图像内容允许在消息中包含视觉信息：

```json
{
  "type": "image",
  "data": "base64-encoded-image-data",
  "mimeType": "image/png"
}
```

图像数据**必须**使用 base64 编码，并包含有效的 MIME 类型。这样就能在需要视觉上下文时进行多模态交互。

<div id="audio-content">
  #### 音频内容
</div>

音频内容允许在消息中包含音频信息：

```json
{
  "type": "audio",
  "data": "base64-encoded-audio-data",
  "mimeType": "audio/wav"
}
```

音频数据必须采用 base64 编码，并且包含有效的 MIME 类型。这使得在音频上下文重要的场景下实现多模态交互成为可能。

<div id="embedded-resources">
  #### 内嵌资源
</div>

内嵌资源允许在消息中直接引用服务器端资源：

```json
{
  "type": "resource",
  "resource": {
    "uri": "resource://example",
    "name": "example",
    "title": "My Example Resource",
    "mimeType": "text/plain",
    "text": "Resource content"
  }
}
```

资源可以包含文本或二进制（blob）数据，并且**必须**包含：

* 有效的资源 URI
* 适当的 MIME 类型
* 文本内容或 base64 编码的 blob 数据（二选一）

内嵌资源使提示模板能够将服务器管理的内容（例如文档、代码示例或其他参考材料）无缝融入对话流程。

<div id="error-handling">
  ## 错误处理
</div>

服务器**应**在常见失败情况下返回标准的 JSON-RPC 错误：

* 提示模板名称无效：`-32602`（参数无效）
* 缺少必填参数：`-32602`（参数无效）
* 内部错误：`-32603`（内部错误）

<div id="implementation-considerations">
  ## 实施注意事项
</div>

1. 服务器在处理前**应当**先验证提示模板的参数
2. 客户端**应当**为大型提示模板列表实现分页处理
3. 双方**应当**遵守能力协商机制

<div id="security">
  ## 安全
</div>

实现**必须**严格校验所有提示模板的输入与输出，以防止注入攻击或未授权访问资源。
