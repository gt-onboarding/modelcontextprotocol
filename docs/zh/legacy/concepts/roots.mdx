---
title: "根路径"
description: "理解 MCP 中的根路径"
---

根路径是 MCP 中用于限定服务器操作范围的概念。它提供了一种机制，使客户端可以告知服务器相关资源及其位置。

<div id="what-are-roots">
  ## 什么是根路径？
</div>

根路径是客户端建议服务器重点关注的 URI。客户端连接到服务器时，会声明服务器应当使用哪些根路径。虽然根路径主要用于文件系统路径，但它也可以是任何有效的 URI，包括 HTTP URL。

例如，根路径可以是：

```
file:///home/user/projects/myapp
https://api.example.com/v1
```

<div id="why-use-roots">
  ## 为什么使用根路径？
</div>

根路径有几个重要用途：

1. **指引**：向服务器说明相关资源及其位置
2. **清晰**：明确哪些资源属于你的工作区
3. **组织**：通过多个根路径可同时处理不同资源

<div id="how-roots-work">
  ## 根路径的工作方式
</div>

当客户端支持根路径时，将：

1. 在建立连接时声明 `roots` 能力
2. 向服务器提供建议的根路径列表
3. 在根路径发生变化时通知服务器（若支持）

尽管根路径仅作参考而非强制约束，服务器应当：

1. 遵循提供的根路径
2. 使用根路径 URI 定位并访问资源
3. 优先在根路径范围内执行操作

<div id="common-use-cases">
  ## 常见用例
</div>

根路径通常用于定义：

- 项目目录
- 代码仓库位置
- API 端点
- 配置位置
- 资源边界

<div id="best-practices">
  ## 最佳实践
</div>

在使用根路径时：

1. 仅建议必要的资源
2. 为根路径使用清晰、明确的名称
3. 监控根路径的可访问性
4. 以平滑方式处理根路径的变更

<div id="example">
  ## 示例
</div>

以下展示了一个典型的 MCP 客户端如何公开根路径：

```json
{
  "roots": [
    {
      "uri": "file:///home/user/projects/frontend",
      "name": "Frontend Repository"
    },
    {
      "uri": "https://api.example.com/v1",
      "name": "API Endpoint"
    }
  ]
}
```

此配置表示服务器应同时聚焦于本地代码仓库和一个 API 端点，并在逻辑上将二者区分开来。
