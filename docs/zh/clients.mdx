<div id="5ire">
  ### 5ire
</div>

[5ire](https://github.com/nanbingxyz/5ire) 是一款开源、跨平台的桌面 AI 助手，可通过 MCP 服务器使用工具（Tools）。

**关键特性：**

- 内置 MCP 服务器，可一键启用或禁用。
- 支持通过修改配置文件添加更多服务器。
- 开源且上手友好，适合初学者。
- 将持续完善对 MCP 的支持。

<div id="agentai">
  ### AgentAI
</div>

[AgentAI](https://github.com/AdamStrojek/rust-agentai) 是一个用于简化构建 AI 代理的 Rust 库。该库提供与 MCP 服务器的无缝集成。

[MCP 服务器集成示例](https://github.com/AdamStrojek/rust-agentai/blob/master/examples/tools_mcp.rs)

**关键特性：**

- 多 LLM —— 支持大多数 LLM API（OpenAI、Anthropic、Gemini、Ollama，以及所有兼容 OpenAI API 的实现）。
- 内置支持 MCP 服务器。
- 使用 Rust 这类类型安全、内存安全的语言创建代理流程。

<div id="agenticflow">
  ### AgenticFlow
</div>

[AgenticFlow](https://agenticflow.ai/) 是一个零代码 AI 平台，帮助你构建可全天候处理销售、营销和创意任务的代理。通过模型上下文协议（MCP）安全连接 2,500+ 个 API 和 10,000+ 个工具（Tools）。

**关键功能：**

- 零代码创建 AI 代理并搭建工作流。
- 通过 MCP 访问包含 10,000+ 个工具（Tools）和 2,500+ 个 API 的庞大库。
- 以简单的 3 步流程连接 MCP 服务器（MCP 服务器）。
- 安全管理连接，可随时撤销访问权限。

**了解更多：**

- [AgenticFlow MCP 集成](https://agenticflow.ai/mcp)

<div id="aiql-tuui">
  ### AIQL TUUI
</div>

[AIQL TUUI] 是一款原生、跨平台的桌面 AI 聊天应用，支持 模型上下文协议（MCP）。它兼容多家 AI 提供商（如 Anthropic、Cloudflare、DeepSeek、OpenAI、Qwen），本地 AI 模型（通过 vLLM、Ray 等），以及聚合 API 平台（如 Deepinfra、OpenRouter 等）。

**核心特性：**

- **动态 LLM API 与智能体切换**：可在不同的 LLM API 与智能体之间即时无缝切换。
- **全面能力支持**：内置支持工具（Tools）、提示（Prompts）、资源（Resources）和采样（Sampling）方法。
- **可配置智能体**：通过智能体设置选择并自定义工具，获得更高的灵活性。
- **高级采样控制**：可调整采样参数，并利用多轮采样获得更优结果。
- **跨平台兼容性**：完全兼容 macOS、Windows 与 Linux。
- **免费且开源（FOSS）**：宽松许可，支持修改与自定义打包应用。

**了解更多：**

- [TUUI 文档](https://www.tuui.com/)
- [AIQL GitHub 仓库](https://github.com/AI-QL)

<div id="amazon-q-cli">
  ### Amazon Q CLI
</div>

[Amazon Q CLI](https://github.com/aws/amazon-q-developer-cli) 是一款开源的终端智能编码助手，具备代理式能力。

**主要特性：**

- 对 MCP 服务器提供完整支持。
- 使用你偏好的文本编辑器编辑提示（Prompts）。
- 通过 `@` 即刻访问已保存的提示（Prompts）。
- 直接在终端中控制和组织 AWS 资源（Resources）。
- 工具（Tools）、配置文件、上下文管理、自动压缩等更多功能！

**开始使用**

```bash
brew install amazon-q
```

<div id="amazon-q-ide">
  ### Amazon Q IDE
</div>

[Amazon Q IDE](https://aws.amazon.com/q/developer) 是一款面向 IDE 的开源“代理式”编码助手。

**主要功能：**

- 支持 VSCode、JetBrains、Visual Studio 和 Eclipse 等 IDE。
- 可在 IDE 中直接控制并管理 AWS 资源。
- 通过 IDE 的用户界面为每个 MCP 工具管理权限。

<div id="apify-mcp-tester">
  ### Apify MCP Tester
</div>

[Apify MCP Tester](https://github.com/apify/tester-mcp-client) 是一个开源 MCP 客户端，使用服务器发送事件（SSE）连接到任意 MCP 服务器。
它是一个独立的 Apify Actor，专为通过 SSE 测试 MCP 服务器而设计，支持 Authorization 头。
它采用纯 JavaScript（传统风格）实现，并托管在 Apify 上，无需任何配置即可运行。

**主要功能：**

- 通过 SSE 连接任意 MCP 服务器。
- 可与 [Apify MCP Server](https://apify.com/apify/actors-mcp-server) 协同工作，以与一个或多个 Apify [Actors](https://apify.com/store) 进行交互。
- 可根据上下文和用户查询动态调用工具（Tools）（若服务器支持）。

<div id="amp">

[VS Code](https://code.visualstudio.com/) 通过 [agent mode](https://code.visualstudio.com/docs/copilot/chat/chat-agent-mode) 将 模型上下文协议（MCP） 与 GitHub Copilot 集成，使你能够在代理式编码工作流中直接使用 MCP 提供的 工具（Tools）。可在 Claude Desktop、工作区或用户设置中配置服务器，提供引导式 MCP 安装，并在输入变量中安全处理密钥，避免泄露硬编码密钥。

**关键特性：**

- 支持 stdio 和 服务器发送事件（SSE） 传输方式
- 按会话为每个代理会话选择 工具（Tools），以获得最佳性能
- 通过重启命令与输出日志轻松调试服务器
- 工具调用支持可编辑输入，并提供“始终允许”开关
- 与现有 VS Code 扩展体系集成，可从扩展中注册 MCP 服务器

- **可配置启动**：可将 MCP 服务器设为随 Warp 自动启动，或按需手动启动
