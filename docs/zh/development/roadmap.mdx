模型上下文协议（MCP）正在快速演进。此页面概述了我们针对未来约**六个月**的关键优先事项与方向的最新思考；随着项目推进，这些内容可能会发生较大变化。要查看近期变更，请查阅**[规范更新日志](/zh/specification/2025-06-18/changelog/)**。

<Note>
  此处提出的想法并非承诺——我们可能会以与文中描述不同的方式解决这些挑战，或其中部分最终并不会落地。这也不是一份_穷尽_清单；我们可能会纳入此处未提及的工作。
</Note>

我们重视社区参与！各个部分都链接到相关讨论，您可以在其中进一步了解并分享见解。

若想从技术角度了解我们的标准化流程，请访问 GitHub 上的 [Standards Track](https://github.com/orgs/modelcontextprotocol/projects/2/views/2)，该视图追踪提案如何推进，直至纳入官方的 [MCP 规范](https://spec.modelcontextprotocol.io)。

<div id="agents">
  ## 代理
</div>

随着 MCP 日益融入代理式工作流，我们正专注于以下关键改进：

* **异步操作**：支持可能持续较久的长时运行任务，并能稳健地处理断线与重连

<div id="authentication-and-security">
  ## 身份验证与安全
</div>

我们正在持续完善授权与安全相关的资源，以提升用户安全性并为开发者提供更佳的体验：

* **指南与最佳实践**：通过指南与最佳实践详述如何安全部署 MCP，帮助开发者避免常见陷阱。
* **动态客户端注册（DCR）的替代方案**：探索 DCR 的替代方案，在兼顾顺畅用户体验的同时应对运维挑战。
* **细粒度授权**：为敏感操作制定更精细的基础授权机制与规范。
* **企业托管授权**：借助单点登录（SSO），为企业提供简化 MCP 服务器授权的能力。
* **安全授权引导输入**：使开发者能够在主 MCP 服务器授权之外，为下游 API 集成安全的授权流程。

<div id="validation">
  ## 验证
</div>

为打造坚实的开发者生态，我们计划投入：

* **参考客户端实现**：以高质量的 AI 应用演示协议功能
* **参考服务器实现**：展示认证模式与远程部署最佳实践
* **合规性测试套件**：自动化验证客户端、服务器和 SDK 对规范的正确实现

这些工具将帮助开发者自信地实现模型上下文协议（MCP），同时确保整个生态系统中的行为一致。

<div id="registry">
  ## 注册表
</div>

为使 MCP 充分发挥潜力，我们需要更顺畅的方式来分发和发现 MCP 服务器。

我们计划开发一个[**MCP Registry**](https://github.com/orgs/modelcontextprotocol/discussions/159)，用于实现集中式的服务器发现与元数据管理。该注册表主要充当 API 层，供第三方市场与发现服务在其之上构建。

<div id="multimodality">
  ## 多模态
</div>

在 MCP 中支持全谱系的 AI 能力，包括：

* **附加模态**：视频及其他媒体类型
* **[流式处理](https://github.com/modelcontextprotocol/specification/issues/117)**：多部分与分块消息，以及用于交互式体验的双向通信
