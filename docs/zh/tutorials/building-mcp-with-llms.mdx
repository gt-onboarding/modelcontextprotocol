本指南将帮助你利用 LLM 构建自定义的模型上下文协议（MCP）服务器和客户端。我们将在本教程中重点介绍 Claude，但你也可以使用任何前沿 LLM 来实现同样的目标。

<div id="preparing-the-documentation">
  ## 准备文档
</div>

在开始之前，收集必要的文档以帮助 Claude 理解模型上下文协议（MCP）：

1. 访问 https://modelcontextprotocol.io/llms-full.txt 并复制完整的文档文本
2. 前往 [MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) 或 [Python SDK 仓库](https://github.com/modelcontextprotocol/python-sdk)
3. 复制 README 文件及其他相关文档
4. 将这些文档粘贴到你与 Claude 的对话中

<div id="describing-your-server">
  ## 描述你的服务器
</div>

在提供完文档后，清晰地向 Claude 描述你想构建的服务器类型。请具体说明：

- 你的服务器将暴露哪些资源
- 它将提供哪些工具
- 它应提供哪些提示
- 它需要与哪些外部系统交互

例如：

```
构建一个 MCP 服务器，能够：
- 连接到我公司使用的 PostgreSQL 数据库
- 将数据表的模式作为资源对外暴露
- 提供用于运行只读 SQL 查询的工具
- 包含用于常见数据分析任务的提示
```

<div id="working-with-claude">
  ## 与 Claude 协作
</div>

在 MCP 服务器上与 Claude 协作时：

1. 先从核心功能入手，再迭代添加更多功能
2. 请 Claude 解释你不理解的代码部分
3. 按需请求修改或优化
4. 让 Claude 协助测试服务器并处理边界情况

Claude 可以帮助实现所有关键的 MCP 功能：

- 资源管理与公开
- 工具的定义与实现
- 提示模板与处理程序
- 错误处理与日志记录
- 连接与传输方式的设置

<div id="best-practices">
  ## 最佳实践
</div>

在使用 Claude 构建 MCP 服务器时：

- 将复杂服务器拆分为更小的模块
- 在推进到下一步前彻底测试每个组件
- 时刻关注安全性——校验输入并合理限制访问
- 编写完善的代码文档，便于后续维护
- 严格遵循 MCP 协议规范

<div id="next-steps">
  ## 后续步骤
</div>

在 Claude 帮助你构建服务器之后：

1. 仔细检查生成的代码
2. 使用 MCP Inspector 工具测试服务器
3. 将其连接到 Claude.app 或其他 MCP 客户端
4. 根据实际使用情况和反馈持续迭代

请记住，随着需求变化，Claude 可以随时帮助你修改并改进你的服务器。

需要更多指导？直接向 Claude 询问有关实现 MCP 功能或排查相关问题的具体问题即可。
