<div id="features">
  ## 功能
</div>

* MCP 客户端和 MCP 服务器的实现支持：
  * 协议[版本兼容性协商](/zh/specification/2025-06-18/basic/lifecycle#initialization)
  * [工具](/zh/specification/2025-06-18/server/tools/) 发现、执行和列表变更通知
  * 基于 URI 模板的[资源](/zh/specification/2025-06-18/server/resources/) 管理
  * [提示模板](/zh/specification/2025-06-18/server/prompts/) 处理与管理
  * 为提示模板和资源 URI 提供[补全](/zh/specification/2025-06-18/server/utilities/completion/) 参数自动补全建议
  * 针对长时运行操作的[进度](/zh/specification/2025-06-18/basic/utilities/progress/) 跟踪
  * [Ping](/zh/specification/2025-06-18/basic/utilities/ping/) 轻量级健康检查机制
    * [服务器 Keepalive](/zh/specification/2025-06-18/basic/utilities/ping#implementation-considerations/) 用于维持活跃的服务器连接
  * 用于向客户端发送结构化日志消息的[日志记录](/zh/specification/2025-06-18/server/utilities/logging/)
  * [根路径](/zh/specification/2025-06-18/client/roots/) 列表管理与通知
  * 支持 AI 模型交互的[采样](/zh/specification/2025-06-18/client/sampling/)
  * [信息征询](/zh/specification/2025-06-18/client/elicitation/) 便于服务器通过客户端向用户请求补充信息
* 多种传输方式实现：
  * 默认传输方式（包含在核心 `mcp` 模块中，无需外部 Web 框架）：
    * 基于 STDIO 的进程间通信传输
    * 基于 Java HttpClient 的 `SSE` 与 `Streamable-HTTP` 客户端传输
    * 基于 Servlet 的 `SSE` 与 `Streamable-HTTP` 服务器传输
  * 可选的基于 Spring 的传输方式（在使用 Spring Framework 时更便捷）：
    * WebFlux `SSE` 与 `Streamable-HTTP` 客户端与服务器传输
    * WebMVC `SSE` 与 `Streamable-HTTP` 传输，用于基于 Servlet 的 HTTP 流式传输
* 同步与异步编程范式均受支持

<Tip>
  核心 `io.modelcontextprotocol.sdk:mcp` 模块提供默认的 `STDIO`、`SSE` 与 `Streamable-HTTP` 客户端和服务器传输实现，无需外部 Web 框架。

  在使用 [Spring AI](https://docs.spring.io/spring-ai/reference/1.1-SNAPSHOT/api/mcp/mcp-overview.html) Framework 时，可按需引入基于 Spring 的传输作为可选依赖，便于集成。
</Tip>

<div id="architecture">
  ## 架构
</div>

该 SDK 采用分层架构，明确分离关注点：

{/* ![MCP Stack Architecture](/images/java/mcp-stack.svg) */}

* **客户端/服务器层（McpClient/McpServer）**：二者均使用 McpSession 进行同步/异步操作，
  其中 McpClient 负责客户端侧的协议操作，McpServer 负责服务器侧的协议操作。
* **会话层（McpSession）**：通过 DefaultMcpSession 实现管理通信模式与状态。
* **传输层（McpTransport）**：通过以下方式处理 JSON-RPC 消息的序列化/反序列化：
  * 核心模块中的 StdioTransport（stdin/stdout）
  * 独立传输模块中的 HTTP“可流式 HTTP”和“SSE”传输（Java HttpClient、Spring WebFlux、Spring WebMVC）

[MCP 客户端](/zh/sdk/java/mcp-client) 是模型上下文协议（MCP）架构中的关键组件，负责与 MCP 服务器建立并管理连接。
它实现了协议的客户端侧。

![Java MCP Client Architecture](/images/java/java-mcp-client-architecture.jpg)

[MCP 服务器](/zh/sdk/java/mcp-server) 是模型上下文协议（MCP）架构中的基础组件，为客户端提供工具、资源和能力。
它实现了协议的服务器侧。

![Java MCP Server Architecture](/images/java/java-mcp-server-architecture.jpg)

关键交互：

* **客户端/服务器初始化**：传输设置、协议兼容性检查、能力协商与实现细节交换。
* **消息流**：带验证的 JSON-RPC 消息处理、类型安全的响应处理与错误处理。
* **资源管理**：资源发现、基于 URI 模板的访问、订阅机制与内容检索。

<div id="dependencies">
  ## 依赖项
</div>

将以下依赖添加到你的项目中：

<Tabs>
  <Tab title="Maven">
    核心 MCP 功能：

    ```xml
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp</artifactId>
    </dependency>
    ```

    核心 `mcp` 模块已内置默认的 `STDIO`、`SSE` 和 `Streamable-HTTP` 传输方式实现，无需依赖外部 Web 框架。

    如果你使用 Spring Framework，并希望采用基于 Spring 的传输方式实现，请添加以下可选依赖之一：

    ```xml
    <!-- 可选：基于 Spring WebFlux 的 SSE 和 Streamable-HTTP 客户端与服务器传输 -->
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp-spring-webflux</artifactId>
    </dependency>

    <!-- 可选：基于 Spring WebMVC 的 SSE 和 Streamable-HTTP 服务器传输 -->
    <dependency>
        <groupId>io.modelcontextprotocol.sdk</groupId>
        <artifactId>mcp-spring-webmvc</artifactId>
    </dependency>
    ```
  </Tab>

  <Tab title="Gradle">
    核心 MCP 功能：

    ```groovy
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp")
      //...
    }
    ```

    核心 `mcp` 模块已内置默认的 `STDIO`、`SSE` 和 `Streamable-HTTP` 传输方式实现，无需依赖外部 Web 框架。

    如果你使用 Spring Framework，并希望采用基于 Spring 的传输方式实现，请添加以下可选依赖之一：

    ```groovy
    // 可选：基于 Spring WebFlux 的 SSE 和 Streamable-HTTP 客户端与服务器传输
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-spring-webflux")
    }

    // 可选：基于 Spring WebMVC 的 SSE 和 Streamable-HTTP 服务器传输
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-spring-webmvc")
    }
    ```
  </Tab>
</Tabs>

* `io.modelcontextprotocol.sdk:mcp-spring-webflux` - 基于 WebFlux 的客户端与服务器，支持 `Streamable-HTTP` 与 `SSE` 传输方式的实现。
  WebFlux 实现可用于响应式应用；而基于 WebClient 的 MCP 客户端可同时用于响应式与命令式应用。
  该选项具有高可扩展性，适合并推荐用于高吞吐量场景。
* `io.modelcontextprotocol.sdk:mcp-spring-webmvc` - 基于 WebMVC 的服务器，为基于 servlet 的应用提供 `Streamable-HTTP` 与 `SSE` 传输方式实现。

<div id="bill-of-materials-bom">
  ### 物料清单（BOM）
</div>

物料清单（BOM）声明了某次发布中所使用的全部依赖项的推荐版本。
在应用的构建脚本中使用 BOM，可以免去你手动指定并维护依赖版本的工作。
你所采用的 BOM 版本将决定实际使用的各依赖项版本。
除非你选择覆盖，否则它还可确保默认使用受支持且经过测试的依赖版本。

将 BOM 添加到你的项目：

<Tabs>
  <Tab title="Maven">
    ```xml
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>io.modelcontextprotocol.sdk</groupId>
                <artifactId>mcp-bom</artifactId>
                <version>0.12.1</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
    ```
  </Tab>

  <Tab title="Gradle">
    ```groovy
    dependencies {
      implementation platform("io.modelcontextprotocol.sdk:mcp-bom:0.12.1")
      //...
    }
    ```

    Gradle 用户也可以借助 Gradle（5.0+）对基于 Maven BOM 的依赖约束的原生支持来使用 MCP BOM。
    这可以通过在 Gradle 构建脚本的 dependencies 部分添加一个 platform 依赖处理方法来实现。
    如上方代码片段所示，随后即可对希望使用的 MCP SDK 模块进行无版本声明。
  </Tab>
</Tabs>

将版本号替换为你希望使用的 BOM 版本。
