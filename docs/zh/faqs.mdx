<div id="what-is-mcp">
  ## 什么是 MCP？
</div>

MCP（Model Context Protocol）是一种标准，让 AI 应用和智能体能够连接并使用你的数据源（例如本地文件、数据库或内容仓库）和工具（例如 GitHub、Google Maps 或 Puppeteer）。

可以把 MCP 看作 AI 应用的通用适配器，类似于 USB-C 之于实体设备。USB-C 作为通用适配器，将设备连接到各类外设与配件。类似地，MCP 提供标准化方式，将 AI 应用连接到不同的数据与工具。

在 USB-C 出现之前，你需要为不同的接口准备不同的线缆。类似地，在 MCP 出现之前，开发者必须为每个希望让 AI 应用使用的数据源或工具构建自定义连接——这既耗时，又常常导致功能受限。现在，有了 MCP，开发者可以轻松为 AI 应用添加连接，让应用从第一天起就更为强大。

<div id="why-does-mcp-matter">
  ## 为什么 MCP 至关重要？
</div>

<div id="for-ai-application-users">
  ### 面向 AI 应用的用户
</div>

MCP 意味着你的 AI 应用可以访问你日常使用的信息和工具，从而更有帮助。AI 不再局限于它“已经知道”的内容，而是能够理解你的特定文档、数据和工作上下文。

例如，借助 MCP 服务器，应用可以从 Google Drive 访问你的个人文档，或从 GitHub 获取与你的代码库相关的数据，从而提供更个性化、更贴合上下文的帮助。

想象你向一位 AI 助手提出请求：“总结上周的团队会议纪要，并为所有人安排后续跟进。”

通过使用由 MCP 驱动的数据源连接，AI 助手可以：

* 通过一个 MCP 服务器连接到你的 Google Drive，读取会议纪要
* 基于纪要了解需要跟进的人
* 通过另一个 MCP 服务器连接到你的日历，自动安排会议

<div id="for-developers">
  ### 面向开发者
</div>

在构建需要访问多种数据源的 AI 应用时，MCP 能够减少开发时间和复杂度。借助 MCP，开发者可以专注于打造出色的 AI 体验，而不必反复创建自定义连接器。

传统上，将应用与数据源连接通常需要为每个数据源、每个应用分别构建一次性的自定义连接。这会导致大量重复性工作——每位希望将 AI 应用连接到 Google Drive 或 Slack 的开发者都得各自实现连接。

MCP 通过支持开发者为数据源构建可被多种应用复用的 MCP 服务器来简化这一过程。例如，借助开源的 Google Drive MCP 服务器，许多不同的应用都能访问 Google Drive 中的数据，而无需各位开发者分别构建自定义连接。

由开源 MCP 服务器组成的生态系统意味着，开发者可以复用现有成果，而非从零开始，从而更轻松地构建强大的 AI 应用，并与用户已依赖的工具和数据源无缝集成。

<div id="how-does-mcp-work">
  ## MCP 是如何运作的？
</div>

<Frame>
  <img src="/images/mcp-simple-diagram.png" />
</Frame>

MCP 通过一个简洁的系统，在你的 AI 应用与数据之间搭建桥梁：

* **MCP 服务器** 连接到你的数据源和工具（如 Google Drive 或 Slack）
* **MCP 客户端** 由 AI 应用（如 Claude Desktop）运行，用于将应用连接到这些服务器
* 在你授予权限后，AI 应用会发现可用的 MCP 服务器
* 随后，AI 模型可以通过这些连接读取信息并执行操作

这种模块化系统意味着无需修改 AI 应用本身即可添加新功能——就像为电脑添加新配件而无需升级整个系统。

<div id="who-creates-and-maintains-mcp-servers">
  ## 谁来创建和维护 MCP 服务器？
</div>

MCP 服务器由以下主体开发和维护：

* Anthropic 的开发者，为常见的工具和数据源构建服务器
* 开源贡献者，为他们使用的工具创建服务器
* 企业开发团队，为其内部系统构建服务器
* 软件提供商，让其应用面向 AI 场景做好准备

一旦为某个数据源创建了开源 MCP 服务器，任何与 MCP 兼容的 AI 应用都可以使用它，从而促成不断扩大的连接生态系统。参见我们的[示例服务器列表](/zh/examples)，或[开始构建你自己的服务器](/zh/quickstart/server)。
