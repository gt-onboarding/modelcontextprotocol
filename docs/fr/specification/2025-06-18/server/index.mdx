Les serveurs fournissent les éléments fondamentaux pour ajouter du contexte aux modèles de langage via le Protocole de contexte de modèle (MCP). Ces primitives permettent des interactions riches entre les clients, les serveurs et les modèles de langage :

* **Invites** : modèles ou instructions prédéfinis qui guident les interactions avec le modèle de langage
* **Ressources** : données ou contenus structurés qui apportent un contexte supplémentaire au modèle
* **Outils** : fonctions exécutables qui permettent aux modèles d’effectuer des actions ou de récupérer des informations

Chaque primitive peut être résumée dans la hiérarchie de contrôle suivante :

| Primitive | Contrôle                  | Description                                         | Exemple                          |
| --------- | ------------------------- | --------------------------------------------------- | -------------------------------- |
| Invites   | Contrôlé par l’utilisateur | Modèles interactifs invoqués au choix de l’utilisateur | Commandes slash, options de menu |
| Ressources | Contrôlé par l’application | Données contextuelles attachées et gérées par le client | Contenu de fichiers, historique Git |
| Outils    | Contrôlé par le modèle     | Fonctions exposées au LLM pour effectuer des actions  | Requêtes POST d’API, écriture de fichiers |
