Les serveurs fournissent les éléments fondamentaux permettant d’ajouter du contexte aux modèles de langage via le Protocole de contexte de modèle (MCP). Ces primitives permettent des interactions riches entre clients, serveurs et modèles de langage :

* **Invites** : modèles ou instructions prédéfinis qui guident les interactions avec le modèle de langage
* **Ressources** : données structurées ou contenus qui apportent un contexte supplémentaire au modèle
* **Outils** : fonctions exécutables qui permettent aux modèles d’effectuer des actions ou de récupérer des informations

Chaque primitive peut être résumée dans la hiérarchie de contrôle suivante :

| Primitive | Contrôle                    | Description                                           | Exemple                          |
| --------- | --------------------------- | ----------------------------------------------------- | -------------------------------- |
| Invites   | Contrôle utilisateur        | Modèles interactifs invoqués au choix de l’utilisateur | Commandes slash, options de menu |
| Ressources | Contrôle application       | Données contextuelles associées et gérées par le client | Contenu de fichiers, historique git |
| Outils    | Contrôle modèle             | Fonctions exposées au LLM pour réaliser des actions    | Requêtes POST d’API, écriture de fichiers |
