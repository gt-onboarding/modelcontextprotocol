title: "Invites"
description: "Créer des modèles d'invites et des workflows réutilisables"
---

Les Invites permettent aux serveurs de définir des modèles d'invites et des workflows réutilisables que les clients peuvent facilement mettre à disposition des utilisateurs et des LLM. Elles offrent un moyen puissant de standardiser et de partager des interactions courantes avec les LLM.

<Note>

Les Invites sont conçues pour être **contrôlées par l'utilisateur** : elles sont exposées par les serveurs aux clients afin que l'utilisateur puisse les sélectionner explicitement pour les utiliser.

</Note>

<div id="overview">
  ## Vue d’ensemble
</div>

Les Invites dans le MCP sont des modèles prédéfinis qui peuvent :

- Accepter des paramètres dynamiques
- Inclure du contexte issu de Ressources
- Enchaîner plusieurs interactions
- Guider des flux de travail spécifiques
- S’afficher comme des éléments d’interface (par exemple des commandes « slash »)

<div id="prompt-structure">
  ## Structure d’invite
</div>

Chaque invite est définie ainsi :

```typescript
{
  name: string;              // Identifiant unique de l’invite
  description?: string;      // Description lisible par l’humain
  arguments?: [              // Liste d’arguments facultative
    {
      name: string;          // Identifiant de l’argument
      description?: string;  // Description de l’argument
      required?: boolean;    // Indique si l’argument est obligatoire
    }
  ]
}
```

<div id="discovering-prompts">
  ## Découverte des invites
</div>

Les clients peuvent découvrir les invites disponibles en envoyant une requête `prompts/list` :

```typescript
// Requête
{
  method: "prompts/list";
}

// Réponse
{
  prompts: [
    {
      name: "analyze-code",
      description: "Analyser le code pour identifier des améliorations potentielles",
      arguments: [
        {
          name: "language",
          description: "Langage de programmation",
          required: true,
        },
      ],
    },
  ];
}
```

<div id="using-prompts">
  ## Utilisation des invites
</div>

Pour utiliser une invite, les clients effectuent une requête `prompts/get` :

````typescript
// Request
{
  method: "prompts/get",
  params: {
    name: "analyze-code",
    arguments: {
      language: "python"
    }
  }
}

// Response
{
  description: "Analyser du code Python afin d'identifier des améliorations possibles",
  messages: [
    {
      role: "user",
      content: {
        type: "text",
        text: "Veuillez analyser le code Python suivant afin d'identifier des améliorations possibles :\n\n```python\ndef calculate_sum(numbers):\n    total = 0\n    for num in numbers:\n        total = total + num\n    return total\n\nresult = calculate_sum([1, 2, 3, 4, 5])\nprint(result)\n```"
      }
    }
  ]
}
````

<div id="example-implementation">
  ## Exemple d’implémentation
</div>

Voici un exemple complet d’implémentation des Invites dans un serveur MCP :

<CodeGroup>

```typescript TypeScript
import { Server } from "@modelcontextprotocol/sdk/server";
import {
  ListPromptsRequestSchema,
  GetPromptRequestSchema,
} from "@modelcontextprotocol/sdk/types";

const PROMPTS = {
  "git-commit": {
    name: "git-commit",
    description: "Generate a Git commit message",
    arguments: [
      {
        name: "changes",
        description: "Git diff or description of changes",
        required: true,
      },
    ],
  },
  "explain-code": {
    name: "explain-code",
    description: "Explain how code works",
    arguments: [
      {
        name: "code",
        description: "Code to explain",
        required: true,
      },
      {
        name: "language",
        description: "Programming language",
        required: false,
      },
    ],
  },
};

const server = new Server(
  {
    name: "example-prompts-server",
    version: "1.0.0",
  },
  {
    capabilities: {
      prompts: {},
    },
  },
);

// Lister les invites disponibles
server.setRequestHandler(ListPromptsRequestSchema, async () => {
  return {
    prompts: Object.values(PROMPTS),
  };
});

// Récupérer une invite spécifique
server.setRequestHandler(GetPromptRequestSchema, async (request) => {
  const prompt = PROMPTS[request.params.name];
  if (!prompt) {
    throw new Error(`Prompt not found: ${request.params.name}`);
  }

  if (request.params.name === "git-commit") {
    return {
      messages: [
        {
          role: "user",
          content: {
            type: "text",
            text: `Generate a concise but descriptive commit message for these changes:\n\n${request.params.arguments?.changes}`,
          },
        },
      ],
    };
  }

  if (request.params.name === "explain-code") {
    const language = request.params.arguments?.language || "Unknown";
    return {
      messages: [
        {
          role: "user",
          content: {
            type: "text",
            text: `Explain how this ${language} code works:\n\n${request.params.arguments?.code}`,
          },
        },
      ],
    };
  }

  throw new Error("Prompt implementation not found");
});
```

```python Python
from mcp.server import Server
import mcp.types as types

# Définir les invites disponibles
PROMPTS = {
    "git-commit": types.Prompt(
        name="git-commit",
        description="Generate a Git commit message",
        arguments=[
            types.PromptArgument(
                name="changes",
                description="Git diff or description of changes",
                required=True
            )
        ],
    ),
    "explain-code": types.Prompt(
        name="explain-code",
        description="Explain how code works",
        arguments=[
            types.PromptArgument(
                name="code",
                description="Code to explain",
                required=True
            ),
            types.PromptArgument(
                name="language",
                description="Programming language",
                required=False
            )
        ],
    )
}

# Initialiser le serveur
app = Server("example-prompts-server")

@app.list_prompts()
async def list_prompts() -> list[types.Prompt]:
    return list(PROMPTS.values())

@app.get_prompt()
async def get_prompt(
    name: str, arguments: dict[str, str] | None = None
) -> types.GetPromptResult:
    if name not in PROMPTS:
        raise ValueError(f"Prompt not found: {name}")

    if name == "git-commit":
        changes = arguments.get("changes") if arguments else ""
        return types.GetPromptResult(
            messages=[
                types.PromptMessage(
                    role="user",
                    content=types.TextContent(
                        type="text",
                        text=f"Generate a concise but descriptive commit message "
                        f"for these changes:\n\n{changes}"
                    )
                )
            ]
        )

    if name == "explain-code":
        code = arguments.get("code") if arguments else ""
        language = arguments.get("language", "Unknown") if arguments else "Unknown"
        return types.GetPromptResult(
            messages=[
                types.PromptMessage(
                    role="user",
                    content=types.TextContent(
                        type="text",
                        text=f"Explain how this {language} code works:\n\n{code}"
                    )
                )
            ]
        )

    raise ValueError("Prompt implementation not found")
```

<div id="best-practices">
  ## Bonnes pratiques
</div>

Lors de l’implémentation des Invites :

1. Utilisez des noms d’Invite clairs et descriptifs
2. Fournissez des descriptions détaillées pour les Invites et leurs arguments
3. Validez tous les arguments requis
4. Gérez les arguments manquants avec élégance
5. Envisagez la gestion de versions pour les modèles d’Invite
6. Mettez en cache le contenu dynamique lorsque cela est pertinent
7. Mettez en place une gestion des erreurs
8. Documentez les formats d’arguments attendus
9. Pensez à la composition des Invites
10. Testez les Invites avec diverses entrées

<div id="ui-integration">
  ## Intégration de l’UI
</div>

Les Invites peuvent être mises en avant dans les interfaces client sous forme de :

- Commandes « slash »
- Actions rapides
- Éléments de menu contextuel
- Entrées de la palette de commandes
- Parcours guidés
- Formulaires interactifs

<div id="updates-and-changes">
  ## Mises à jour et changements
</div>

Les serveurs peuvent notifier les clients des modifications d’Invites :

1. Fonctionnalité du serveur : `prompts.listChanged`
2. Notification : `notifications/prompts/list_changed`
3. Le client récupère à nouveau la liste des Invites

<div id="security-considerations">
  ## Considérations de sécurité
</div>

Lors de la mise en œuvre des invites :

- Validez tous les arguments
- Nettoyez/sanisez les saisies utilisateur
- Envisagez une limitation du taux (rate limiting)
- Mettez en place des contrôles d’accès
- Auditez l’utilisation des invites
- Gérez correctement les données sensibles
- Validez le contenu généré
- Implémentez des délais d’expiration (timeouts)
- Prenez en compte les risques d’injection d’invites
- Documentez les exigences de sécurité
