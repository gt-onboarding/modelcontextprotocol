<div id="what-is-mcp">
  ## Qu'est-ce que MCP ?
</div>

MCP (Model Context Protocol) est une norme permettant aux applications et agents d’IA de se connecter à vos sources de données (p. ex. fichiers locaux, bases de données ou dépôts de contenu) et à vos outils (p. ex. GitHub, Google Maps ou Puppeteer) et de travailler avec eux.

Voyez MCP comme un adaptateur universel pour les applications d’IA, à l’image de l’USB‑C pour les appareils physiques. L’USB‑C sert d’adaptateur universel pour connecter des appareils à divers périphériques et accessoires. De même, MCP offre un moyen standardisé de connecter les applications d’IA à différentes données et à divers outils.

Avant l’USB‑C, il fallait des câbles différents pour des connexions différentes. De même, avant MCP, les développeurs devaient créer des connexions personnalisées pour chaque source de données ou outil avec lequel ils voulaient que leur application d’IA fonctionne — un processus long qui aboutissait souvent à des fonctionnalités limitées. Désormais, avec MCP, les développeurs peuvent facilement ajouter des connexions à leurs applications d’IA, rendant celles‑ci bien plus puissantes dès le premier jour.

<div id="why-does-mcp-matter">
  ## Pourquoi le MCP est-il important ?
</div>

<div id="for-ai-application-users">
  ### Pour les utilisateurs d’applications d’IA
</div>

MCP permet à vos applications d’IA d’accéder aux informations et aux outils que vous utilisez au quotidien, les rendant bien plus utiles. Plutôt que d’être limitées à leurs connaissances préexistantes, elles peuvent désormais comprendre vos documents, vos données et votre contexte de travail spécifiques.

Par exemple, en utilisant des serveurs MCP, les applications peuvent accéder à vos documents personnels sur Google Drive ou à des données concernant votre codebase depuis GitHub, offrant une aide plus personnalisée et pertinente selon le contexte.

Imaginez demander à un assistant d’IA : « Résume les notes de la réunion d’équipe de la semaine dernière et planifie des suivis avec tout le monde. »

En s’appuyant sur des connexions à des sources de données via MCP, l’assistant d’IA peut :

* Se connecter à votre Google Drive via un serveur MCP pour lire les notes de réunion
* Déterminer qui nécessite un suivi à partir des notes
* Se connecter à votre calendrier via un autre serveur MCP pour planifier automatiquement les réunions

<div id="for-developers">
  ### Pour les développeurs
</div>

Le MCP réduit le temps et la complexité de développement lors de la création d’applications d’IA devant accéder à diverses sources de données. Avec le MCP, les développeurs peuvent se concentrer sur la conception d’excellentes expériences d’IA plutôt que de recréer sans cesse des connecteurs personnalisés.

Traditionnellement, relier des applications à des sources de données exigeait de bâtir des connexions personnalisées, ad hoc, pour chaque source et chaque application. Cela engendrait un travail considérable et redondant — chaque développeur souhaitant connecter son application d’IA à Google Drive ou Slack devait créer sa propre connexion.

Le MCP simplifie cela en permettant aux développeurs de créer des serveurs MCP pour des sources de données, qui sont ensuite réutilisables par différentes applications. Par exemple, grâce au serveur MCP Google Drive open source, de nombreuses applications peuvent accéder aux données de Google Drive sans que chaque développeur ait à créer une connexion personnalisée.

Cet écosystème open source de serveurs MCP permet aux développeurs de capitaliser sur l’existant plutôt que de repartir de zéro, ce qui facilite la création d’applications d’IA puissantes qui s’intègrent parfaitement aux outils et aux sources de données sur lesquels leurs utilisateurs s’appuient déjà.

<div id="how-does-mcp-work">
  ## Comment fonctionne MCP ?
</div>

<Frame>
  <img src="/images/mcp-simple-diagram.png" />
</Frame>

MCP crée un pont entre vos applications d’IA et vos données grâce à un système simple :

* Les **serveurs MCP** se connectent à vos sources de données et à vos outils (comme Google Drive ou Slack)
* Les **clients MCP** sont exécutés par des applications d’IA (comme Claude Desktop) pour les connecter à ces serveurs
* Lorsque vous donnez votre autorisation, votre application d’IA découvre les serveurs MCP disponibles
* Le modèle d’IA peut ensuite utiliser ces connexions pour lire des informations et effectuer des actions

Ce système modulaire permet d’ajouter de nouvelles capacités sans modifier les applications d’IA elles-mêmes — comme ajouter de nouveaux accessoires à votre ordinateur sans mettre à niveau tout votre système.

<div id="who-creates-and-maintains-mcp-servers">
  ## Qui crée et maintient les serveurs MCP ?
</div>

Les serveurs MCP sont développés et maintenus par :

* Des développeurs chez Anthropic qui créent des serveurs pour des outils et des sources de données courants
* Des contributeurs open source qui créent des serveurs pour les outils qu’ils utilisent
* Des équipes de développement en entreprise qui construisent des serveurs pour leurs systèmes internes
* Des éditeurs de logiciels qui rendent leurs applications prêtes pour l’IA

Une fois qu’un serveur MCP open source est créé pour une source de données, il peut être utilisé par n’importe quelle application d’IA compatible MCP, contribuant à un écosystème de connexions en pleine croissance. Consultez notre [liste d’exemples de serveurs](/fr/examples), ou [commencez à créer votre propre serveur](/fr/quickstart/server).
