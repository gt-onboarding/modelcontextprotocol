<div id="core-mcp-concepts">
  ### Concepts clés de MCP
</div>

Les serveurs MCP peuvent offrir trois principaux types de fonctionnalités :

1. **[Ressources](/fr/docs/learn/server-concepts#resources)** : des données de type fichier que les clients peuvent lire (comme des réponses d’API ou des contenus de fichiers)
2. **[Outils](/fr/docs/learn/server-concepts#tools)** : des fonctions que le LLM peut appeler (avec l’approbation de l’utilisateur)
3. **[Invite](/fr/docs/learn/server-concepts#prompts)** : des modèles prédéfinis qui aident les utilisateurs à accomplir des tâches spécifiques

Ce tutoriel se concentrera principalement sur les outils.

    ### Journalisation dans les serveurs MCP

    Lors de l’implémentation de serveurs MCP, faites attention à la manière dont vous gérez la journalisation :

    **Pour les serveurs utilisant STDIO :** N’écrivez jamais sur la sortie standard (stdout). Cela inclut :

    * Instructions `print()` en Python
    * `console.log()` en JavaScript
    * `fmt.Println()` en Go
    * Fonctions stdout similaires dans d&#39;autres langages

    Écrire sur stdout corrompra les messages JSON-RPC et fera tomber votre serveur.

    **Pour les serveurs reposant sur HTTP :** La journalisation sur la sortie standard convient, car elle n’interfère pas avec les réponses HTTP.

    ### Bonnes pratiques

    1. Utilisez une bibliothèque de journalisation qui écrit sur stderr ou dans des fichiers.

    ### Exemples rapides

    ```python
    # ❌ À éviter (STDIO)
    print("Traitement de la requête")

    # ✅ Recommandé (STDIO)
    import logging
    logging.info("Traitement de la requête")
    ```

    ## Créer votre serveur

    ### Importation des packages et configuration de l’instance

    Ajoutez ceci au début de votre `weather.py` :

    ```python
    from typing import Any
    import httpx
    from mcp.server.fastmcp import FastMCP

    # Initialiser le serveur FastMCP
    mcp = FastMCP("weather")

    # Constantes
    NWS_API_BASE = "https://api.weather.gov"
    USER_AGENT = "weather-app/1.0"
    ```

    La classe FastMCP utilise les annotations de type Python et les docstrings pour générer automatiquement des définitions d’outils, ce qui facilite la création et la maintenance des Outils MCP.

    ### Fonctions d’aide

    Ensuite, ajoutons nos fonctions utilitaires pour interroger et formater les données provenant de l’API du National Weather Service :

    ```python
    async def make_nws_request(url: str) -> dict[str, Any] | None:
        """Effectuer une requête vers l’API NWS avec une gestion des erreurs appropriée."""
        headers = {
            "User-Agent": USER_AGENT,
            "Accept": "application/geo+json"
        }
        async with httpx.AsyncClient() as client:
            try:
                response = await client.get(url, headers=headers, timeout=30.0)
                response.raise_for_status()
                return response.json()
            except Exception:
                return None

    def format_alert(feature: dict) -> str:
        """Formater une alerte sous forme de chaîne lisible."""
        props = feature["properties"]
        return f"""
    Événement : {props.get('event', 'Inconnu')}
    Zone : {props.get('areaDesc', 'Inconnue')}
    Gravité : {props.get('severity', 'Inconnue')}
    Description : {props.get('description', 'Aucune description disponible')}
    Consignes : {props.get('instruction', 'Aucune consigne spécifique fournie')}
    """
    ```

    ### Implémentation de l’exécution des Outils

    Le gestionnaire d’exécution des outils est chargé d’exécuter concrètement la logique de chaque outil. Ajoutons-le :

    ```python
    @mcp.tool()
    async def get_alerts(state: str) -> str:
        """Obtenir les alertes météo pour un État américain.

        Args:
            state: Code d’État américain à deux lettres (ex. : CA, NY)
        """
        url = f"{NWS_API_BASE}/alerts/active/area/{state}"
        data = await make_nws_request(url)

        if not data or "features" not in data:
            return "Impossible d’obtenir les alertes ou aucune alerte trouvée."

        if not data["features"]:
            return "Aucune alerte active pour cet État."

        alerts = [format_alert(feature) for feature in data["features"]]
        return "\n---\n".join(alerts)

    @mcp.tool()
    async def get_forecast(latitude: float, longitude: float) -> str:
        """Obtenir les prévisions météo pour un lieu.

        Args:
            latitude: Latitude du lieu
            longitude: Longitude du lieu
        """
        # First get the forecast grid endpoint
        points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
        points_data = await make_nws_request(points_url)

        if not points_data:
            return "Impossible d’obtenir les données de prévision pour ce lieu."

        # Get the forecast URL from the points response
        forecast_url = points_data["properties"]["forecast"]
        forecast_data = await make_nws_request(forecast_url)

        if not forecast_data:
            return "Impossible d’obtenir la prévision détaillée."

        # Mettre les périodes en forme pour une lecture facile
        periods = forecast_data["properties"]["periods"]
        forecasts = []
        for period in periods[:5]:  # Afficher uniquement les 5 prochaines périodes
            forecast = f"""
    {period['name']}:
    Température : {period['temperature']}°{period['temperatureUnit']}
    Vent : {period['windSpeed']} {period['windDirection']}
    Prévisions : {period['detailedForecast']}
    """
            forecasts.append(forecast)

        return "\n---\n".join(forecasts)
    ```

    ### Exécution du serveur

    Enfin, initialisons et lançons le serveur :

    ```python
    if __name__ == "__main__":
        # Initialiser et lancer le serveur
        mcp.run(transport='stdio')
    ```

    ## Création de votre serveur

    ### Importation des packages et configuration de l’instance

    Ajoutez ceux-ci en haut de votre `src/index.ts` :

    ```typescript
    import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
    import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
    import { z } from "zod";

    const NWS_API_BASE = "https://api.weather.gov";
    const USER_AGENT = "application-meteo/1.0";

    // Créer une instance de serveur
    const server = new McpServer({
      name: "weather",
      version: "1.0.0",
      capabilities: {
        resources: {},
        tools: {},
      },
    });
    ```

    ### Fonctions d’assistance

    Ensuite, ajoutons nos fonctions utilitaires pour interroger et formater les données de l’API du National Weather Service :

    ```typescript
    // Fonction utilitaire pour effectuer des requêtes à l’API NWS
    async function makeNWSRequest<T>(url: string): Promise<T | null> {
      const headers = {
        "User-Agent": USER_AGENT,
        Accept: "application/geo+json",
      };

      try {
        const response = await fetch(url, { headers });
        if (!response.ok) {
          throw new Error(`Erreur HTTP ! Statut : ${response.status}`);
        }
        return (await response.json()) as T;
      } catch (error) {
        console.error("Erreur lors de la requête NWS :", error);
        return null;
      }
    }

    interface AlertFeature {
      properties: {
        event?: string;
        areaDesc?: string;
        severity?: string;
        status?: string;
        headline?: string;
      };
    }

    // Mise en forme des données d’alerte
    function formatAlert(feature: AlertFeature): string {
      const props = feature.properties;
      return [
        `Événement : ${props.event || "Inconnu"}`,
        `Zone : ${props.areaDesc || "Inconnue"}`,
        `Gravité : ${props.severity || "Inconnue"}`,
        `Statut : ${props.status || "Inconnu"}`,
        `Titre : ${props.headline || "Aucun titre"}`,
        "---",
      ].join("\n");
    }

    interface ForecastPeriod {
      name?: string;
      temperature?: number;
      temperatureUnit?: string;
      windSpeed?: string;
      windDirection?: string;
      shortForecast?: string;
    }

    interface AlertsResponse {
      features: AlertFeature[];
    }

    interface PointsResponse {
      properties: {
        forecast?: string;
      };
    }

    interface ForecastResponse {
      properties: {
        periods: ForecastPeriod[];
      };
    }
    ```

    ### Mise en œuvre de l’exécution des outils

    Le gestionnaire d’exécution d’outils est chargé d’exécuter la logique de chaque outil. Ajoutons-le :

    ```typescript
    // Enregistrer les outils météo
    server.tool(
      "get_alerts",
      "Obtenir les alertes météo pour un État",
      {
        state: z.string().length(2).describe("Code d’État à deux lettres (p. ex. CA, NY)"),
      },
      async ({ state }) => {
        const stateCode = state.toUpperCase();
        const alertsUrl = `${NWS_API_BASE}/alerts?area=${stateCode}`;
        const alertsData = await makeNWSRequest<AlertsResponse>(alertsUrl);

        if (!alertsData) {
          return {
            content: [
              {
                type: "text",
                text: "Impossible de récupérer les données d’alertes",
              },
            ],
          };
        }

        const features = alertsData.features || [];
        if (features.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: `Aucune alerte active pour ${stateCode}`,
              },
            ],
          };
        }

        const formattedAlerts = features.map(formatAlert);
        const alertsText = `Alertes actives pour ${stateCode} :\n\n${formattedAlerts.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: alertsText,
            },
          ],
        };
      },
    );

    server.tool(
      "get_forecast",
      "Obtenir les prévisions météo pour un lieu",
      {
        latitude: z.number().min(-90).max(90).describe("Latitude du lieu"),
        longitude: z
          .number()
          .min(-180)
          .max(180)
          .describe("Longitude du lieu"),
      },
      async ({ latitude, longitude }) => {
        // Récupérer les données du point de grille
        const pointsUrl = `${NWS_API_BASE}/points/${latitude.toFixed(4)},${longitude.toFixed(4)}`;
        const pointsData = await makeNWSRequest<PointsResponse>(pointsUrl);

        if (!pointsData) {
          return {
            content: [
              {
                type: "text",
                text: `Impossible de récupérer les données du point de grille pour les coordonnées : ${latitude}, ${longitude}. Cet emplacement peut ne pas être pris en charge par l’API NWS (seuls les emplacements aux États-Unis sont pris en charge).`,
              },
            ],
          };
        }

        const forecastUrl = pointsData.properties?.forecast;
        if (!forecastUrl) {
          return {
            content: [
              {
                type: "text",
                text: "Impossible d’obtenir l’URL de prévision à partir des données du point de grille",
              },
            ],
          };
        }

        // Récupérer les données de prévision
        const forecastData = await makeNWSRequest<ForecastResponse>(forecastUrl);
        if (!forecastData) {
          return {
            content: [
              {
                type: "text",
                text: "Impossible de récupérer les données de prévision",
              },
            ],
          };
        }

        const periods = forecastData.properties?.periods || [];
        if (periods.length === 0) {
          return {
            content: [
              {
                type: "text",
                text: "Aucune période de prévision disponible",
              },
            ],
          };
        }

        // Format forecast periods
        const formattedForecast = periods.map((period: ForecastPeriod) =>
          [
            `${period.name || "Inconnu"} :`,
            `Température : ${period.temperature || "Inconnu"}°${period.temperatureUnit || "F"}`,
            `Vent : ${period.windSpeed || "Inconnu"} ${period.windDirection || ""}`,
            `${period.shortForecast || "Aucune prévision disponible"}`,
            "---",
          ].join("\n"),
        );

        const forecastText = `Prévisions pour ${latitude}, ${longitude} :\n\n${formattedForecast.join("\n")}`;

        return {
          content: [
            {
              type: "text",
              text: forecastText,
            },
          ],
        };
      },
    );
    ```

    ```typescript
      console.error("Erreur critique dans main() :", error);
      process.exit(1);
    });
    ```

    ### Journalisation dans les serveurs MCP

    Lors de la mise en œuvre de serveurs MCP, soyez attentif à la façon dont vous gérez la journalisation :

    **Pour les serveurs utilisant STDIO :** N’écrivez jamais sur la sortie standard (stdout). Cela inclut :

    * Instructions `print()` en Python
    * `console.log()` dans JavaScript
    * `fmt.Println()` dans Go
    * Fonctions stdout similaires dans d’autres langages

    Écrire sur stdout corrompra les messages JSON-RPC et fera tomber votre serveur.

    **Pour les serveurs basés sur HTTP :** La journalisation sur la sortie standard convient, car elle n’interfère pas avec les réponses HTTP.

    ### Bonnes pratiques

    1. Utilisez une bibliothèque de journalisation qui écrit sur stderr ou dans des fichiers.
    2. Assurez-vous que toute bibliothèque de journalisation configurée n’écrit pas sur STDOUT

    ```kotlin
        // Créer l’instance du serveur MCP avec une implémentation basique
        val server = Server(
    ```

    Ensuite, ajoutons des fonctions et des classes de données pour interroger l’API du National Weather Service et convertir ses réponses :

    ```kotlin
    // Fonction d’extension pour récupérer les informations de prévision pour une latitude et une longitude données
    suspend fun HttpClient.getForecast(latitude: Double, longitude: Double): List<String> {
        val points = this.get("/points/$latitude,$longitude").body<Points>()
        val forecast = this.get(points.properties.forecast).body<Forecast>()
        return forecast.properties.periods.map { period ->
            """
                ${period.name}:
                Température : ${period.temperature} ${period.temperatureUnit}
                Vent : ${period.windSpeed} ${period.windDirection}
                Prévisions : ${period.detailedForecast}
            """.trimIndent()
        }
    }

    // Fonction d’extension pour récupérer les alertes météorologiques pour un État donné
    suspend fun HttpClient.getAlerts(state: String): List<String> {
        val alerts = this.get("/alerts/active/area/$state").body<Alert>()
        return alerts.features.map { feature ->
            """
                Événement : ${feature.properties.event}
                Zone : ${feature.properties.areaDesc}
                Sévérité : ${feature.properties.severity}
                Description : ${feature.properties.description}
                Consignes : ${feature.properties.instruction}
            """.trimIndent()
        }
    }

    @Serializable
    data class Points(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val forecast: String)
    }

    @Serializable
    data class Forecast(
        val properties: Properties
    ) {
        @Serializable
        data class Properties(val periods: List<Period>)

        @Serializable
        data class Period(
            val number: Int, val name: String, val startTime: String, val endTime: String,
            val isDaytime: Boolean, val temperature: Int, val temperatureUnit: String,
            val temperatureTrend: String, val probabilityOfPrecipitation: JsonObject,
            val windSpeed: String, val windDirection: String,
            val shortForecast: String, val detailedForecast: String,
        )
    }

    @Serializable
    data class Alert(
        val features: List<Feature>
    ) {
        @Serializable
        data class Feature(
            val properties: Properties
        )

        @Serializable
        data class Properties(
            val event: String, val areaDesc: String, val severity: String,
            val description: String, val instruction: String?,
        )
    }
    ```

    ### Mise en œuvre de l’exécution des Outils

    Le gestionnaire d’exécution d’outils est chargé d’exécuter la logique de chaque outil. Ajoutons-le :

    ```kotlin
    // Créer un client HTTP avec une configuration de requête par défaut et la négociation de contenu JSON
    val httpClient = HttpClient {
        defaultRequest {
            url("https://api.weather.gov")
            headers {
                append("Accept", "application/geo+json")
                append("User-Agent", "WeatherApiClient/1.0")
            }
            contentType(ContentType.Application.Json)
        }
    // Installer le plugin de négociation de contenu pour la sérialisation/désérialisation JSON
        install(ContentNegotiation) { json(Json { ignoreUnknownKeys = true }) }
    }

    // Enregistrer un outil pour récupérer les alertes météo par État
    server.addTool(
        name = "get_alerts",
        description = """
            Récupérer les alertes météo pour un État américain. En entrée : code d’État à deux lettres (p. ex. CA, NY)
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("state") {
                    put("type", "string")
                    put("description", "Code d’État à deux lettres (p. ex. CA, NY)")
                }
            },
            required = listOf("state")
        )
    ) { request ->
        val state = request.arguments["state"]?.jsonPrimitive?.content
        if (state == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("Le paramètre « state » est requis."))
            )
        }

        val alerts = httpClient.getAlerts(state)

        CallToolResult(content = alerts.map { TextContent(it) })
    }

    // Enregistrer un outil pour récupérer les prévisions météo par latitude et longitude
    server.addTool(
        name = "get_forecast",
        description = """
            Récupérer les prévisions météo pour une latitude/longitude donnée
        """.trimIndent(),
        inputSchema = Tool.Input(
            properties = buildJsonObject {
                putJsonObject("latitude") { put("type", "number") }
                putJsonObject("longitude") { put("type", "number") }
            },
            required = listOf("latitude", "longitude")
        )
    ) { request ->
        val latitude = request.arguments["latitude"]?.jsonPrimitive?.doubleOrNull
        val longitude = request.arguments["longitude"]?.jsonPrimitive?.doubleOrNull
        if (latitude == null || longitude == null) {
            return@addTool CallToolResult(
                content = listOf(TextContent("Les paramètres « latitude » et « longitude » sont requis."))
            )
        }

        val forecast = httpClient.getForecast(latitude, longitude)

        CallToolResult(content = forecast.map { TextContent(it) })
    }
    ```

    Après avoir exécuté `dotnet new console`, un nouveau projet C# sera créé.

    Vous pouvez également créer une application C# à l’aide de l’[assistant de projet de Visual Studio](https://learn.microsoft.com/en-us/visualstudio/get-started/csharp/tutorial-console?view=vs-2022).
    Après avoir créé le projet, ajoutez le package NuGet pour le SDK et l’hébergement du Model Context Protocol (MCP).

    ```bash
    # Ajouter le package NuGet Model Context Protocol SDK
    dotnet add package ModelContextProtocol --prerelease
    # Ajouter le package NuGet .NET Hosting
    dotnet add package Microsoft.Extensions.Hosting
    ```

    Passons maintenant à la création de votre serveur.

    ## Créer votre serveur

    Ouvrez le fichier `Program.cs` dans votre projet et remplacez son contenu par le code suivant :

    ```csharp
    using Microsoft.Extensions.DependencyInjection;
    using Microsoft.Extensions.Hosting;
    using ModelContextProtocol;
    using System.Net.Http.Headers;

    var builder = Host.CreateEmptyApplicationBuilder(settings: null);

    builder.Services.AddMcpServer()
        .WithStdioServerTransport()
        .WithToolsFromAssembly();

    builder.Services.AddSingleton(_ =>
    {
        var client = new HttpClient() { BaseAddress = new Uri("https://api.weather.gov") };
        client.DefaultRequestHeaders.UserAgent.Add(new ProductInfoHeaderValue("weather-tool", "1.0"));
        return client;
    });

    var app = builder.Build();

    await app.RunAsync();
    ```

    <Note>
      Lors de la création de `ApplicationHostBuilder`, veillez à utiliser `CreateEmptyApplicationBuilder` plutôt que `CreateDefaultBuilder`. Cela garantit que le serveur n’écrit aucun message supplémentaire dans la console. Cette précaution n’est nécessaire que pour les serveurs utilisant le transport STDIO.
    </Note>

    Ce code met en place une application console de base qui utilise le SDK du Model Context Protocol pour créer un serveur MCP avec le transport E/S standard.

    ### Fonctions utilitaires de l’API Météo

    Créez une classe d’extension pour `HttpClient` afin de simplifier la gestion des requêtes JSON :

    ```csharp
    using System.Text.Json;

    internal static class HttpClientExt
    {
        public static async Task<JsonDocument> ReadJsonDocumentAsync(this HttpClient client, string requestUri)
        {
            using var response = await client.GetAsync(requestUri);
            response.EnsureSuccessStatusCode();
            return await JsonDocument.ParseAsync(await response.Content.ReadAsStreamAsync());
        }
    }
    ```

    Ensuite, définissez une classe avec les gestionnaires d’exécution d’outils pour interroger l’API du National Weather Service et convertir ses réponses :

    ```csharp
    using ModelContextProtocol.Server;
    using System.ComponentModel;
    using System.Globalization;
    using System.Text.Json;

    namespace QuickstartWeatherServer.Tools;

    [McpServerToolType]
    public static class WeatherTools
    {
        [McpServerTool, Description("Obtenir les alertes météo pour un État américain.")]
        public static async Task<string> GetAlerts(
            HttpClient client,
            [Description("L’État américain pour lequel récupérer les alertes.")] string state)
        {
            using var jsonDocument = await client.ReadJsonDocumentAsync($"/alerts/active/area/{state}");
            var jsonElement = jsonDocument.RootElement;
            var alerts = jsonElement.GetProperty("features").EnumerateArray();

            if (!alerts.Any())
            {
                return "Aucune alerte active pour cet État.";
            }

            return string.Join("\n--\n", alerts.Select(alert =>
            {
                JsonElement properties = alert.GetProperty("properties");
                return $"""
                        Événement : {properties.GetProperty("event").GetString()}
                        Zone : {properties.GetProperty("areaDesc").GetString()}
                        Gravité : {properties.GetProperty("severity").GetString()}
                        Description : {properties.GetProperty("description").GetString()}
                        Consignes : {properties.GetProperty("instruction").GetString()}
                        """;
            }));
        }

        [McpServerTool, Description("Obtenir les prévisions météo pour un lieu.")]
        public static async Task<string> GetForecast(
            HttpClient client,
            [Description("Latitude du lieu.")] double latitude,
            [Description("Longitude du lieu.")] double longitude)
        {
            var pointUrl = string.Create(CultureInfo.InvariantCulture, $"/points/{latitude},{longitude}");
            using var jsonDocument = await client.ReadJsonDocumentAsync(pointUrl);
            var forecastUrl = jsonDocument.RootElement.GetProperty("properties").GetProperty("forecast").GetString()
                ?? throw new Exception($"Aucune URL de prévisions fournie par {client.BaseAddress}points/{latitude},{longitude}");

            using var forecastDocument = await client.ReadJsonDocumentAsync(forecastUrl);
            var periods = forecastDocument.RootElement.GetProperty("properties").GetProperty("periods").EnumerateArray();

            return string.Join("\n---\n", periods.Select(period => $"""
                    {period.GetProperty("name").GetString()}
                    Température : {period.GetProperty("temperature").GetInt32()}°F
                    Vent : {period.GetProperty("windSpeed").GetString()} {period.GetProperty("windDirection").GetString()}
                    Prévisions : {period.GetProperty("detailedForecast").GetString()}
                    """));
        }
    }
    ```

<div id="whats-happening-under-the-hood">
  ## Ce qui se passe sous le capot
</div>

Lorsque vous posez une question :

1. Le Client MCP envoie votre question à Claude
2. Claude analyse les Outils disponibles et décide lesquels utiliser
3. Le Client MCP exécute les Outils choisis via le Serveur MCP
4. Les résultats sont renvoyés à Claude
5. Claude formule une réponse en langage naturel
6. La réponse s’affiche pour vous !

<div id="troubleshooting">
  ## Dépannage
</div>

<AccordionGroup>
  <Accordion title="Problèmes d’intégration de Claude for Desktop">
    **Récupérer les logs de Claude for Desktop**

    La journalisation de Claude.app liée au MCP est écrite dans des fichiers de log situés dans `~/Library/Logs/Claude` :

    * `mcp.log` contient les journaux généraux sur les connexions MCP et les échecs de connexion.
    * Les fichiers nommés `mcp-server-SERVERNAME.log` contiennent les journaux d’erreurs (stderr) du serveur correspondant.

    Vous pouvez exécuter la commande suivante pour lister les logs récents et suivre les nouveaux en temps réel :

    ```bash
    # Vérifier les logs de Claude pour des erreurs
    tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
    ```

    **Le serveur n’apparaît pas dans Claude**

    1. Vérifiez la syntaxe de votre fichier `claude_desktop_config.json`
    2. Assurez-vous que le chemin vers votre projet est absolu et non relatif
    3. Redémarrez complètement Claude for Desktop

    **Les appels d’outils échouent silencieusement**

    Si Claude tente d’utiliser les Outils mais qu’ils échouent :

    1. Consultez les logs de Claude pour identifier des erreurs
    2. Vérifiez que votre serveur se build et s’exécute sans erreurs
    3. Essayez de redémarrer Claude for Desktop

    **Rien de tout cela ne fonctionne. Que faire ?**

    Veuillez consulter notre [guide de débogage](/fr/legacy/tools/debugging) pour des outils de diagnostic plus efficaces et des instructions plus détaillées.
  </Accordion>

  <Accordion title="Problèmes avec l’API météo">
    **Erreur : échec de la récupération des données de point de grille**

    Cela signifie généralement que :

    1. Les coordonnées sont en dehors des États-Unis
    2. L’API du NWS rencontre des problèmes
    3. Vous êtes soumis à une limitation de débit (rate limiting)

    Correctifs :

    * Vérifiez que vous utilisez des coordonnées situées aux États-Unis
    * Ajoutez un petit délai entre les requêtes
    * Consultez la page d’état de l’API du NWS

    **Erreur : aucune alerte active pour [STATE]**

    Ce n’est pas une erreur : cela signifie simplement qu’il n’y a pas d’alerte météo en cours pour cet État. Essayez un autre État ou revenez lors d’un épisode de mauvais temps.
  </Accordion>
</AccordionGroup>
