Les serveurs fournissent les éléments fondamentaux pour ajouter du contexte aux modèles de langage via
le Model Context Protocol (MCP). Ces primitives permettent des interactions riches entre clients, serveurs et modèles
de langage :

* **Invites** : Modèles ou instructions prédéfinis qui guident les interactions
  du modèle de langage
* **Ressources** : Données ou contenus structurés qui fournissent un contexte supplémentaire au modèle
* **Outils** : Fonctions exécutables qui permettent aux modèles d’effectuer des actions ou de récupérer
  de l’information

Chaque primitive se résume dans la hiérarchie de contrôle suivante :

| Primitive | Contrôle                   | Description                                             | Exemple                          |
| --------- | -------------------------- | ------------------------------------------------------- | -------------------------------- |
| Invites   | Contrôlé par l’utilisateur | Modèles interactifs lancés au choix de l’utilisateur    | Commandes « slash », options de menu |
| Ressources | Contrôlé par l’application | Données contextuelles associées et gérées par le client | Contenu de fichiers, historique Git |
| Outils    | Contrôlé par le modèle     | Fonctions exposées au LLM pour effectuer des actions    | Requêtes POST d’API, écriture de fichiers |
