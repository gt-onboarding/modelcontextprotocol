<Info>**Révision du protocole** : 2024-11-05</Info>

Le Protocole de contexte de modèle (MCP) prend en charge, au besoin, le suivi de la progression des opérations de longue durée au moyen de messages de notification. Chaque partie peut envoyer des notifications de progression pour fournir des mises à jour sur l’état de l’opération.

<div id="progress-flow">
  ## Flux de progression
</div>

Lorsqu’une partie souhaite _recevoir_ des mises à jour sur la progression d’une requête, elle inclut un
`progressToken` dans les métadonnées de la requête.

- Les jetons de progression **DOIVENT** être une valeur de chaîne ou d’entier
- Les jetons de progression peuvent être choisis par l’expéditeur par n’importe quel moyen, mais **DOIVENT** être uniques
  pour toutes les requêtes actives.

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "some_method",
  "params": {
    "_meta": {
      "progressToken": "abc123"
    }
  }
}
```

Le destinataire **PEUT** ensuite envoyer des notifications de progression contenant :

- Le jeton de progression d’origine
- La valeur de progression actuelle
- Une valeur « total » facultative

```json
{
  "jsonrpc": "2.0",
  "method": "notifications/progress",
  "params": {
    "progressToken": "abc123",
    "progress": 50,
    "total": 100
  }
}
```

- La valeur `progress` **DOIT** augmenter à chaque notification, même si la valeur totale est
  inconnue.
- Les valeurs `progress` et `total` **PEUVENT** être des nombres à virgule flottante.

<div id="behavior-requirements">
  ## Exigences de comportement
</div>

1. Les notifications d’avancement DOIVENT uniquement faire référence à des jetons qui :
   - Ont été fournis dans une requête active
   - Sont associés à une opération en cours

2. Les destinataires des requêtes d’avancement PEUVENT :
   - Choisir de ne pas envoyer de notifications d’avancement
   - Envoyer des notifications à la fréquence qu’ils jugent appropriée
   - Omettre la valeur totale si elle est inconnue

```mermaid
sequenceDiagram
    participant Sender
    participant Receiver

    Note over Sender,Receiver: Requête avec jeton d’avancement
    Sender->>Receiver: Method request with progressToken

    Note over Sender,Receiver: Mises à jour d’avancement
    loop Mises à jour d’avancement
        Receiver-->>Sender: Notification d’avancement (0.2/1.0)
        Receiver-->>Sender: Notification d’avancement (0.6/1.0)
        Receiver-->>Sender: Notification d’avancement (1.0/1.0)
    end

    Note over Sender,Receiver: Opération terminée
    Receiver->>Sender: Method response
```

<div id="implementation-notes">
  ## Notes de mise en œuvre
</div>

- Les émetteurs et les récepteurs **DEVRAIENT** suivre les jetons de progression actifs
- Les deux parties **DEVRAIENT** appliquer une limitation de débit pour éviter la saturation
- Les notifications de progression **DOIVENT** s’arrêter une fois l’opération terminée
