Les serveurs offrent les blocs de base essentiels pour ajouter du contexte aux modèles de langage via le Protocole de contexte de modèle (MCP). Ces primitives permettent des interactions riches entre clients, serveurs et modèles de langage :

* **Invites** : Modèles ou instructions prédéfinis qui guident les interactions avec le modèle de langage
* **Ressources** : Données structurées ou contenu offrant un contexte supplémentaire au modèle
* **Outils** : Fonctions exécutables qui permettent aux modèles d’effectuer des actions ou de récupérer de l’information

Chaque primitive se résume selon la hiérarchie de contrôle suivante :

| Primitive | Contrôle                   | Description                                           | Exemple                          |
| --------- | -------------------------- | ----------------------------------------------------- | -------------------------------- |
| Invites   | Contrôlé par l’utilisateur | Modèles interactifs invoqués au choix de l’utilisateur | Commandes « slash », options de menu |
| Ressources | Contrôlé par l’application | Données contextuelles jointes et gérées par le client | Contenu de fichiers, historique Git |
| Outils    | Contrôlé par le modèle     | Fonctions exposées au LLM pour effectuer des actions  | Requêtes POST d’API, écriture de fichiers |
