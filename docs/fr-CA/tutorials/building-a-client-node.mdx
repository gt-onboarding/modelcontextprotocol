  ## Configuration requise

  Avant de commencer, assurez-vous que votre système satisfait à ces exigences :

  * Ordinateur Mac ou Windows
  * Node.js version 16 ou plus installée
  * npm (fourni avec Node.js)

  ## Configuration de l’environnement

  Commencez par créer un nouveau projet Node.js :

  ```bash
  # Créer le répertoire du projet
  mkdir mcp-client
  cd mcp-client

  # Initialiser le projet npm
  npm init -y

  # Installer les dépendances
  npm install @modelcontextprotocol/sdk @anthropic-ai/sdk dotenv
  npm install -D typescript @types/node

  # Créer la configuration TypeScript
  npx tsc --init

  # Créer les fichiers nécessaires
  mkdir src
  touch src/client.ts
  touch .env
  ```

  Mettez à jour votre fichier `package.json` pour ajouter la configuration nécessaire :

  ```json
  {
    "type": "module",
    "scripts": {
      "build": "tsc",
      "start": "node build/client.js"
    }
  }
  ```

  Mettez à jour votre fichier `tsconfig.json` avec les paramètres appropriés :

  ```json
  {
    "compilerOptions": {
      "target": "ES2022",
      "module": "Node16",
      "moduleResolution": "Node16",
      "outDir": "./build",
      "rootDir": "./src",
      "strict": true,
      "esModuleInterop": true,
      "skipLibCheck": true,
      "forceConsistentCasingInFileNames": true
    },
    "include": ["src/**/*"]
  }
  ```

  ## Configurer votre clé d’API

  Vous aurez besoin d’une clé d’API Anthropic à partir de la [console Anthropic](https://console.anthropic.com/settings/keys).

  Créez un fichier `.env` :

  ```bash
  ANTHROPIC_API_KEY=votre_clef_ici
  ```

  Ajoutez `.env` à votre `.gitignore` :

  ```bash
  echo ".env" >> .gitignore
  ```

  ## Gestion des connexions au serveur

  Ensuite, implémentez la méthode pour vous connecter à un Serveur MCP :

  ```typescript
  async connectToServer(serverScriptPath: string): Promise<void> {
      const isPython = serverScriptPath.endsWith(".py");
      const isJs = serverScriptPath.endsWith(".js");

      if (!isPython && !isJs) {
        throw new Error("Le script serveur doit être un fichier .py ou .js");
      }

      const command = isPython ? "python" : "node";

      this.transport = new StdioClientTransport({
        command,
        args: [serverScriptPath],
      });

      this.client = new Client(
        {
          name: "mcp-client",
          version: "1.0.0",
        },
        {
          capabilities: {},
        }
      );

      await this.client.connect(this.transport);

      // Liste les outils disponibles
      const response = await this.client.request(
        { method: "tools/list" },
        ListToolsResultSchema
      );

      console.log(
        "\nConnecté au serveur avec les outils :",
        response.tools.map((tool: any) => tool.name)
      );
    }
  ```

  ## Logique de traitement des requêtes

  Ajoutez maintenant la fonctionnalité principale pour traiter les requêtes et gérer les appels d’outils :

  ```typescript
  async processQuery(query: string): Promise<string> {
      if (!this.client) {
        throw new Error("Client not connected");
      }

      // Initialiser le tableau de messages avec la requête de l’utilisateur
      let messages: Anthropic.MessageParam[] = [
        {
          role: "user",
          content: query,
        },
      ];

      // Obtenir les Outils disponibles
      const toolsResponse = await this.client.request(
        { method: "tools/list" },
        ListToolsResultSchema
      );

      const availableTools = toolsResponse.tools.map((tool: any) => ({
        name: tool.name,
        description: tool.description,
        input_schema: tool.inputSchema,
      }));

      const finalText: string[] = [];
      let currentResponse = await this.anthropic.messages.create({
        model: "claude-3-5-sonnet-20241022",
        max_tokens: 1000,
        messages,
        tools: availableTools,
      });

      // Traiter la réponse et tout appel d’outil
      while (true) {
        // Ajouter la réponse de Claude au texte final et aux messages
        for (const content of currentResponse.content) {
          if (content.type === "text") {
            finalText.push(content.text);
          } else if (content.type === "tool_use") {
            const toolName = content.name;
            const toolArgs = content.input;

            // Exécuter l’appel d’outil
            const result = await this.client.request(
              {
                method: "tools/call",
                params: {
                  name: toolName,
                  arguments: toolArgs,
                },
              },
              CallToolResultSchema
            );

            finalText.push(
              `[Appel de l’outil ${toolName} avec les arguments ${JSON.stringify(toolArgs)}]`
            );

            // Ajouter la réponse de Claude (incluant l’utilisation d’un outil) aux messages
            messages.push({
              role: "assistant",
              content: currentResponse.content,
            });

            // Ajouter le résultat de l’outil aux messages
            messages.push({
              role: "user",
              content: [
                {
                  type: "tool_result",
                  tool_use_id: content.id,
                  content: [
                    { type: "text", text: JSON.stringify(result.content) },
                  ],
                },
              ],
            });

            // Obtenir la prochaine réponse de Claude avec les résultats de l’outil
            currentResponse = await this.anthropic.messages.create({
              model: "claude-3-5-sonnet-20241022",
              max_tokens: 1000,
              messages,
              tools: availableTools,
            });

            // Ajouter l’interprétation des résultats de l’outil par Claude au texte final
            if (currentResponse.content[0]?.type === "text") {
              finalText.push(currentResponse.content[0].text);
            }

            // Continuer la boucle pour traiter tout autre appel d’outil
            continue;
          }
        }

        // S’il n’y a pas d’appel d’outil dans la réponse, arriver ici
        break;
      }

      return finalText.join("\n");
    }
  ```

  ## Interface de clavardage interactive

  Ajoutez la boucle de conversation et la fonctionnalité de nettoyage :

  ```typescript
  async chatLoop(): Promise<void> {
      console.log("\nClient MCP démarré !");
      console.log("Tapez vos requêtes ou « quit » pour quitter.");

      // Using Node's readline for console input
      const rl = readline.createInterface({
        input: process.stdin,
        output: process.stdout,
      });

      const askQuestion = () => {
        rl.question("\nRequête : ", async (query: string) => {
          try {
            if (query.toLowerCase() === "quit") {
              await this.cleanup();
              rl.close();
              return;
            }

            const response = await this.processQuery(query);
            console.log("\n" + response);
            askQuestion();
          } catch (error) {
            console.error("\nErreur :", error);
            askQuestion();
          }
        });
      };

      askQuestion();
    }

    async cleanup(): Promise<void> {
      if (this.transport) {
        await this.transport.close();
      }
    }
  ```

  ## Point d’entrée principal

  Enfin, ajoutez la logique d’exécution principale à l’extérieur de la classe :

  ```typescript
  // Main execution
  async function main() {
    if (process.argv.length < 3) {
      console.log("Utilisation : ts-node client.ts <chemin_vers_le_script_du_serveur>");
      process.exit(1);
    }

    const client = new MCPClient();
    try {
      await client.connectToServer(process.argv[2]);
      await client.chatLoop();
    } catch (error) {
      console.error("Erreur :", error);
      await client.cleanup();
      process.exit(1);
    }
  }

  // Exécuter main si ceci est le module principal
  if (import.meta.url === new URL(process.argv[1], "file:").href) {
    main();
  }

  export default MCPClient;
  ```

  ## Exécuter le client

  Pour exécuter votre client avec n’importe quel Serveur MCP :

  ```bash
  # Compilez le code TypeScript. Assurez-vous de relancer cette commande chaque fois que vous mettez à jour « client.ts » !
  npm run build

  # Exécutez le client
  node build/client.js path/to/server.py  # pour les serveurs Python
  node build/client.js path/to/server.js  # pour les serveurs Node.js
  ```

  Le client :

  1. Se connecter au serveur indiqué
  2. Répertorier les outils disponibles
  3. Démarrez une session de clavardage interactive où vous pouvez :
     * Saisir des requêtes
     * Voir l’exécution des outils
     * Obtenir des réponses de Claude

  ## Explication des composants clés

  #### 1. Initialisation du client

  * La classe `MCPClient` s’initialise avec la gestion de session et des clients API
  * Configure le client MCP avec des capacités de base
  * Configure le client Anthropic pour les interactions avec Claude

  #### 2. Connexion au serveur

  * Prend en charge les serveurs Python et Node.js
  * Valide le type de script du serveur
  * Configure des canaux de communication adéquats
  * Répertorie les Outils disponibles lors de la connexion

  #### 3. Traitement des requêtes

  * Maintient le contexte de la conversation
  * Gère les réponses de Claude et les appels d’outils
  * Gère l’acheminement des messages entre Claude et les Outils
  * Combine les résultats en une réponse cohérente

  #### 4. Interface interactive

  * Fournit une interface simple en ligne de commande
  * Gère les saisies de l’utilisateur et affiche les réponses
  * Comprend une gestion des erreurs de base
  * Permet une terminaison en douceur

  #### 5. Gestion des ressources

  * Nettoyage adéquat des ressources
  * Gestion des erreurs en cas de problèmes de connexion
  * Procédures d’arrêt en douceur

  ### Points de personnalisation courants

  1. **Gestion des outils**
     * Modifier `processQuery()` pour gérer des types d’outils spécifiques
     * Ajouter une gestion des erreurs personnalisée pour les appels d’outils
     * Mettre en œuvre un formatage de réponse propre à l’outil

  2. **Traitement des réponses**
     * Personnaliser la mise en forme des résultats d’outils
     * Ajouter un filtrage ou une transformation des réponses
     * Mettre en place une journalisation personnalisée

  3. **Interface utilisateur**
     * Ajouter une interface graphique ou Web
     * Offrir une sortie console enrichie
     * Ajouter l’historique des commandes ou l’autocomplétion

  ### Bonnes pratiques

  1. **Gestion des erreurs**
     * Encapsulez toujours les appels d’Outils dans des blocs try-catch
     * Fournissez des messages d’erreur explicites
     * Gérez les problèmes de connexion de manière élégante

  2. **Gestion des ressources**
     * Utilisez des méthodes de nettoyage appropriées
     * Fermez les connexions une fois terminé
     * Gérez les déconnexions du serveur

  3. **Sécurité**
     * Rangez les clés d’API de façon sécuritaire dans `.env`
     * Validez les réponses du serveur
     * Soyez prudent avec les autorisations des outils

  ### Dépannage

  #### Problèmes de chemin d’accès du serveur

  * Vérifiez à nouveau le chemin vers votre script de serveur
  * Utilisez des chemins absolus si les chemins relatifs ne fonctionnent pas.
  * Pour les utilisateurs de Windows, utilisez des barres obliques (/) ou des barres obliques inverses échappées (\\)
  * Vérifiez que le fichier serveur a la bonne extension (.py ou .js)

  Exemple d’utilisation correcte du chemin d’accès :

  ```bash
  # Chemin relatif
  node build/client.js ./server/weather.js

  # Chemin absolu
  node build/client.js /Users/username/projects/mcp-server/weather.js

  # Chemin Windows (les deux formats fonctionnent)
  node build/client.js C:/projects/mcp-server/weather.js
  node build/client.js C:\\projects\\mcp-server\\weather.js
  ```

  #### Problèmes de connexion

  * Vérifiez que le script du serveur existe et qu’il a les autorisations appropriées
  * Vérifiez que le script du serveur est exécutable
  * Assurez-vous que les dépendances du script serveur sont installées
  * Essayez d’exécuter le script du serveur directement pour vérifier s’il y a des erreurs

  #### Problèmes d’exécution des Outils

  * Vérifiez les journaux du serveur pour des messages d’erreur
  * Vérifier que les paramètres d’entrée de l’outil correspondent au schéma
  * Assurez-vous que les dépendances des Outils sont disponibles
  * Activer la journalisation de débogage pour suivre le déroulement de l’exécution
